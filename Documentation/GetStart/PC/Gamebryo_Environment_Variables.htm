Gamebryo LightSpeed Environment Variables

EMERGENT_PATH is the path to the root installation of Gamebryo LightSpeed. LightSpeed makes use of this key environment variable on the PC during development. The installer will set this variable for you based on where you install LightSpeed, but it is helpful to know about this variable if you're working with multiple releases, or if you're alternating between a pristine installation and an experimental "sandbox", for example.

Examples of how this environment variable is used include:

  • The Gamebryo tools pipeline uses EMERGENT_PATH to find tool plugin DLLs that are needed to function in the following location, by default:

    %EMERGENT_PATH%\SDK\Win32\DLL\ToolPlugins

  • Tools and other applications also find shader libraries, by default, in the following location:

    %EMERGENT_PATH%\SDK\Win32\DLL\ShaderLibs

  • Tools and other applications find shaders, by default, in the following location for DX9, D3D10, D3D11, and Xbox360:

    %EMERGENT_PATH%\Media\Shaders\Data

  • Tools and other applications find shaders, by default, in the following location for Playstation 3:

    %EMERGENT_PATH%\Media\Shaders\Data\PS3

  • NiSample uses EMERGENT_PATH to find user interface textures when they are not found relative to the sample application. Specifically, NiSample::Initialize looks in the following location:

    %EMERGENT_PATH%\Media\Samples\SampleUI

    to locate textures used in UI controls for all Gamebryo demos if they are not found in a path relative to the demo application. Such an approach simplifies creating demos outside of the Gamebryo Samples folder without having to copy the shared user interface textures.
