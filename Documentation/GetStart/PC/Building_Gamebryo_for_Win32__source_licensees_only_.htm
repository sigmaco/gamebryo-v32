Build Guidelines for Win32

Emergent provides projects that you can use to build Gamebryo LightSpeed's major subsystems, libraries, tools, and samples for PC. These projects are available only to customers who purchase the source version of Gamebryo LightSpeed.

Build Order

The following is the recommended build order for Gamebryo LightSpeed.

  1. Foundation
  2. Foundation PhysX
  3. Core Runtime
  4. Game Framework
  5. Game Framework PhysX
  6. Core Runtime Services
  7. Core Runtime PhysX Services
  8. Online
  9. Samples
  10. Tools

Building Foundation

The Foundation libraries are required for all other libraries in Gamebryo. You can find the .sln file for Foundation in the following locations, depending on whether you're using Visual Studio 2005 or 2008:

%EMERGENT_PATH%\Build\Foundation\Win32\VC80\Foundation.sln

%EMERGENT_PATH%\Build\Foundation\Win32\VC90\Foundation.sln

Building Foundation PhysX

The Foundation PhysX libraries are used by all other LightSpeed PhysX libraries. You should have the PhysX SDK installed and the PHYSXINSTALLPATH set. You can find the .sln file for Foundation PhysX in the following locations, depending on whether you're using Visual Studio 2005 or 2008:

%EMERGENT_PATH%\Build\Foundation\Win32\VC80\FoundationPhysX.sln

%EMERGENT_PATH%\Build\Foundation\Win32\VC90\FoundationPhysX.sln

Building Core Runtime

For each section of Gamebryo, there is a solution file that includes all of the project files that are related to it. The solutions are contained in the folders %EMERGENT_PATH%\Build\CoreRuntime\Win32\VC80 or %EMERGENT_PATH%\Build\CoreRuntime\Win32\VC90.

The following solutions are provided:

Solution File Description
Core00_ThirdPartyCode Contains the projects under the Source\3rdParty folder. These projects include LibPNG and zLib.
Core01_EngineLibs Contains the relative projects under the EngineLibs folder. These projects include NiSystem, NiMain, NiMesh, NiDX9Renderer, NiD3D10Renderer, ecrD3D11Renderer, NiAnimation, NiCollision, NiEnvironment, NiFloodgate, NiParticle, NiPortal, and NiTerrain.
Core02_PhysXLibs Builds projects under the Source\CoreRuntime\PhysXLibs folder applicable to Win32. These projects include NiPhysX, NiPhysXParticle, NiPhysXFluid, and NiTerrainPhysX.
Core03_ToolLibs Contains the relative projects under the ToolLibs folder.
Core04_ToolLibsPhysX Builds projects under the Source\CoreRuntime\ToolLibs folder that require PhysX.
Core05_ShaderLibs Contains the relative projects under the ShaderLibs folder. These projects include NiBinaryShaderLib, NSBShaderLib, NSFParserLib, NiD3DXEffectShaderLib, NiD3D10EffectShaderLib, NiD3D10BinaryShaderLib, ecrD3D11EffectShaderLib, ecrD3D11BinaryShaderLib, NiCgShaderLib, NiCommonMaterialLib, NiMaterialNodeXMLLibraryReader.
Core06_UtilityLibs Contains the relative projects under the UtilityLibs folder. These projects include NiCursor, NiFont, NiInput, NiUserInterface, NiVisualTracker, NiSceneGraphUpdateService, NiMetricsOutput, NiSettingsDialog, NiDX9RendererSetup, NiD3D10RendererSetup, ecrD3D11endererSetup, NiBinkVideo, and NiMilesAudio.
Core07_AppFrameworks Contains the relative projects under the AppFrameworks folder. These projects include NiApplication and NiSample.

If you are performing a full rebuild of the source tree, the Core##_* solutions must be built in the order in which they are numbered. Each project in Gamebryo will have knowledge of the location of other projects contained in its 'section'. For example, NiMain will know where to find the header files from NiSystem. Other than their section-mates, they will look to the sdk folder. So, ShaderLibs is dependent on EngineLibs being built prior to it so that the correct header files from the EngineLibs are used when building.

  • Samples solutions depend on the Core solutions
  • Tools solutions also depend on Core solutions
  • There is no dependency relationship between Samples and Tools

Building Game Framework

The Game Framework libraries are required to take advantage of the Emergent entity system and simulator. You can find the .sln file for Game Framework in the following locations, depending on whether you're using Visual Studio 2005 or 2008:

%EMERGENT_PATH%\Build\GameFramework\Win32\VC80\GameFramework.sln

%EMERGENT_PATH%\Build\GameFramework\Win32\VC90\GameFramework.sln

Building Game Framework PhysX

The Game Framework PhysX solution include PhysX services that interact directly with Game Framework. You can find the .sln file for the Game Framework PhysX services in the following locations, depending on whether you're using Visual Studio 2005 or 2008:

%EMERGENT_PATH%\Build\GameFramework\Win32\VC80\GameFrameworkPhysX.sln

%EMERGENT_PATH%\Build\GameFramework\Win32\VC90\GameFrameworkPhysX.sln

Game Framework PhysX requires the efdPhysX library from Foundation.

Building Core Runtime Services

The Game Framework solution includes Core Runtime Services projects. You must build the Core Runtime before building the Game Framework solution. You can find the .sln file for Core Runtime Services in the following locations, depending on whether you're using Visual Studio 2005 or 2008:

%EMERGENT_PATH%\Build\GameFramework\Win32\VC80\CoreServices.sln

%EMERGENT_PATH%\Build\GameFramework\Win32\VC90\CoreServices.sln

You must build Core Runtime and Game Framework before building the Core Runtime Services.

Building Core Runtime PhysX Services

The Core Services PhysX services are in a separate solution. These services provide PhysX functionality the interact with both Game Framework and Core Runtime Services. Examples include the Ragdoll services and the PhysX Terrain Service. You can find the .sln file for Core Runtime PhysX in the following locations, depending on whether you're using Visual Studio 2005 or 2008:

%EMERGENT_PATH%\Build\GameFramework\Win32\VC80\CoreServicesPhysX.sln

%EMERGENT_PATH%\Build\GameFramework\Win32\VC90\CoreServicesPhysX.sln

Building Online

The Online libraries are required to take advantage of the Online subsystem which uses the Replication Service and Replicated Entities. You can find the .sln file for Online in the following locations, depending on whether you're using Visual Studio 2005 or 2008:

%EMERGENT_PATH%\Build\Online\Win32\VC80\Online.sln

%EMERGENT_PATH%\Build\Online\Win32\VC90\Online.sln

Building Samples

The following solutions are available in the folder %EMERGENT_PATH%\Build\Samples\Win32\VC90 for building sample applications:

Solution File Description
Samples01_Tutorials Contains the projects under the Samples\Tutorials\Graphics folder.
Samples02_TutorialsPhysX Contains the projects under the Samples\Tutorials\PhysX folder.
Samples03_Demos Contains the relative projects under the Samples\GraphicsTechDemos folder.
Samples04_DemosPhysX Contains the projects under the Samples\GraphicsTechDemos folder that require PhysX. By convention, the names of these demos start with "PhysX".
Samples05_GameDemos Contains the relative projects under the Samples\GameDemos folder.
Samples06_GameDemosPhysX Contains the projects under the Samples/GameDemos folder that require PhysX. These projects include Lenguins and Dodgeball.
Samples07_MangledMetal Contains the MangledMetalproject, from under the Samples\GameDemos\MangledMetal. This project requires Game Framework to have been built.
Samples08_GameFrameworkSamples Contains the relative projects under the Samples\GameFrameworkTechDemos folder. These projects requires Game Framework to have been built.
Samples09_OnlineSamples Contains the relative projects under the Samples\OnlineTechDemos folder. These projects requires Online to have been built.

Building Tools

The following solutions for building tools are available in the folder %EMERGENT_PATH%\Build\Tools\Win32\VC90:

Solution File Description
Tools01_DeveloperTools Contains the relative projects under the Source\Tools\DeveloperTools folder.
Tools02_DeveloperToolsPhysX Contains the relative projects under the Source\Tools\DeveloperTools folder that require PhysX.
Tools03_AnimationTool Contains the AnimationTool project.
Tools04_AssetViewer Contains the AssetViewer project.
Tools06_ToolPlugins Contains the relative projects under the Source\Tools\ToolPlugins folder.
Tools07_ToolPluginsPhysX Contains the relative projects under the Source\Tools\ToolPlugins folder that require PhysX.
Tools08_MaxPlugin Contains the MaxPlugin project.
Tools09_MayaPlugin Contains the MayaPlugin project.
Tools10_XSIPlugin Contains the XSIPlugin project.
Toolbench Contains the Toolbench project. This solution is only available for VC90.
RevisionControl Contains the RevisionControl C++ support library projects. This solution is only available for VC90.
ToolbenchRevisionControl Contains the Toolbench plug-ins that integrate into revision control. This solution is only available for VC90.
ToolbenchFBX Contains the Toolbench plug-ins that integrate with FBX libraries to export entities. This solution is only available for VC90.
