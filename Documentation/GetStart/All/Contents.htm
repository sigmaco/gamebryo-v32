Gamebryo Directory Orientation

The Gamebryo codebase contains a large amount of code, and it can be difficult at first to understand how the engine is organized on disk. Before the directory structure of the engine can be explored, it is important to understand how Gamebryo is organized functionally.

Gamebryo's functionality is broken down into a number of subsystems, of which the primary subsystems are Foundation, CoreRuntime, and GameFramework.

Many high-level directories in the engine, such as Build, Samples, and Source, are further broken down by subsystem.

Top-level Directory

Directory Contents
Build Build files for each supported platform.
Documentation Product documentation.
Media Common media files, including shaders and other runtime assets
Samples Demos, tutorials, models, and sample game applications that demonstrate Gamebryo usage.
sdk Pre-compiled libraries and headers.
Source Source code for the engine, including runtime libraries and tools.

Build Directory

The Build directory contains the project files that are required to build Gamebryo on each supported platform. They are organized, first, by subsystem, and then by platform, and finally by supported compiler. The build projects within each of these lowest-level directories are named to indicate the correct build order, so building the projects successively should result in a fresh build of Gamebryo.

Refer to the Build documentation for information on how to build Gamebryo.

Documentation Directory

The documentation directory contains the product documentation (which you are currently viewing). The documentation's start page is: %EMERGENT_PATH%\Documentation\HTML\documentation.htm.

Media Directory

The Media directory contains art and game data that are shared across several applications. It also contains xml files for data driven enums that generate unique ids at run time.

Samples Directory

The Samples directory contains examples of Gamebryo usage. For more information on how this directory is organized, see Samples Directory Orientation.

SDK Directory

Gamebryo headers and libraries are found in the sdk directory. The sdk directory contains a subdirectory for each supplied Gamebryo platform:

  • sdk\Win32 for Windows
  • sdk\PS3 for PS3
  • sdk\Xbox360 for Xbox360

Each of these platform-specific subdirectories includes all of the header files for the Gamebryo libraries. The libraries are built as static libraries on all platforms, with dynamic link libraries (DLLs) also supplied for Win32. See the platform-specific release notes for details about each platform.

Source Directory

The Source directory contains source code for Gamebryo's runtime libraries and tools. For more information on how this directory is organized, see Source Directory Orientation.
