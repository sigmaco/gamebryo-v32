Known Issues and Limitations

This section lists unexpected issues and limitations in the D3D11 renderer. For a list of features and platform limitations, see the Features and Limitations of the D3D11 Renderer.

General Notes

All Releases

  • There is a signed/unsigned mismatch between the blend indices used on skinning geometry and the shaders that are used to render them. This will generate a D3D11 warning in ID3D11Device::CreateInputLayout that can be safely ignored.
  • Any NiRect passed into NiRenderer::ClearBuffer function will be ignored, and the entire buffer will be cleared instead.
  • The D3D11Renderer does not record any metrics.
  • Refer to D3D11 Feature Support in the D3D11Renderer for discussion of the support of D3D11-specific features.
  • D3D11 requires that the input parameters of pixel or geometry shaders exactly match the outputs of the previous shader stage, with the exception of system-generated values. This means that, for example, the input to pixel shaders must include an entry with the SV_POSITION semantic, since such a parameter must be output from a vertex or geometry shader.
  • D3D11 does not allow shader programs lower than shader model 4.0 to be compiled. To compile shader programs written to earlier models, pass in the D3D10_SHADER_ENABLE_BACKWARDS_COMPATIBILITY flag and compile for shader model 4.0, and use the appropriate "vs_4_0_level_9_*" or "ps_4_0_level_9_*" shader targets. The D3D11BinaryShaderLib does this automatically when it encounters such shader programs.
  • Stream Output has been disabled in the D3D11 renderer because of unexpected difficulties getting the system working.
