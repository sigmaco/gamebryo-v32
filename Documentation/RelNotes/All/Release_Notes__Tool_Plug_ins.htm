Added Features, Changed Behavior, and Fixed Bugs

Gamebryo LightSpeed 3.2

  • Fixed an issue which could sometimes cause the mesh instancing plug-in to delete meshes from an exported scene.

Gamebryo LightSpeed 3.1.1

  • Fixed some issues with per-sequence and per-object-per-sequence animation compression that could cause it to not compress or even expand the size of data stored.

Gamebryo LightSpeed 3.1

  • Changed the behavior of the on-target viewers to adjust the aspect ratio of all cameras found in the scene to match the aspect ratio of the display monitor. Previously, only the default camera was adjusted, and any exported cameras were left at their original aspect ratio (typically 4:3).
  • The SceneGraphOptimization tool was modified to have an option to remove partial path names. The names of nodes were changed so that they are always unique within a single scene. If this is not desired you can set this option to shorten the name to only be that of the object.

Gamebryo LightSpeed 3.0

  • Fixed an issue with the PS3 on-target viewer where it failed to unregister the TTY callback when shut down (which would manifest as an error dialog reporting "Failed to register TTY stream").
  • Stripify Geometry plug-in will be deprecated in the next release. Vertex cache optimization process that is done in the SceneGraphOptimization plug-in is a superior alternative.
  • Fixed a random crash bug during the vertex cache optimization phase.
  • Added the Material Modifier Plug-in.
  • Removed n-squared sized allocation from animation compression increasing export efficiency.
  • Added ThumbnailGenerator tool plug-in to generate thumbnails for previewing in the tools and the Windows shell.
  • Fixed a bug in code used by Scene Graph Optimization that was causing sibling mesh objects to be incorrectly merged, despite having different extra data. This sometimes caused incorrect export of PhysX collision shapes, among other things.

Gamebryo 2.6

  • The PhysX Extractor tool plug-in marks meshes as non-optimizable if PhysX requires the vertex ordering to remain unchanged. This enables PhysX Extractor and Scene Graph Optimization to be used in any order.
  • Modified the Controller Extractor plug-in such that it generates KF files which contain NiSequenceData objects instead of NiControllerSequence objects. Gamebryo 2.6 utilizes two classes to represent animated sequences: NiSequenceData and NiControllerSequence. NiSequenceData objects contain the shareable, heavyweight data of exported animation sequences saved in KF files. NiControllerSequence objects are temporary runtime entities created upon sequence activation. These lighter weight objects maintain instance-based state to play back the animated sequence on a particular actor. Older versions of Gamebryo utilized a single class, also named NiControllerSequence, which handled both the shareable data as well as the instance-based state. This improvement reduces the runtime memory footprint when sequences are shared by multiple characters. To enhance runtime performance the new sequence format relies upon NiEvaluator objects as opposed to NiInterpolator objects for sampling sequence animation data. Gamebryo 2.6 supports KF files from Gamebryo 1.2 and later. If needed, Gamebryo automatically converts to the new NiSequenceData format during the loading process.
  • The PhysX Ragdoll tool plug-in no longer makes any assumption about the ordering of actors in a joint when setting actor parent pointers. This fixes problems with excessive jerkiness in some rag-dolls.
  • Modified the Controller Extractor plug-in such that during animation compression, specifying a compression override per sequence or object will properly override the global compression settings.
  • Changed the Controller Extractor plug-in such that it no longer consolidates NiTransformController objects into a single NiMultiTargetTransformController. Instead, NiTransformController objects are removed from the scene graph. These controllers are no longer required for sequence animation since the NiMultiTargetPoseHandler communicates directly with scene graph transform nodes.
  • Modified how the Controller Extractor plug-in handles the transform on the accumulation root in NIF files. The plug-in moves the transform on the accumulation root to its non-accumulation child node such that the root's transform is always identity within the NIF file. This change in behavior makes the transforms stored in NIF files consistent with any exported sequences. By definition, the NIF file stores a constant scene graph (i.e. a pose rather than an animation). Since poses never use accumulation flags the accumulation root in the NIF file must have an identity transform.
  • Removed from the Controller Extractor plug-in the ability to store controllers in KF files. This option had previously been provided to generate KF files that were compatible with NIF files exported from a version earlier than Gamebryo 1.2. Pre-1.2 NIF files did not contain any time controllers for their sequence objects; the KF files did instead. In contrast, Gamebryo 1.2 NIF files contain the time controllers and the KF files do not (they only contain interpolators). The KF files produced by the Controller Extractor are compatible with NIF files from Gamebryo 1.2 and later. This bumped the plug-in version to 2.6.
  • When specifying a compression setting per object or sequence in the Controller Extractor plug-in, that setting will now override the global compression setting in all cases. Previously, if the global compression setting specified 'no compression,' the override would not be applied.

Gamebryo 2.5

  • Updated the Fur Generator Plug-in to be a generic fur generation utility.
  • Fixed the Save Source Texture Renderer Data plug-in to work properly with Tool Plug-in Batch.
  • Added an option to the Controller Extractor tool plug-in to enable the removal of NiPSResetOnLoopCtlr objects from particle systems involved in sequences, thus disabling particle system looping.
  • Added MeshFinalize plug-in to facilitate the Mesh Profile System.
  • Updated SceneGraphOptimization and CreateABV plug-ins to support new NiMesh library.
  • Added vertex cache and view independent optimizations to the SceneGraphOptimization plug-in. This optimization is mutually exclusive with stripification and generally gives better performance so using vertex cache optimization instead of stripification is recommended and the new default behavior.
  • Corrected the behavior of the Cancel button on the script editor dialog. The Cancel button now leaves the original script unchanged regardless of what is done in the dialog before cancel is used. Operations such as adding new scripts or renaming scripts are not affected by this change.
  • Changed Multi NIF Exporter to insert any cameras within the sub scene to be exported into the stream after the scenegraph in order to make behavior match that of the NIF Exporter.
  • Removed "Triangle to Strip Ratio" check from AssetAnalyzer plug-in. Since the Stripify Geometry plug-in uses degenerate triangles to connect meshes instead of creating additional strips, this check is no longer necessary.
  • Removed deprecated scene statistics feature from Scene Viewer D3D.

Gamebryo 2.3.0

  • Added the ability to the Controller Extractor plug-in to exclude specified types of controllers from being included in sequences. This bumped the plug-in version to 2.3.
  • The Stripify Geometry plug-in now ensures that portal object dependencies are honored when replacing meshes with stripified meshes.
  • Added the ability to launch the PhysX Nif Viewer on Xbox directly from Max or Maya. See the documentation for the Xbox 360 Viewer tool plug-in.
  • Added the ability to launch the PhysX Nif Viewer on PS3 directly from Max or Maya. See the documentation for the PS3 Viewer tool plug-in.

Gamebryo 2.2.1

  • Fixed issue where Scene Graph Optimization plug-in would inappropriately remove UV texture sets from instanced geometry.
  • Fixed issue where Portal Converter plug-in would not always identify tagged portals exported from Maya.
  • Fixed memory leaks occurring in Portal Converter plug-in.
  • Fixed issue in Portal Converter plug-in where lights that affected rooms would loose their targets.

Gamebryo 2.2.0

  • Modified the NifExporter and NifImporter tool plug-ins to support streaming extra objects outside of the scene graph. The version number of these plug-ins has been updated accordingly.
  • A new SaveSourceTextureRendererData plug-in has been added, in support of new capabilities for direct-loading textures on supported platforms. This functionality is designed to increase loading performance, reduce the load-time memory footprint, improve general memory usage, and allow for platform-specific extensions. See also Texture Streaming Options for Internal Textures, the export options for 3ds max or Maya, and the description of NiPersistentSrcTextureRendererData in the Main Class Reference for more details.
  • Added the PhysX Cleaner tool plug-in to remove the extra data produced when PhysX content is exported from Max or Maya but is not required by the application.
  • Added the PhysX Extractor tool plug-in to export PhysX content authored in Max or Maya.
  • Added the PhysX Maya Ragdoll tool plug-in to help with setting up rag-dolls in Maya. The tool plug-in re-associates PhysX actors with skeleton bones.
  • Added the PhysX NIF Viewer viewer plug-in to launch the PhysX NIF Viewer tool for previewing PhysX content.

Gamebryo 2.1.0

  • Fixed a bug in the Controller Extractor export plug-in that could result in a crash upon export if some sequences don't get created because they don't contain any animation keys but other sequences do. Now, an export error is output listing the sequences that did not created.

Gamebryo 2.0.1

  • Addressed two issues in the Xenon On-Target Viewer that were causing failure of file copies to the remote Xenon development console.

Gamebryo 2.0.0

  • Removed all NiSceneGamePad related code from the SceneViewerDX9DLL plug-in. This code has been deprecated for several releases and has been known to cause crashes on certain models of consumer PC gamepads.
  • Removed many platform-specific plugins and tool libraries for platforms that are no longer supported, such as SceneViewerDX8DLL, TextureSwizzleDLL, NiTextureSwizzle and NiExportPlatformGeometry, and the on-target viewers for the unsupported consoles.
  • Added support for animation compression to Controller Extractor.
  • Replaced _access with NiFile::Access in NifExporter and NiControllerExtractor plugins.
  • Added a new method to NiPlugin that determines whether or not the plugin can instantiate a default version of its options. This allows a plug-in to not appear in the UI for adding a new plug-in to a script.
  • The PortalConverterDLL has been rewritten for this release and has been assigned the version number 2.0. It now uses a more robust convex hull finder, which results in more accurate and simpler room definitions (i.e. fewer wall objects per room). It can also handle multiple independently-tagged room groups per file/scene. Finally, a bug that caused manually-tagged walls to be ignored has been fixed, allowing artists to avoid automatic wall generation and generate an optimal set of wall planes manually.

Known Issues and Limitations

MaterialModifier

  • The MaterialModifier plugin will not resize compressed (dds) textures unless a full mipmap pyramid is supplied within the texture.

MultiNifExport

  • If MultiNif Export is used in conjunction with Scene Graph Optimizations, it is possible for the resulting NIF to have an NiMesh as a root level object. For some systems, this can be problematic. To work ensure that the top level object is an NiNode, from within the art package, tag any node destined for MultiNif export with NiOptimizeKeep. This will ensure that the node is not optimized out.

Controller Extractor

  • If a sequence is set to accumulate rotations and its "front facing" axis is not the positive x-axis, the Controller Extractor plug-in will modify the animation keys appropriately to ensure that the sequence will work well with other sequences that do not accumulate rotations about that axis. If your accumulation root has a nonzero local rotation around the non-accumulated axes, then this process may incorrectly calculate the modified animation keys. To avoid this, ensure that the accumulation root for your character has identity rotations on all axes except the accumulated axis.

Mesh Instancing

  • Gamebryo currently does not support hardware skinning and mesh instanced objects. If the plug-in detects meshes that have hardware skinning attached they will be ignored. CPU skinned objects are supported.
  • Morphed objects and CPU skinned objects in a scene that are instances (exact copies) will always be converted to hardware instances, with the result that all the instances have exactly the same animation at runtime. In effect, there is only one object that remains for animation purposes, and all the others are copies. Do not use the Mesh Instancing plug-in if you would like independently controlled morphed or CPU skinned objects, or model the morphed and skinned objects in a separate file from the object you would like hardware instanced.

Normal Unifier

  • The Normal Unifier only works if the two adjacent objects which are to have their normals unified share the same world transform.
