Multiplayer Rapid Iteration: Script Debugging

Using the Script Debugger during a multiplayer rapid iteration session is fully supported.

Whenever a breakpoint is encountered in any instance of the game, all instances of the game will pause simulation execution. At this point, you can step through your script and examine it in the usual way. Resuming execution will cause all instances to resume simulation at the point they were at before the breakpoint was hit.

It is often helpful to know which instance of the game is the one that reached your breakpoint. To help with this, the NetID of the instance that triggered the break is reported in the Toolbench Output View each time the Script Debugger breaks. You will need to toggle on the Debug channel in the Output View UI and check the Details column for the process ID associated with the breakpoint notification. You will see a message similar to: Script Debugger breakpoint reached -- Break process ID 4.
