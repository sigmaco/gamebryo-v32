Multiplayer Rapid Iteration: Introduction

Multiplayer rapid iteration describes the rapid iteration workflow when you are working with two or more instances of an in-development online game. In this scenario, one instance of the game typically serves as the Host, while other instances act as Peers. All of the instances are connected for rapid iteration, and changes made in Toolbench are simultaneously updated in all of the game instances.

We use the PeerGame sample application to demonstrate multiplayer rapid iteration, with two instances of the game running (one Host and one Peer).
