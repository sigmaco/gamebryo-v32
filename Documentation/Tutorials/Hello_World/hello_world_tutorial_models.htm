Hello World: Creating New Models

A model is a data-driven description of a class of Entity from which many Entity instances are created and initialized. A model describes an Entity's properties, behaviors, and relationships to other models. In this section, you'll use the Entity Modeling Tool (EMT) to create new models. (Later, you'll create Entities in the World Builder that are based on these models.)

Creating Your First Model

To create a model, first double-click the HelloWorld.emtproj item in the Game Solution browser in Toolbench. This opens Project Explorer. Select the HelloWorld folder, and then right-click and select Add > New Model from the context menu.

Adding new model to HelloWorld

Tip

If you do not see the left-hand pane in the EMT Project Explorer, click the "Folders" button to display/hide this pane.

This creates a new entry in your Project Explorer in the HelloWorld folder. By default, the entry will be called "Model." In this tutorial, this model will represent a floor tile that you'll use to build your level. So, go ahead now and rename the model to "HallFloor".

Tip

When you create the model, it will have focus in the Project Explorer. So, you should be able to rename it simply by typing. If you need to, you can right-click the model to rename it.

Mix-in Standard Model Library Mesh Model

Ensure that you're in the Model Editor and not the Diagram Editor by looking for the "Entity Model" tag just below the HelloWorld.emtproj tab. Your new HallFloor model is represented in the Model Editor as a small box. You now need to mix in the Standard Model Library's Mesh model so that the HallFloor model can render in your scene. To do this, expand StandardModelLibrary in Project Explorer and click and drag the Mesh model onto your HallFloor model. The new mix-in relationship between your HallFloor model and the Mesh model is now represented in the Model Editor.

Mixing in Standard Model Library Mesh model

Override the NifAsset Property

We've mixed in the Standard Model Library Mesh model. Now we need to specify a NIF file that will be associated with our new model. To do this, we'll update the mix-in properties for our HallFloor model and override the "NifAsset" property. First, click to select the HallFloor model in the EMT diagram.

In the Entity Model Editor, click the Mixin Properties tab. Here, you can see all of the properties that are inherited from parent models, sorted by their parent model.

Overriding the NifAsset property

Expand the Mesh category of Mixin Properties and click the checkbox beside the "NifAsset" property, click the ellipsis button (...) beside the default value text field, and then browse to your media directory (C:\Emergent\HelloWorldTutorial\Media\Graphics). Select the HallFloor.nif file.

Associating an asset with the NifAsset property

Mixing in and overriding properties is a powerful way to model Entities in your application.

Creating the Character Model

Now that you know how to create a new model using the EMT, let's create a model that represents our character in the world.

  1. In EMT Project Explorer, add a new model by right-clicking the HelloWorld folder and selecting Add > New Model from the context menu.
  2. Give your new model the name "Jack".
  3. Mix in the Standard Model Library's Actor model. (An actor is an in-game Entity that has animations.)
  4. Override Jack's "KfmAsset" property and set it to Jack.kfm from the Graphics directory.
  5. Save your work.
