Make a copy of BaseGame project and assets

The LightSpeed demo project directories contain the complete project and assets for the BaseGame demo. Those files are copied to a separate location so that you can modify BaseGame to add online functionality.

From a command prompt that has the EMERGENT_PATH environment variable set correctly, run the CopySampleAssets.bat script, which you can find in the PeerGame source directory. The script takes one argument, which specifies the directory to which you want to copy the files:

cd %EMERGENT_PATH%\Samples\OnlineTechDemos\PeerGame 
CopySampleAssets.bat C:\Emergent\PeerGameTutorial

Add OnlineModelLibrary EMT Project

You need to add the OnlineModelLibrary EMT project to your copy of the BaseGame solution. This will allow Toolbench to use these models.

Launch Toolbench, then open the solution file in C:\Emergent\PeerGameTutorial\Samples\GameFramework\BaseGame\BaseGame.gsl.

Right-click the solution, choose Add > Existing Content from the context menu, and browse to the OnlineModelLibrary\OnlineModelLibrary.emtproj file.

Now we need to add references to the OnlineModelLibrary EMT project within the BaseGame EMT project. This enables us to create models in the BaseGame EMT project that mix-in the models from OnlineModelLibrary. Double-click the BaseGame EMT project to open it, then expand the project so you can see the References folder. Right-click the folder and choose Add Reference, then browse to OnlineModelLibrary.emtproj.

Now, right-click BaseGame.block in the Game Solution View, choose Rename, and change the name of the file to PeerGame.block.

Moving on...

Okay, we've copied the BaseGame solution and prepared it for the addition of online functionality. To add this functionality, we'll mix-in some new models from the OnlineModelLibrary to the existing BaseGame model hierarchy. See Extending Entities for Online.
