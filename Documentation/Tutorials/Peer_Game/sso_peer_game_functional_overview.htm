Peer Game Demo Functional Overview

The Peer Game application is driven by the Gamebryo Game Framework, a few C++ files that are specific to the application, and some application-specific data, including script files for implementing game logic. It expands on the Base Game application by adding simple matchmaking and online gameplay.

The application contains one level that includes the following:

  • Four rooms (one hidden)
  • Ccomputer-controlled characters (called AIJack)
  • One player-controlled character (called Avatar) for each connected player
  • Multiple PhysX-based triggers
  • Doors that open and close
  • A box that players can pick up

The level and the entities used by the application are created in Toolbench. The entity behaviors are written in Lua.

The primary purpose of Peer Game is to illustrate an application based on the Emergent Game Framework that includes networking features. Peer Game demonstrates:

  • Programmer features:
    • Rudimentary matchmaking—the first process started on a system will put itself into host mode and listen for connections from other application instances
    • How to set up entity replication
    • How to propagate behaviors to remote peers to activate peer-side functionality such as playing an animation
