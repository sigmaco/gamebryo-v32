Base Game: C++ Project Changes

This section describes project changes that we will make to support the new features in Base Game. You can either create a project from scratch, or you can reuse and modify the Hello World project and source files. Regardless of which path you take, you will have a Visual Studio project that includes the libraries and include paths that Base Game needs.

Creating Your Project from Scratch

If you did not follow the Hello World tutorial, copy one of the template .vcproj files from the following path to your working directory.

%EMERGENT_PATH%\Documentation\HTML\Tutorials\Base_Game\templates

Then, create Main.cpp, BaseGame.h, and BaseGame.cpp using the code from BaseGame Tutorial Code Samples. Save these files in your working directory. In this tutorial, the working directory is C:\Emergent\BaseGameTutorial.

Creating Your Project Based on the Hello World Tutorial and Source Files

If you followed the Hello World tutorial, you can reuse your existing Visual Studio project and source files. Copy the .sln, .vcproj, .cpp, and .h files to your working directory.

Note

This tutorial refers to the BaseGame.h and .cpp files. If you're starting from the Hello World tutorial, these are functionally equivalent to the HelloWorld.h and .cpp files created in the Hello World tutorial. You don't have to rename these files / classes for this tutorial, just remember that they're equivalent.

If you're re-using your existing Hello World Visual Studio project, you also need to make some updates to the project to support PhysX in Base Game.

Additional Include Paths

We need to add some additional include search paths to the project. The paths we need are:

$(PHYSXINSTALLPATH)/SDKs/PhysXLoader/include;$(PHYSXINSTALLPATH)/SDKs/Cooking/include;$(PHYSXINSTALLPATH)/SDKs/Physics/include;$(PHYSXINSTALLPATH)/SDKs/Foundation/include

Additional Link Library Path

We'll be including PhysX libraries, so add the following to your link search path:

$(PHYSXINSTALLPATH)/SDKs/lib/win32
