Product Tour: Data Driven Games

Gamebryo LightSpeed supports the development of data-driven games. In these games, most of the content is loaded from data files and is not hard-coded into the game executable. Whether you are a member of a 200+ person big-budget team or a member of a smaller start-up, LightSpeed tools and workflows enable you to negotiate the complexity of getting content flowing into your game.

Guiding Principles

Gamebryo LightSpeed is built upon the following principles:

  • Rapid iteration—change the art, entities, levels, and behaviors and see the effect in your game without recompiling. This minimizes the need for game restarts and reduces development delays.
  • Rapid prototyping and production—quickly move game content into a playable form, enabling you to evaluate technology and gameplay mechanics and assess the look and feel of levels and assets, and build on that work throughout the production cycle.
  • One workflow from prototype to publish

For more information about the design principles that underpin Gamebryo LightSpeed, see Gamebryo LightSpeed Architecture Design Principles.

The Tools

You'll be exploring most of the LightSpeed tools as you proceed through this tour. To start you off, here is a brief introduction to the principal tools that we'll discuss:

  • Toolbench—An extensible platform for hosting game content tools.
  • Entity Modeling Tool—A plug-in to Toolbench that allows you to create game object definitions.
  • World Builder—A plug-in to Toolbench that allows you to create game levels by placing game objects.
  • Script Debugger—A plug-in to Toolbench that allows you to create and debug game logic scripts
  • DCC exporters - Integrated exporters for 3ds Max, Maya, and XSI that allow artists to create 3D game content.
  • Asset Viewer—A tool that enables you to view 3D game content files outside of the game executable. (Note that the name of this tool is a bit of a misnomer; you can only view NIF file assets in the tool.)
  • Animation Tool—A tool that you can use to view and manipulate animated characters outside of the game executable.

The Game Framework

At the heart of the game code is a modular, extensible Game Framework. Within the Framework, each major unit of functionality is created in one or more system services. For example, all rendering is controlled by the Render Service. Many of these services are already rapid-iteration aware and will automatically accept updates as developers update their data. It is expected that game developers will want to integrate their own services into the Game Framework.

MangledMetal

We provide a very simple demo game called MangledMetal. This game is built on the Game Framework and installs by default in the following location:

%EMERGENT_PATH%\Samples\GameDemos\MangledMetal\

We'll be using this demo as a stand-in for your game throughout this tour.

MangledMetal takes place in a world of mechanical, steam-powered beings called WarJacks from Privateer Press' tabletop role-playing game. The game is essentially a basic dungeon-crawler game with enemies to kill and environments to explore.

At this point, you should launch the MangledMetal game executable. From the Windows Start menu, select Emergent > Gamebryo LightSpeed > GameDemos > MangledMetal.

The following are some highlights of the MangledMetal demo:

  • Character animation tied to user input
  • Particle systems
  • Terrain with defined walkability regions
  • PhysX-enabled props that respond to attacks
  • PhysX-enabled ragdolls when characters die
  • Dynamic shadows
  • Simple Lua-based enemy AI
  • Data-driven gameplay events and responses
  • Asynchronous loading of assets

For now, take some time to explore MangledMetal. For information about how to move your character around the world, see MangledMetal Demo Controls.
