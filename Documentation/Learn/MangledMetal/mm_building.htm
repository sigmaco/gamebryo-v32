Building MangledMetal

MangledMetal on Windows comes with project files to build using Microsoft Visual C++ 2005 (VC8.0) and Microsoft Visual Studio C++ 2008 (VC9.0). All the prerequisites you need to build should be prebuilt in your Gamebryo installation.

Open the Windows solution files in %EMERGENT_PATH%\Build\Samples\Win32. If you are using Visual C++ 2005, use the VC80 directory; if you are using Visual C++ 2008, use the VC90 directory. Then select Samples07_MangledMetal.sln.

Choose your build configuration and select Build > Build Solution from the Visual C++ menu.

As part of the build process, the file "Config.ini" will be copied into the target build directory. The config.ini file controls many features of how MangledMetal will run. See MangledMetal configuration options for more information.

MangledMetal Build Configurations

MangledMetal has many build configurations on various platforms. The MangledMetal prerequisites should be prebuilt with the standard Gamebryo installation, but if you do choose to rebuild the base Gamebryo libraries, remember to build them on the platform, configuration, and Visual C++ version that matches your MangledMetal configuration.

Every platform supports basic Debug, Release, and Shipping configurations. Debug mode has memory debugging turned on, logging enabled by default, and other debugging assistance built in. It is built with few if any compiler optimizations. Release mode has compiler optimizations turned on and memory debugging turned off; logging is compiled in but disabled by default. Shipping mode strips all nonessential code, including logging, and enables maximum compiler optimization.

MangledMetal Deployment

After the project files are built they will, by default, be automatically deployed by platform-specific deploy scripts. Automatic deployment on all platforms can be bypassed by defining an environment variable named EGB_SKIP_DEPLOYMENT_{PlatformName} where {PlatformName} is one of PS3, WIN32 or XENON, and assigning it the value "enabled".

There is a deploy script associated with each project in that project's directory. These files can be customized for the deployment needs of a particular site. Refer to the platform-specific Build Configuration topics below for more details about platform deployment.

About Config.ini

MangledMetal requires a configuration file named Config.ini to be located in the same directory as the executable at runtime. Since configuration files contain platform-specific information, the names of the packaged versions of the files referred to by the projects are qualified by platform name of the form Config_{PlatformName}.ini where {PlatformName} is one of PS3, Win32 or Xbox360.

Since these files are automatically renamed to Config.ini when necessary during the deployment step mentioned above there is no need to edit any of the Config.ini files directly unless the deployment step is bypassed as described. Any desired changes to the configuration can be made in the project version of the file and will be deployed as part of the next project build.
