PeriodicMultiInvoker Model

Found in project: TriggersLibrary

Traits: Active

Mixes In: PeriodicTriggerSource, MultiTargetResponse, Placeable

Description

Complete trigger type which calls multiple trigger targets periodically after being invoked.

Properties

Name Type Source Model Default Value Traits Description
StartDelay UInt32 PeriodicTriggerSource 8000 Delay after BeginPeriodic is called before trigger fires for the first time.
PeriodicInterval Float32 PeriodicTriggerSource 8000 Interval between trigger firings, in milliseconds.
TriggerID String AbstractTriggerSource Identifier for this trigger. This can be useful to script code if it wants to track which trigger source caused an OnTrigger call.
ActivationCount UInt32 AbstractTriggerResponse 0 Number of times this trigger has been activated.
MaxActivations UInt32 AbstractTriggerResponse 1 Maximum number of times this trigger is allowed to be activated. If the TriggerSource detects something that would have triggered, it should not call OnTrigger if ActivationCount >= MaxActivations.
TargetList EntityRef List MultiTargetResponse List List of entities to inform. These entities should all implement the OnTrigger behavior.
Position Point3 Placeable 0, 0, 0 The world-space position of the entity.
Rotation Rotator Placeable 0, 0, 0 The world-space rotation of the entity.
Scale Float32 Placeable 1 The world-space scale of the entity.

Behaviors

Name Target Type Source Model Traits Description
BeginPeriodic Lua PeriodicTriggerSource Called to indicate that the trigger source should begin periodically firing.
CheckContinue Lua PeriodicTriggerSource Called to see if the trigger source should continue firing. A custom implementation can be created to test entity-specific conditions. The default implementation checks for a property 'shoudlContinue'; if this property exists and is 'false', the trigger will stop firing.
