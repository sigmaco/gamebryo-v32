MangledMetal configuration options

MangledMetal uses a single command-line option, Configuration=, to select other runtime options from a Gamebryo configuration file named Config.ini. The value you pass to the Configuration= option selects a different portion of the Config.ini file based on its value. For example, invoking MangledMetal with the command line:

MangledMetal.exe Configuration=EmptyBattlefield

will tell MangledMetal to use the [Data.EmptyBattlefield] section of the configuration file. If no configuration option is specified, MangledMetal will default to the Level03 configuration. Within the selected section, the following entries have meaning to MangledMetal.

Entry Meaning
BlockFile Tells which .xblock file to load. The .xblock file is the output of World Builder exporting a .block level file. Block files that come with MangledMetal can be found at the root of the data directory.
StatsFile Name of the run statistics file to generate. This file generates a list of the percentage time spent in each service, and the absolute time in each service. If you do not specify a name, one will be constructed based on the default block file name and platform. The statistics file is written to the current directory (generally the same directory as the executable file) and has the extension ".xml".
MaxTime If you want to automatically exit after a certain period of time, as you would when running a performance or unit test, you can set MaxTime to a floating point value indicating how long to run in seconds. Setting MaxTime to 0, or not setting it at all, allows the demo to run until stopped.
EnableShadows Set to True or False to enable or disable shadows. The default is True if the value is not set. Note that you can also enable or disable shadows in your world block file; if shadows are disabled in the block file, this value will have no effect.

For the PC version, MangledMetal uses ServiceManager config settings to yield CPU when the game does not have focus. This allows a locally running Toolbench to be more responsive when rapidly iterating on content. These settings can be tuned as desired by increasing or reducing the sleep times in the config block.

; Share CPU with Toolbench when iterating on content
[ServiceManager]
Sleep=0
DeactivatedSleep=100
