MangledMetal character rigging

Player Character

Scene Structure

The Maya file of the Khador Light Warjack is organized into four layers in the scene file, visible from the Channel Box/Layer Editor viewport. These are:

  • Rigid_Bodies layer which contains all the members of the ragdoll group and nimaInternalPhysics object
  • control layer which contains all the members of the control_objects group
  • skinned_geometry layer which contains the actual mesh geometry for the khador_light_warjack, ice_axe, and motion trail
  • joints layer which contains the skinning bones and ice_axe.

Text Keys

The khador_light_warjack uses some specific text and morph keys to communicate certain events to the demo application:

  • Character movement cycles(walk, run, and so on) use morph keys to ensure a smooth transition based on the character's movement speed
  • Attack cycles contains a single text key named 'apex' used to determine when to test if an attack is successful. Without this text key, an attack will not deal damage.

Animation Layers

The khador_light_warjack has animations divided between 4 animation layers. These are

  • The root character node which contains the majority of movement sequences
  • The lower body which contains a single movement sequence
  • The upper body layer which contains the majority of attack sequences
  • The material layer which contains material animation sequences

The lower and upper body animation layers are used to allow the character to move forward while attacking, playing a movement animation on the lower body while playing attack animations on the upper body.

Enemies

Enemy characters follow all of the same conventions as the Player Character with two exceptions:

  • Enemies cannot move and attack at the same time, so they have all their animations on a single layer.
  • Enemy character files have no Control visibility layer. Enemies are rigged to Biped and do not require additional animation control objects.

Process Scripts

All MangledMetal characters are exported using a custom script called MangledMetal PhysX Character. This script is similar to the default character export script with the additional following plugins:

  • PhysX Extractor, with a Scale of .01
  • PhysX Ragdoll, with a Skeleton Node named 'pelvis_j', Ragdoll Node named 'ragdoll', and Postfix named 'RigidBody'
  • Material Modifier
