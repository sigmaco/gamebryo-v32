Introduction to the MeshInstancing Demo

The MeshInstancing demo shows how to use Gamebryo's support for hardware instancing. Due to the overhead involved in executing a single draw call, rendering a scene with a large number of objects can be detrimental to performance when each object requires its own draw call. Hardware instancing is a feature that allows a single mesh to be rendered many times with different transformations in a single draw call. Since only one draw call is required, thousands of objects can be rendered with no additional CPU overhead.

The MeshInstancing demo makes use of Gamebryo's support for hardware instancing to render thousands of recursive gears. The demo allows the user to toggle hardware instancing and dynamic updating of the gears as well as enabling them to adjust the number of gears that are rendered. It should be noted that not all devices have support for hardware instancing. This demo will still run on such hardware, but normal rendering mechanics will be used.
