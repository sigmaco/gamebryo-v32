The ecr Library

The ecr library contains a set of services and support classes that interface between the Game Framework and the Core Runtime. This subset of all Core Runtime services is essential for any application that uses the Game Framework and Core Runtime. For example, the Scene Graph Service manages loading, caching, and cloning scene graphs (NIF files and associated data). The Render Service manages rendering of entities and their scene graphs. Any application that renders a scene will make use of the ecr library.

The following sections discuss each service in the ecr library:

  • The Camera Service provides low level management of cameras.
  • The Light Service manages lighting, including shadows.
  • The Pick Service provides built-ins for picking operations.
  • The Render Service and related classes manage objects for rendering, including the renderer object (platform specific objects of class NiRenderer).
  • The Scene Graph Service manages scene graphs (objects of class NiAVObject). This includes asset management for NIF files, updating of scene graphs and other operations.
  • The Shader Service manages the loading of shaders for rendering.
