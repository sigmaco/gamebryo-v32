Introduction to Object Systems

Gamebryo is based upon several low-level systems, which are used throughout the engine and tools.  These common, low-level classes provide basic behaviors such as object management, reference counting, object persistence, and fast run-time typing.  This manual explains the application-level interfaces that most Gamebryo developers are likely to use.

The systems described here include:

  • Smart pointers (reference counting)
  • Fast run-time type information (RTTI)
  • Scene graph cloning (copying)
  • Streaming (file-based object persistence)
  • Object name strings
  • Management of extra data specific to an application
  • Time controllers (animation)
  • Threading With NiThread

Several of these features are encapsulated in common Gamebryo base classes. These classes include:

  • NiRefObject supports reference counting and smart pointers.
  • NiObject (an NiRefObject-derived class) adds support for RTTI, cloning, streaming and viewer strings.
  • NiObjectNET (an NiObject-derived class) supports names, extra data, and time controllers.

For advanced developers who need more information, information on Object Systems Internals is available.
