Time Controllers

NiObjectNET also provides support for taking actions on itself over time. The base class for these actions is NiTimeController. The relevant members of NiObjectNET are:

  • void PrependController(NiTimeController* pkControl);
  • void SetControllers(NiTimeController* pkControl);
  • NiTimeController* GetControllers() const;
  • void RemoveController(NiTimeController* pkControl);
  • void RemoveAllControllers();

NiObjectNET contains these controllers in an list data structure—in fact, the list implementation is internal to NiTimeController (Each NiTimeController includes a "next" pointer). This list container allows for an ordered traversal of the time-based behaviors for a given NiObjectNET. The time controller functions above allow for time controllers to be added to the head of the time controller list (Prepend), to replace the entire list of controllers (Set), get the head of the controllers list (Get), find and remove a given controller (Remove), and clear the object's time controller list (RemoveAll).

In general, applications should never call PrependController, SetController, or RemoveController to add or remove an NiTimeController from its target. Instead, the NiTimeController::SetTarget function should be used for this purpose, since it will automatically take care of inserting or removing the NiTimeController into the NiObjectNET's list of controllers.

NiTimeController objects are reference counted. (NiTimeController is derived from NiObject, which is derived from NiRefObject.) The member in NiObjectNET that points to the head of the time controller list is a smart pointer. The member in NiTimeController that points to the next NiTimeController object in the list is also a smart pointer. Care must be taken therefore, when detaching NiTimeController objects from an NiObjectNET. The detachment will result in removal of a smart pointer reference to the NiTimeController object, and will therefore delete the NiTimeController object if there are no other smart pointer references to it.

Details on the behaviors of specific time controllers can be found in the documentation for the Animation library.
