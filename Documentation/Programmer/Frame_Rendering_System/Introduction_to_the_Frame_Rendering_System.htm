Introduction to the Frame Rendering System

The Gamebryo frame rendering system provides an easy way to construct the pipeline of rendering operations necessary to perform complex rendering effects within a frame. It is provided in addition to the manual rendering methods that have existed in Gamebryo for some time.

Without this system, applications would be forced to include a good bit of custom code that would be executed each frame to render the appropriate objects to the screen. This code would rarely be portable between applications and could be sometimes difficult to conceptualize.

Applications that use the frame rendering system do all their configuration work up front, during application initialization. Then, at render time, complex effects can be performed automatically and easily swapped out for other effects. Additionally, code that uses the frame rendering system is more portable than standard code, and it is easier to understand the various rendering steps that are occurring during a frame.
