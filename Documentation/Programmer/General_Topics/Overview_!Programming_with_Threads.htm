Threading Overview

With an increasing number of multiprocessor machines available on the market, idle time available while waiting for hardware events like vertical retrace, and the potential for next generation consoles to have multiple processors, there are performance benefits to be gained by using multiple threads in an application. Because of performance impacts that would occur, Gamebryo is not transparently thread safe. This document outlines some reasons why Gamebryo is not completely thread safe under all circumstances, gives guidelines for how to multithread an application safely with Gamebryo, explains how several key areas of Gamebryo are synchronized, and discusses possible problems that can emerge from these approaches.
