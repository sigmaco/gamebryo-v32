Creating Content in the Application Itself

Gamebryo does not require all scene graphs and other data to be preprocessed off-line, although the content export tools optimize data when writing NIF files, by default. As a result, applications may choose to create some of their content on the fly in the application itself.

For example, an application could create a fractal landscape or a randomly generated office interior during application startup, or at any time during the application's lifetime. In this case, content "importation" may only involve reading an application-designed script file or parameter file using C++ streams. In fact, no file reading may be required at all, in which case the geometry data only exists in memory.
