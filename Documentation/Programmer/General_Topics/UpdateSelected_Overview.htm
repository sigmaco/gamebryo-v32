UpdateSelected Overview

NiAVObject::UpdateSelected is similar to NiAVObject::Update. However, UpdateSelected adds per-object control of Update features, which usually results in much faster update times than those for Update. Also, UpdateSelected offers an option for entire subtrees to be treated as rigid bodies, simplifying the updating of world bound information (which Update does not support).

However, using UpdateSelected correctly requires some additional steps specifically, to add information to scene graph nodes that specify known movement patterns for the node. In general, you cannot simply replace a call to Update with UpdateSelected and expect it to work correctly. You must:

  1. Correctly set the UpdateSelected flags (discussed below), either automatically with NiAVObject::SetSelectiveUpdateFlags, or manually with the object flag accessor functions in NiAVObject.
  2. Call Update and UpdateNodeBound once (usually at application startup), before any calls to UpdateSelected. When UpdateSelected is subsequently called, the scene graph will be updated correctly with UpdateSelected's rigid world bound update.

Please refer to the Update, UpdateSelected, UpdateNodeBound, and SetSelectiveUpdateFlags.
