Game initialization overview

The game usually has its own service that deals with game related operations. The game service will initialize as part of the Service Manager initialization. The following is a list of services that are commonly required to initialize before the game service:

  • Config Manager
  • Message Service
  • Entity Manager
  • Scheduler
  • Asset Locator Service
  • Flat Model Manager
  • Entity Loader Service
  • Light Service
  • Camera Service
  • Render Service
  • Animation Service
  • SceneGraph Service
  • Input Service
  • Pick Service

After all of the services are initialized and it is the game service's turn, the OnInit() life cycle event method is called. This usually consists of waiting for the needed services to complete initialization, subscribing to the needed messages, the loading request for your block file, and other calls that need to happen before the game starts.

For details on move from application initialization into block and entity initialization see Block Loading Sequence.
