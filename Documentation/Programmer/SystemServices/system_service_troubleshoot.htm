Troubleshooting system services

Emergent recommends the following troubleshooting methods:

  • Read through and/or review the steps outlined in Creating a system service. Is it possible that you missed or misinterpreted a step?
  • If the SystemService is setup properly but not functioning, a good place to start is in the OnInit function to verify that the prerequisite Services are loaded. If not, check log log messages concerning failures in dependend services. If a required service is missing, the system will assert in debug and log an error.
  • If the system service is being initialized properly and OnTick is being called, but things still are not functioning as expected, it is likely that there are problems with the message subscription. Be sure any subscriptions are subscribing on the correct category and EE_HANDLER macros are correctly used. See Working with messages for more detailed information about the Message System.
  • Most system services found in Gamebryo code will be using a config.ini file to load some specific settings. Check to make sure that the data in the config.ini file is correct for your system service. Check to make sure the config file is being loaded inside your system service code. The efd::ConfigManager class is for this purpose.
