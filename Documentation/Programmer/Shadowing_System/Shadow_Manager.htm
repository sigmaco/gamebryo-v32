Shadow Manager

The shadow manager controls the entire Gamebryo shadowing system. Most interactions with the shadowing system will occur through the shadow manager. This manager is responsible for organizing all the NiShadowGenerator objects, NiShadowMap objects, NiShadowTechnique objects, NiShadowClickGenerator objects, and shadow write materials into a single API. An application using the shadowing system must initialize the shadow manager at startup and shutdown the shadow manager when the application terminates.

Applications request the shadow manager to provide the set of render clicks needed to render the shadow maps for the active shadow generators. When this request occurs, the shadow manager passes all the shadow generators to the active shadow click generator. The shadow click generator executes its algorithm to determine which shadow maps to create and to which shadow generators it will assign those maps. The shadow click generator then produces the shadow render clicks to render the shadow casters to the shadow maps. After generation, the application must ensure that these shadow render clicks are executed with the frame rendering system.

Initialization and shutdown of the shadow manager is performed in NiApplication and NiSample. NiApplication and NiSample both create a render step to encapsulate all the shadow render click objects and execute this render step before the main render step is executed. This allows all NiApplication or NiSample based applications to use the shadowing system without having to perform any extra work.
