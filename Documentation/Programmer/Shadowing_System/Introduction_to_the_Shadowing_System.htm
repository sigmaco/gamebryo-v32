Introduction to the Shadowing System

Prior to the introduction of Gamebryo's shadowing system, applications that wanted to apply shadow maps to geometry would use custom shaders that had to be manually applied to each geometry object, or they would have to extend the NiFragmentMaterial class and create a material with shadow mapping support. The introduction of the shadowing system adds support for shadow maps into NiStandardMaterial and adds additional annotation and semantic hooks that allow custom shaders to retrieve shadow maps from the shadowing system.

The Gamebryo shadowing system provides an easy and automated way to generate shadow maps for dynamic lights and apply those shadow maps to geometry. The system is extensible, allowing users to adjust it to best fit their needs. It also provides a default level of functionality that allows users to setup shadows through the Gamebryo tool chain out of the box. It contains built in methods for determining how to produce, assign, and render to shadow maps. The system additionally provides several different shadow techniques that provide different rendering results and allow users to balance between image quality and performance.

The shadowing system is built on top of the frame rendering system so it is required that frame rendering system be enabled in an application for the shadowing system to function. Enabling the frame rendering system allows the shadowing system to provide the application with all the NiRenderClick objects needed to update the active shadow maps for the current frame.
