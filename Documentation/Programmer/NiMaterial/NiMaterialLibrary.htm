NiMaterialLibrary

Material libraries are the way in which new materials are injected into the Gamebryo material system and art pipeline. A material library contains a set of material descriptors and has the ability to generate materials as needed.

DLL-Based Application Usage

Making use of a specific material library depends on the type of application the material library is being used in. DLL-based applications, like the Max and Maya plug-ins, AssetViewer, and AnimationTool search through a directory of DLL's and add any material library DLL's that are found using the NiMaterialLibrary::LoadMaterialLibraryDLL method. The extension of material library DLL's depends on whether or not the application is debug (*.dlm) or release (*.nlm). Material library DLL's must implement the functions outlined in NiMain\NiMaterialLibraryInterface.h.

LIB-Based Application Usage

For statically linked libraries, include the appropriate lib file for the build configuration and make sure to include the header file that specifies the SDM object for that library somewhere in the application source code.
