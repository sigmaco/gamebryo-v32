Gamebryo Legacy Pipeline

Gamebryo ships with a legacy pipeline for two reasons: first, some graphics hardware may not be capable of using the standard material as implemented by NiStandardMaterial; and second, customers who have developed applications and art assets using older versions of the engine may want to continue using it.

Note that not all renderers may support the legacy pipeline. Refer to the renderer documentation for more information.

The following diagram details how each texture is applied to the results of the previous multitexturing operations. The pipeline starts at the top and flows to the bottom. Pairs of texture effects drawn side-by-side must be applied simultaneously, or else the texture effect to the right must be ignored. These texture effects to the right are the optional effects that are only supported on some multitexture hardware.

Texture Pipeline

Almost no application would use all of these stages. Stages that are not used are simply omitted. The first stage, whatever it might be, is drawn normally without a combine operation, unless alpha blending is enabled. Please see the section below on the limitations of alpha blending and multitexturing.

Each stage is basically equivalent to all others. The multitexture stage includes information determining the image to be applied, the operation to be used to apply it to the incoming color (if any), the filtering mode, and the texture coordinate selection, generation and transform modes.
