Console-specific issues for logging, asserts, and debugging

When an assertion fails while the game is running in a debugger, the debugger will break into the code at the point where the assertion failed. If you are running the game on Windows, an assertion failure may cause the JIT debugger to break even if the game is not running in the debugger.

The log files are created using the efd::File interface and placed in the directory where the game executable is located, unless a different path is specified in the configuration file or in the code.

When the DebugOut type log destination is used, the log messages are output using the following functions:

Platform Function
Windows, XBox360 OutputDebugStringA
PS3 printf
