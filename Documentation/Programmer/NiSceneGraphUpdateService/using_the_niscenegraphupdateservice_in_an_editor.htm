Using the NiSceneGraphUpdateService Library in an Editor

The NiSceneGraphUpdateService library can be embedded in a Windows application that acts as an editor (or somehow manipulates scene graph objects). The NiEditorSceneGraphUpdate singleton allows the editor to insert scene graph objects that are automatically replicated to one or more viewers. It is often necessary for the editor to subclass the NiEditorSceneGraphUpdate class in order to access protected methods used to manage the replicated objects.

When an editor (like the Gamebryo Maya plug-in) is using NiEditorSceneGraphUpdate, it basically creates a scene graph and generates unique IDs for every object in the scene graph. Then as changes are made to Maya objects, a individual nodes are exported to Gamebryo scene graph objects which are reinserted into the NiEditorSceneGraphUpdate singleton. This results in the object being updated in all the viewers.

The editor is responsible for:

  1. Creating (or loading) the scene graph
  2. Assigning a unique ID to each object in the scene graph
  3. Inserting the objects and IDs into the NiSceneGraphUpdate singleton for replication to one (or more) viewers
