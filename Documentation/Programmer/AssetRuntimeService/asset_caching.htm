Caching Assets

The Asset Factory Manager and associated asset factories do not currently support caching of assets. Services that load assets must implement their own caching system, as is done for most Gamebryo system services.

We recommend that caches be indexed with the physical location of the assets, as it avoids ambiguity. This means that the service must perform its own asset location requests in order to resolve a urn to a physical location with which to index the cache. The physical location may then be used in the asset request message if the asset is not present in the cache.
