Asset Infrastructure Reference

The following is a list of system services that form the Asset Runtime Service. In order to gain a detailed understanding of each of the components listed, you should review the functionality that is provided by each of these services.

Asset Loading System

A single service is responsible for loading assets:

  • efd::AssetFactoryManager

Asset Locator System

The system comprises two services that allow applications to locate their assets at run-time.

  • efd::AssetConfigService
  • efd::AssetLocatorService

Asset Controller

This is a Framework application that serves as a communication hub, remote asset lookup server, asset metadata manager, and file system monitoring and notification system. It comprises several services.

  • efd::AssetServer
  • efd::AssetConfigProxy
  • efd::AssetMonitorService
  • efd::ChannelManagerService
  • Tagging Extensions based on efd::IAssetTagger

Asset Rapid Iteration

Rapid Iteration capability in an application is provided by egf::RapidIterationSystemService, which also wraps some dependency services.

  • egf::RapidIterationSystemService
  • efd::ReloadManager
  • efd::NetService
  • Change notification handlers based on efd::AssetChangeHandler
