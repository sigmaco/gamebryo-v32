Introduction to Static Multitexturing

Static multitextures represent surface characteristics of an object. The texture coordinates for a static multitexture are usually defined by a set of texture coordinates that are a part of the NiMesh object itself. In that sense, the texture is "static"—it is fixed to the surface of the object. However, materials may choose to generate them in vertex and pixel shaders if desired. Like base textures, static multitextures are applied using Map objects in a NiTexturingProperty. Each type of static multitexture has a Map "slot" in the NiTexturingProperty, just as the Base Map has its own slot in the NiTexturingProperty.

The Map objects in an NiTexturingProperty define how each static multitexture is to be mapped. Note that an NiTexturingProperty is like any other kind of property—another property of its kind further down the tree will override the property that is higher in the tree. As a result, it is not possible to define various static multitexture Map objects for a single object in multiple different NiTexturingProperty objects. Each new property will completely override all of the static multitextures in a property that is higher in the tree, even if the lower property defines a type of multitexture as disabled (a NULL Map). In general usage, applications tend to have only one NiTexturingProperty property on a simple path from the root of the scene graph to a leaf.
