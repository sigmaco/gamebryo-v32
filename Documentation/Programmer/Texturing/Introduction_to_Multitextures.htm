Introduction to Multitextures

Gamebryo supports the ability to render objects whose polygons have multiple textures applied simultaneously. This feature can allow for some incredible effects. The interfaces of Gamebryo support a basically unlimited number of textures per object. However, note that Gamebryo renderers may have definite limits on the number and kind of textures that may be applied to an object. For details, see the renderer manual for the particular renderer you intend to use.

Gamebryo supports two basic forms of multitexturing via its default pipeline: static and projected. An easy way to understand the difference between the two is by a simple physical analogy. Static multitextures represent surface properties of the objects to which they are attached, while projected multitextures represent spatial phenomena such as the reflected environment or a shaft of light coming from a window in the scene. These forms differ in the way that the effects are specified, as well as the general types of effects that are supported. The two forms of multitexture may be used together or independently. The following sections introduce the two forms of multitexture and describe the differences.

At a high level, multitexturing is the use of more than one texture simultaneously on the same set of polygons. By combining the multiple textures in different ways, subtle or striking effects can be created. These effects will be described below and include (but are not limited to) pseudo-lighting effects, slide and movie projector effects, specular highlights, reflective and glossy surfaces, shadows, and overlaid details.

The basic technical distinction between the two classes is equally simple. Static multitextures are attached as a Map to an NiTexturingProperty, and are mapped using texture coordinates defined in the geometric objects to which the NiTexturingProperty is attached.

On the other hand, projected multitextures are applied to a set of objects via the same scoping mechanism used by dynamic vertex lights and are mapped to the geometry via dynamically-generated texture coordinates that are computed inside the low-level renderer.

Gamebryo supports several different types of multitextured effects within each of the two categories. They are summarized in the tables below, and each will be detailed later in this document:

Projected Texture Effect Description
Light map(s) Complex lighting effects such as spot lights and slide projectors
Shadow map(s) Darkening effects, such as moving shadows cast on a wall
Environment map Mirror-like reflections of the local environment seen from shiny objects
Fog map Volumetric haze effects
Note: NiStandardMaterial does not implement fog maps. As such, they are not recommended for use.
Static Texture Effect Description
Base texture The basic appearance of the surface of a diffusely reflective object
Bump map A fine-scale representation of the bumpiness and shininess of the object — used to perturb and attenuate an environment map.
Dark map A low-resolution representation of the diffuse lighting on an object
Detail map A high-frequency texture that alleviates the blurriness of objects seen close-up
Gloss map A fine-scale representation of the shininess of the object — used to attenuate an environment map (not supported on all platforms)
Glow map A representation of the emissive quality of the object surface — used to create the impression of glowing lights on the surface of the object
Decal map(s) Decals laid upon the surface of an object, such as skid marks or bullet marks
