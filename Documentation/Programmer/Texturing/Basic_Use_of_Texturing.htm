Basic Use of Texturing

The following sections describe the basics of texturing—how textures are applied in code and the settings that affect how they are applied. The sections below are designed to describe single texturing, such as using a base map with NiStandardMaterial, but multiple textures on a single object will be described later as an extension.

In order to use texturing in Gamebryo with NiStandardMaterial, an application needs to do only two things above and beyond what is needed for untextured rendering. First, the application must ensure that the object, an instance of NiMesh, to be textured has one or more sets of texture coordinates (or UV coordinates) in one or more of its NiDataStream objects. Second, they must attach a rendering property that defines the texturing image that is to be applied and how it is to be applied.

In order to specify the texture image and texture application settings for an object, the application must attach an NiTexturingProperty at or above the object in the hierarchy. The NiTexturingProperty contains all the information required to apply a texture (or multiple textures) to an object.
