Projected Multitexture Usage Basics

Projected texturing effects are implemented via NiTextureEffect, which is a subclass of NiDynamicEffect. NiTextureEffect is a sibling class of NiLight, and shares some information in common with lights—scoping and transforms are two examples. Every NiTextureEffect object contains most of the settings that we have come to expect in textured effects: filtering, clamping, and an image pointer. These basic settings are available with each projected texturing effect, and define the effect in the same way as they did in the static multitexturing case. However, rather than an index that references a static set of texture coordinates on an object, the texture coordinates for a projected texturing effect are set via a group of settings that define how the texture coordinates are generated on the fly.
