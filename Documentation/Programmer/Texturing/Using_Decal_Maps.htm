Using Decal Maps

  • Image: An RGBA image with the color representing the visible decal and the alpha channel representing the cutout around the visible decal. Care should be taken to add a large "halo" of the decal edge colors outside of the visible areas, as mipmapping can cause color shifting if the background color in the translucent regions is very different from the colors in the visible areas. This is similar to the "dark edges" phenomenon often seen when colorkey transparency is used with bilerp-ed texture filtering.
  • Filter: Generally, a mipmapped filter is used.
  • Clamping: Varies, but CLAMP_S_CLAMP_T is most common, since decals are often much smaller than the underlying surface.
  • Texture Coordinates: Generally set to place the decal in a small region somewhere on the object surface. As a result, texture coordinates are often negative or greater than 1.0 at vertices.
