Using Dark Maps

  • Image: The image for a Dark Map should be the lighting incident upon the surface. White is fully lit, and black is completely unlit. Some applications use special Dark Maps that can be shared by multiple objects by packing multiple Dark Maps into the same texture. If mipmapping is not used, this is very safe. If mipmapping is used, the adjacent (unrelated) Dark Maps that are tiled into the single texture image can smear together via the mipmap filtering.
  • Filter: FILTER_BILERP is a safe bet—mipmapping is generally unnecessary, as light maps tend to be quite small and low frequency. Also, by using a non-mipmapped mode, multiple lightmaps can be paged into a single texture without fear of "smearing".
  • Clamping: Generally, Dark Maps are not repeated over a surface, so CLAMP_S_CLAMP_T is often used to ensure that texture coordinates of 1.0 and 0.0 do not inadvertently wrap over to the other side of the texture.
  • Texture Coordinates: This depends on the effect. In some cases, applications will use one-dimensional subsets of a texture as a gradient light map. This allows one texture to be used as a Dark Map for many surfaces. In other cases, a light map is mapped exactly once across an entire surface.
