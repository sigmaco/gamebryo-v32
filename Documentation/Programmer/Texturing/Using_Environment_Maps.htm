Using Environment Maps

Environment map example

Effect

An Environment Map, as described above, creates the appearance of a shiny object, in whose surface the "reflection" of the surrounding environment may be seen.

Filter

Any filtering mode may be used, but mipmapped modes are recommended, as areas of high curvature on the surfaces of mapped objects can lead to large parts of the texture mapped onto small areas of geometry.

Clamping

All texture coordinates in SPHERE_MAP mode are in the range 0.0-1.0 by definition, so it is best to use CLAMP_S_CLAMP_T to avoid any wrapping at the exact edge of the texture.

Effect mode

ENVIRONMENT_MAP

Texture coordinate mode

SPHERE_MAP—the environment map was the reason that this mode was created. This will map the normal vectors of the object into the environment texture.

Image

The image for a sphere map is rather complex, due to the type of mapping that SPHERE_MAP applies to the vertex normal vectors. Each pixel in the image represents a different direction in world space. Not all pixels in the image are ever used, however. See the section below on world-space sphere mapping.

Transforms

By default, the sphere-mapping coordinate-generation mode uses the local and world transforms of the environment map object to affect the texture lookup. However, renderers may add additional functions that allow the application to specify that environment maps are to be in camera-space, not world-space. In these cases, the transforms on the environment object are ignored, See the specific renderer documentation for details.
