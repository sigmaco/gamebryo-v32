Static Multitexture Usage Basics

Static multitexturing is enabled and controlled by attaching an NiTexturingProperty to a scene graph node and attaching a Map object to the Base Map slot in that property. Adding static multitextures is as simple as creating more Map objects, one for each multitexture to be applied to the object, and attaching the Map to the desired Multitexture Map slot in the same NiTexturingProperty. In this way, the interface is consistent across the Base Map and all other Maps. The only difference between single texturing and multitexturing is the need to attach multiple different Map objects to the different slots in the NiTexturingProperty.

Note that Map objects are not reference-counted, and should not be shared across multiple NiTexturingPropertys or across multiple map settings in the same NiTexturingProperty. Also, applications should allocate their Map objects using NiNew, and then pass them into the NiTexturingProperty. Once a Map is passed to the NiTexturingProperty, the property "owns" it and will delete it as needed.

Setting the Map value for a particular type of effect (Dark Map, Detail Map, and so on) to NULL will turn off that type of effect. Also, setting the texture pointer to NULL in any attached Map object will disable the effect.

The one setting that tends to be different for multitexture Map objects and Base Map Map objects is the texture coordinate index, set using NiTexturingProperty::Map::SetTextureIndex. This is used to index a different set of texture coordinates for the multitexture. Coordinate Specification describes how these sets of texture coordinates may be added to objects. Texture coordinates can be changed programmatically; see Adding or Updating Texture Coordinates On-the-fly.
