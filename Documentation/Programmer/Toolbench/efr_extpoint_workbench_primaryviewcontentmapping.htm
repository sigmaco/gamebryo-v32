PrimaryViewContentMapping (Extension-Point)

Emergent.Toolbench.Workbench.PrimaryViewContentMapping

Because content can be mapped to any number of views and because views can be docked anywhere in Toolbench. There needs to exist a method by which content can be bound to a view so that that content will remain opened until its primary view is closed. This extension point allows you to create that mapping.

<extension point="Emergent.Toolbench.ContentService.PrimaryViewContentMapping">
  <mapping content-type-id="Emergent.ImagePlugin.ImageDocument">
    <primary-view id="Emergent.ImagePlugin.ImageView"/>
    ...
  </mapping>
</extension>

mapping

Attribute Type Description
content-type-id String The unique id of the content type that the primary views will be associated with.

primary-view

Attribute Type Description
id String The unique id of the view that is a primary view of this content type.
