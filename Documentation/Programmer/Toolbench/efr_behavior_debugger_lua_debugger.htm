Lua Behavior Debugger

Beyond the limited scheduler/debugger integration points discussed above, the implementation of Behavior Debugger modules quickly becomes language specific. Emergent provides a Lua Behavior Debugger reference implementation, which can be examined as a starting point for support of additional languages.

The Lua Behavior Debugger makes use of the Lua Debug API to install a C++ debug hook function that is invoked on Call, Return, and Line events. These notifications allow the debugger to monitor the flow of script execution and to evaluate user specified step requests and conditional breakpoints.

When a break is hit, the debugger runs evaluation routines to examine the state of the call stack, local variables, and user watch expressions. All of these records are then messaged to the connected Toolbench debugging session. The MessageService also continues to receive incoming messages from Toolbench, which provides commands for executing immediate statements and stepping or resuming behavior execution.
