Introduction to the Behavior Debugger

Remote behavior debugging consists of two components.

  • One or more language-specific Behavior Debugger modules are linked into the Emergent Game Framework to interact with the behavior schedulers.
  • A language-independent Script Debugger based in Toolbench provides a remote UI for interactively debugging game behaviors.

Note

"Behavior Debugger" refers to engine code tightly integrated with the behavior system and scripting run-time, whereas "Script Debugger" refers to a tool-side front end that could be repurposed to debug any type of scripting.
