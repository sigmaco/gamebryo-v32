SystemService (Extension-Point)

Emergent.Toolbench.Framework.SystemService

The SystemService extension point provides a facility for specifying services that need to be started in the future. This is done by registering initializers which are responsible for starting a service. These initializers can be queried by a category for organization. For instance, when World Builder starts it will check for all SystemService extensions with a category of "Emergent.Toolbench.WorldBuilder" and trigger those initializers (which will, in response, start services).

<extension point="Emergent.Toolbench.Framework.SystemService">
  <initializer class="Emergent.Toolbench.Framework.SomeSystemService"
               category="SomeCategory"/>
</extension>

initializer

Identifier Type Description
class String The class is the full class name of the initializer class responsible for initializing the service. The initializer class must implement the Emergent.Toolbench.Framework.IFrameworkServiceInitializer interface.
category String The category is a string identifying a category to which the service belongs. This can be useful for organizing services by category so that all the services of a specific category can be started at a specific time.
