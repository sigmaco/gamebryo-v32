ContentAspectType (Extension-Point)

Emergent.Toolbench.ContentService.ContentAspectType

The content aspect type extension allows users to "view" a piece of content in a different way. Another way to think about aspects is that they provide a way of extending the functionality of existing content types. As an example consider the undo/redo system. Instead of keeping the data for undo/redo separate from the content, we created a ActionHistoryAspect that works with content and is attached to every type of content and tracks the history of actions executed on that content without having to implement history tracking on the content itself. If your content aspect type does not list any supported-content types it is assumed that your aspect works with every content type as would be the case with ActionHistoryAspect.

<extension point="Emergent.Toolbench.ContentService.ContentAspectType">
  <aspect id="Emergent.Toolbench.Actions.ActionHistoryAspect"
          class="Emergent.Toolbench.Actions.ActionHistoryAspect">
    <supported-content id="..."/>
    ...
  </aspect>
</extension>

aspect

Attribute Type Description
id String The name of the category that content types should be listed under when presented to the user.
class String The full name of the class that implements the functionality of the aspect. The class must inherit from the Emergent.Toolbench.ContentService.AbstractContentAspect class.

supported-content

Attribute Type Description
id String The unique id of the content type that this aspect supports.
