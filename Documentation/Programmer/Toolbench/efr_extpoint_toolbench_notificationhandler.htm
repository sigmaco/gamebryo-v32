NotificationHandler (Extension-Point)

Emergent.Toolbench.NotificationHandler

The NotificationHandler extension point allows plug-in authors to expose important events coming from their plug-in to other potentially unloaded plug-ins. Any plug-in wanting to listen to these notifications needs only to add a NotificationHandler extension point and a class that implements the Emergent.Toolbench.INotificationHandler interface. If your plug-in is unloaded and desires to be loaded, it will be when the notification occurs.

Currently Toolbench only exposes two notifications, Toolbench.PluginLoading and Toolbench.PluginLoaded.

<extension point="Emergent.Toolbench.NotificationHandler">
  <handler id="Toolbench.PluginLoaded"
           param="Emergent.Toolbench.UI.Workbench"
           class="Emergent.Toolbench.UI.WelcomeView.NotifyWorkbenchLoad"
           load-plugin="true"/>
  ...
</extension>

handler

Attribute Type Description
id String The unique identifier of the notification you are handling notification.
param String [optional] The parameter to look for when being notified. So in the above example, we wish to be notified when the "Toolbench.PluginLoaded" notification is sent, but only when the parameter being sent is "Emergent.Toolbench.UI.Workbench". If you want your notification to always be called, you should not specify a parameter.
class String The fullname of the class that will handle the notification. The class must implement the Emergent.Toolbench.INotificationHandler interface.
load-plugin Boolean [optional] [default: false] The load-plugin attribute allows you to control whether or not your plug-in should be loaded in order to process this notification.
