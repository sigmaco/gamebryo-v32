Toolbench Plug-in Best Practice: Studio Customization

Don't miss opportunities to allow users to customize or further extend new functionality. This means considering new extension points to open up features to user plug-ins. New extension points should include an XML schema for validation. Consider providing C# interfaces or extensible base classes where studio customization seems desirable. Also note that XML documentation can be enabled in C# project settings, allowing in-line XML comments to appear as API help text in auto-complete when users code against the assembly. These XML help files are generated for all Emergent-provided Toolbench plug-ins.
