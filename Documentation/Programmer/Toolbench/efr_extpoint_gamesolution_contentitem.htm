ContentItem (Extension-Point)

Emergent.Toolbench.GameSolution.ContentItem

The ContentItem extension point allows users to create new embedded content that can be added to the solution that does not represent files but instead things like servers, consoles, or any other object someone would want to group in the game solution as an asset.

<extension point="Emergent.Toolbench.GameSolution.ContentItem">
  <item id="Emergent.Toolbench.GameSolution.UI.DiskFolderItem"
      name="Folder"
      content-type="Emergent.Toolbench.GameSolution.UI.DiskFolderItem"
      description="A folder that also represents a directory on disk."
      classifier="true"
      icon-collapsed="Icons\folder.png"
      icon-expanded="Icons\folder_open.png"/>
    <child content-type="any" allowed="true"/>
  </item>
</extension>

item

Identifier Type Description
id String The id is a uniquely identifying string for the content item.
name String The name is a user friendly name for this type of content.
content-type String A uniquely identifying string representing the content's type.
class String [optional] The full class name for the class that stores the content item. The class must extend the GameSolutionItemExtension class.
description String [optional] Describes the content.
icon-collapsed String [optional] Icon to show when the content item is collapsed.
icon-expanded String [optional] Icon to show when the content item is expanded.
uses-file Boolean [optional] Determines whether or not the content uses a file.
always-open Boolean [optional] Determines whether or not the content is always open.
classifier Boolean [optional] Determines whether this item is a classifier (like a folder).
order Integer [optional] Sets the order of appearance of the item in the Solution Explorer.
persistence Enum [optional] Indicates if the item manages its own persistence for its sub-hierarchy.
  • Normal
  • External
  • None

child

Identifier Type Description
content-type String Type of content this content can have as a child.
allowed Boolean Determines if the child relationship is allowed.

parent

Identifier Type Description
content-type String Type of content that can parent this content.
allowed Boolean Determines if the parent relationship is allowed.
