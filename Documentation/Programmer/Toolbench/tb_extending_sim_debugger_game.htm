Adding Simulation Debugger Support

Before using the Simulation Debugger with a game, the game code must enable support for the tool. This setup requirement is minimal and the Game Framework samples already include it.

Be sure that the game creates the RapidIterationService and then add the following code to create the Sim Debugger singleton in the services setup. 

#ifndef EE_CONFIG_SHIPPING
    // Create the Sim Debugger. It is managed by the RapidIterationService.
    RapidIterationSystemServicePtr spRapidIteration =
        m_spServiceManager->GetSystemServiceAs<RapidIterationSystemService>();
    if (spRapidIteration)
        spRapidIteration->CreateSimDebugger();
#endif

The CreateSimDebugger method optionally takes a pointer to an instance of a SimDebugger-derived class. This parameter can be specified if a custom SimDebugger class is desired, although many Simulation Debugger extensions can be accomplished without this.
