ClosingHandler (Extension-Point)

Emergent.Toolbench.Workbench.ClosingHandler

The closing handler extension point allows you to hook via an extension the closing event generated by Workbench. This event is triggered when Workbench is in the process of being closed, but has not yet verified that it is in fact going to close. By implementing this extension point you can intercept that event and perform some operation that can stop the closing of Workbench. An example of this is the exit handler used by the content dialogs. That plug-in uses it to notify the user that they have unsaved documents and allows the user to cancel the closing process.

Note: Your plug-in must have been loaded for your closing handler to be called.

<extension point="Emergent.Toolbench.Workbench.ClosingHandler">
  <handler class="Emergent.Toolbench.UI.ContentDialogs.ExitHandlerAction"/>
</extension>

handler

Attribute Type Description
class String The full name of the class to use when calling the closing handler. The class must implement the Emergent.Toolbench.UI.Workbench.IWorkbenchClosingHandler interface.
