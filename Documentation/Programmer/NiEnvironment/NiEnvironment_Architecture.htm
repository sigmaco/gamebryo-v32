NiEnvironment Architecture

The NiEnvironment system relies on a number of customizable objects to create a dynamic and realistic skybox rendered in real time.

  • NiEnvironment
    • This class is the main management interface of the NiEnvironment system. It maintains all the objects and provides the main interface for interaction to the user. Most of the other classes are accessible through the NiEnvironment interface.
  • NiAtmosphere
    • This class is used to define the different attributes that will be used in sky color and light scattering calculations. Parameters can be modified to create a highly customizable sky (Color wave lengths, scattering modifiers, sun intensity, etc.)
  • NiSky
    • This object represents the actual geometry for the sky. It is an abstract class that can be implemented to create custom skyboxes. An example NiSkyDome implementation is provided. It can also manage blending between different sky textures.
  • NiTimeOfDay
    • An optional component of NiEnvironment, this class is designed to handle animation and transition states between different sky and atmosphere settings. It allows the user to set key frames at different times of the day varying each parameter between pairs of keyframes.

For convenience, NiSkyRenderView class is also provided to facilitate rendering of the environment within the NiFrame rendering system.

Although most use cases will be handled by the EnvironmentService and through Toolbench, this part of the documentation will detail how the environment system works in order to facilitate possible customizing and improving of the provided tools.
