Tools Programming: Introduction

Gamebryo LightSpeed ships with many powerful tools designed to be extensible by a studio in order to meet the custom demands of a particular project.

Toolbench Customization

Toolbench is the application plug-in framework used as the foundation for many of the tools that ship with Gamebryo LightSpeed. The following are the major tools hosted with-in the Toolbench architecture.

  • Entity Modeling Tool
  • World Builder
  • Script Debugger
  • Simulation Debugger

Toolbench is designed from the ground up to be completely driven by plug-ins. Plug-ins are extensible by other plug-ins through the use of an extension and extension point mechanism. This mechanism provides the ability for plug-ins to define their own contract with other plug-ins in the system without being forced to rely on a common plug-in API. In fact, the tools listed above are hosted by the Toolbench framework and consist of several different plug-ins.

In order to get started writing plug-ins for Toolbench, see the Toolbench Introduction section of the documentation.

Tool Plug-in Framework

In addition to the powerful exporter support for Max, Maya, and XSI, Gamebryo LightSpeed also ships with a plug-in framework specifically targeted at the art pipeline. When a scene is exported from a DCC tool, the scene itself is passed through a series of tool plug-ins. These tool plug-ins provide capabilities such as processing geometry data, converting from one file format to another, saving platform specific data, and so forth. See this list all of the tool plug-ins that ship with Gamebryo LightSpeed. To get started writing custom plug-ins for the tool art pipeline, see this introduction to the tool framework.
