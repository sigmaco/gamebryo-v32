Cloning PhysX Content

The Gamebryo-PhysX Core Runtime classes typically support cloning. The exceptions are objects that are shared, such as collision meshes. Under most circumstances, applications will clone NiPhysXProp objects, which also clones the snapshot for the prop and all the sources and destinations that the prop has stored. For additional information on cloning, see the Object Systems Cloning documentation.

A Gamebryo scene graph does not know about PhysX content, and vice versa. Therefore, to correctly clone both a scene graph and its PhysX content, applications must explicitly create and specify a NiCloningProcess which should be used for both the scene graph cloning operation and the PhysX prop cloning. It is also necessary to clone the scene graph before the PhysX content, like this: 

NiCloningProcess kCloning;
NiNodePtr spNodeOut = (NiNode*)spRootIn->Clone(kCloning);
NiPhysXPropPtr spPropOut = (NiPhysXProp*)spPropIn->Clone(kCloning);

Note that cloning does not instantiate the cloned PhysX content or add it to a scene (even if the prop that is cloned is in a scene).

You can clone an entire NiPhysXScene, and hence all the props in it. The resulting clone is not associated with any NxScene object.

Extending Cloning

Cloning is part of the functionality provided by the Gamebryo-PhysX descriptor classes. See the documentation on extending descriptors for information on how to extend the cloning functionality to capture more PhysX state.
