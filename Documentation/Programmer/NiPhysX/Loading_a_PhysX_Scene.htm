Loading a PhysX Scene

NiPhysX content is streamed using Gamebryo's standard streaming mechanisms. After loading a NiStream object, use the RTTI interface to find objects of class NiPhysXProp among the top-level objects.

Props loaded in this way have all the necessary data to create a PhysX object stored in their snapshot, but the PhysX objects are not yet created. To instantiate the objects, add the prop to a scene.

Modifying Streamed Snapshots

Due to limitations in the art tool support for PhysX, it is frequently necessary to tweak the scene snapshot. This must be done after loading and before adding the prop to a scene. Use the NiPhysXProp::GetSnapshot function to get the snapshot for a prop and then traverse its contents to modify state as desired. The demos contain examples of this.

Loading SDK or Scene Parameters

PhysX SDK global parameters are streamed in top-level NiPhysXSDKDesc objects, and scene parameters are stored in NiPhysXScene objects. To reset the SDK parameters, look for one of these in the loaded top-level objects and push it to the SDK. Applications can hold several of these objects to enable caching of SDK parameter sets.
