Tuning Scheduler performance

You can measure the performance of the Scheduler at run-time by using a feature that periodically dumps performance metrics to your application's log file. The following illustrates a typical log statement of this type generated by the HelloWorld application.

2009/03/09 17:01:42.377|Lvl2|Scheduler|||Entity/Behavior statistics (average for last 17.20 seconds):
	36.00 entities
	0.00 executing entities
	0.00 behaviors executed per entity per second|
2009/03/09 17:01:42.377|Lvl2|Scheduler|||Entity/Behavior statistics (average since scheduler startup: 113.27 seconds):
	35.00 entities
	0.00 executing entities
	0.00 behaviors executed per entity per second|

Improvements to the performance of the Scheduler may be realized by adjusting the maximum number of behavior threads that the Scheduler allows at one time. This parameter defaults to 255 executing behavior threads, but may be adjusted up or down by changing the configuration value for Scheduler.MaxThread.
