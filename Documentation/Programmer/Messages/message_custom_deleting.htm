Deleting custom message types

If you want to remove a custom message type that you added, the process is basically the reverse of adding one.

To delete a custom message type

  1. The existing message id can be removed if it is no longer used.
  2. The defined message class can be removed. In the case of MessageWrapper only, references to MessageWrapper that use the custom class id need to be removed. After removing both the message id and message class, the compiler should help you find any references that also need to be removed or updated.
  3. Any EE_HANDLER* macros that refer to the custom message type must either be removed or modified to use a different message type and message id.
  4. Subscriptions only need to change if the class that handles the message does not receive any other messages on the same Category.
  5. Make sure all attempts to send the custom message are removed or changed to a different message type.
