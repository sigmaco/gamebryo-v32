Registering your ShaderLibrary with the ShaderFactory

To have the streaming system properly access the shaders from a library, the application must register the streaming system with the shader factory. This step is accomplished with a single function call to NiShaderFactory::LoadAndRegisterShaderLibrary. Static builds also need to include the top level header for each shader library that will be loaded. It is safe to include this file in dynamic builds.

One final thing applications will likely do at this point is to set the program directory for the NiD3DShaderProgramFactory. This step is accomplished via the NiD3DShaderProgramFactory::SetProgramDirectory function call, and sets the folder where shader program files will be found.

For a complete example of registering a shader library with the system, as well as the implementation of an example class creation callback function, please refer to the ShaderSample sample.
