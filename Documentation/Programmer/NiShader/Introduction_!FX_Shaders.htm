Introduction to FX Shaders

Gamebryo's NiShader system is a powerful way for applications to make use of advanced rendering techniques, and the NSF/NSB system makes it easy to write advanced shaders and integrate their development into the art pipeline. Gamebryo also allows developers to to use Microsoft's Effect file format to create Gamebryo shaders just as they would use NSF files, through the NiD3DXEffectShaderLib, NiD3D10EffectShaderLib, and ecrD3D11EffectShaderLib libraries. Not only does this allow developers to write shaders in Microsoft's High-Level Shader Language (HLSL) and make use of a number of third-party applications for authoring FX files, but it also opens up a wealth of shader effects, available online in FX files, for immediate use with Gamebryo. Because the library is based on Gamebryo's NiShader architecture, it would be useful to read the documentation for the system before trying to understand this implementation of the system.

When the NiD3DXEffectShaderLib is linked with an application, either statically or dynamically, the application treats any FX files in a similar manner to how it would treat an NSB file, and creates a Gamebryo shader library representing that file, containing all the techniques in that file as shaders. An application can request a shader from the library, which will create an NiD3DXEffectShader that encapsulates everything that is needed to render an object with a given technique. By making use of the annotations and semantics that the D3D Effect format provides, the parameters in the technique can be linked to Gamebryo data, just as is possible with NSF/NSB files. In addition, the library provides the same data to the artist's tools that the NSF/NSB files do, allowing effects contained in FX files to be integrated into the art pipeline to the same degree.

Gamebryo can load either text-based FX shaders, or binary FX shaders that have been compiled with fxc.exe. Compiled FX shaders should be use the extension ".FXO" to distinguish them from uncompiled FX shaders.

The NiD3D10EffectShaderLib and ecrD3D11EffectShaderLib libraries can be used to load D3D10 / D3D11 shaders just as the NiD3DXEffectShaderLib is used to load D3D9 shaders. For the most part, these libraries are used in exactly the same manner, but refer to this page for a discussion of the differences.
