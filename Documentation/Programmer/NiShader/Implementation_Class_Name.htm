Implementation Class Name

Implementations now support class name declarations that allow for applications to extend the system by deriving their own NiBinaryShader-based classes.

Class names are declared via the following entry in an implementation section of an NSF file:

ClassName = NameOfClass

Where "NameOfClass" can be any string or quoted string the developer desires.

Each implementation has its own class name to allow for fallback-techniques to use different derived classes. The ClassName declaration should not be made more than once in an implementation.

When loading a shader implementation that contains a class name, the system will call the register CreateNiBinaryShader callback, passing in the class name. At this point, applications may create their own NiBinaryShader-derived class instance, or they may pass back NULL indicating that the system should create a standard NiBinaryShader instance.

For more information on this topic, please see Extending NSF/NSB Shaders.
