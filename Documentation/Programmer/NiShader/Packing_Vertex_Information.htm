Packing Vertex Information

Your NSF file must specify at least one packing definition for the data that it expects in the data streams held by NiMesh. Note that prior to the introduction of NiMesh, this definition dictated how the data would be packed into vertex buffers at runtime but is now used only for generating an NiMeshMaterialBinding object. Below is the packing definition from our example shader:

PackingDef PosNormUV0BinormalTangent
    {
        PD_Stream   0
            PDP_Position    0   PDT_Float3
            PDP_Normal      1   PDT_Float3
            PDP_TexCoord0   2   PDT_Float2
            PDP_Tangent     3   PDT_Float3
            PDP_Binormal    4   PDT_Float3
            PDP_Color       5   PDT_UByteColor
    }

Note that we must specify the full packing definition even though only the parameter values such as PDP_Position and the component counts such as the value of 3 from PDT_Float3 are relevant since the parser has not been fully updated.
