Stream Output

When a rendering pass needs to make use of stream output, it will declare the buffers to which it will output in the StreamOutput section. The buffers that are used must have been previously declared in a Stream Output Buffer section.

The stream output cis specified like this:

StreamOutput
{
    Stream Out Targets
    {
        <BufferName>
        ...
    }
    StreamOutAppend = <Append>
}

Where:

  • Append> is an Boolean value. If the value is true, then the data streamed out during this rendering pass will be appended to the end of the data in the current buffer, assuming the buffer is large enough. If the value is false, then the data streamed out will overwrite the data currently in the buffer. If the StreamOutAppend section is left out of the declaration, the default value will be false.

Stream Out Buffer Example

The following NSF snippet defines a semantic adapter table for an implementation:

StreamOutput
{
    StreamOutTargets
    {
        OutputBuffer0
        OutputBuffer1
    }
    StreamOutAppend = false
}

This declaration indicates that two buffers, OutputBuffer1 and OutputBuffer2, will be written to during this pass. They will be overwritten rather than appended to.

The ShaderSample demo also demonstrates the use of stream-out buffers using a NSF in its StreamOutTestEffect effect.
