Requirements for working with behaviors

Before creating a new behavior the developer should be familiar with the following tools and concepts:

  • C++ and/or one of the supported scripting languages that are used to implement behaviors:
    • Python
    • Lua
  • The Entity Modeling Tool (EMT) which is used to define new behaviors and associate them with a model.
  • The Gamebryo Game Framework.

Scripts can access the services provided by the Game Framework via the built-in API methods included in the framework itself and in additional game modules. Therefore to write anything beyond very basic behaviors one also needs to be familiar with the APIs of the built-ins provided with the product or added by the developer. The APIs provided by Gamebryo are summarized in the table below.

Name Library Header Files Description
Behavior API egf
  • BehaviorAPI.h
  • bapiEntity.h
 Provides access to the basic functions of the system including the ability to:
  • Log events
  • Message other entities
  • Modify entity properties
Animation Services egmAnimation
  • egmAnimationBindings.h
  • Query and modify the animations associated with an entity.
PhysX egfPhysX
ecrPhysX
  • egmPhysXBindings.h
  • ecrPhysXBindings.h
  • Control of PhysX service.
  • Receive notification about triggered events.
  • Manipulate physically-controlled Entities.
Terrain Services egmGamebryoTerrain
  • egmGamebryoTerrainBindings.h
  • Query terrain information.
Gamebryo Services egfGamebryoServices
  • bapiCoreRuntime.h
  • Exposure of fundamental Gamebryo Core Runtime services including, scene graph manipulation, ray casting, and transforms

Additional services can be exposed via custom built-ins implemented by the developer.
