Introduction to Immediate Mode

Immediate Mode refers to the mode of operation where all input data is retrieved from the device via polling. Each device type contains a set of functions that provide the application with the ability to determine what the state of the input device was at the time of the last update. A small amount of state information is also stored internally to allow for the detection of edge cases, such as a button/key being pressed or released since the last update.

When operating in immediate mode, the application should call NiInputSystem::UpdateAllDevices once per frame to ensure that the current 'snapshot' of input is correct when they process it.

Note

On some platforms, calling UpdateAllDevices too often or quickly can cause buffering errors in the platform-specific APIs used under the hood of NiInput. In general, the input system should be updated once per rendered frame. If your application runs at a very high frame rate, you should consider making your calls to update and process input at a specific time interval rather than a frame interval.

For more information on processing input while in immediate mode, please refer to Processing Input in Immediate Mode.
