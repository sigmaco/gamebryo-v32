NiDI8InputSystem

: public NiInputSystem

<NiDI8InputSystem.h>

Description

NiDI8InputSystem implements the Win32-specific version of the NiInputSystem. Although the class is named for DirectInput, it does use XInput to handle input from the Xbox360 game pads.

Constructors

The constructor for ;NiDI8InputSystem should be viewed as a protected function. It is public to allow the static function NiInputSystem::Create to call it. Applications wishing to instance the Win32 version of the input system should use the static NiInputSystem::Create function, passing in the DI8 version of the CreateParams class.

The CreateParams Sub-Class

NiDI8InputSystem::DI8CreateParams : public NiInputSystem::CreateParams

This sub-class contains additional initialization information required for the Win32 implementation of the NiInputSystem. It includes the following members (and corresponding accessor functions to them):

m_hOwnerInst - the application instance handle.

m_hOwnerWnd - the application window handle.

m_apcMappingNames - an array of 'names' to show for each control on the game pad in the DirectInput configuration screen. They correspond to the NiDI8InputSystem::DefaultActions described below.

Supported Enumerations

The NiDI8InputSystem class contains an enumeration representing the mapping of DirectInput controls to Gamebryo-defined controls. These are detailed below:

NiDI8InputSystem::DefaultActions

The DefaultActions enumeration identifies the Gamebryo-defined buttons used in the DirectInput action mapping initialization. The DI8 implementation uses DirectInput action mapping to ties game pad controls to recognized Gamebryo definitions. An application should never have to deal with these enumerations, but they are listed to provide technical background for the system. It contains the following values:

Supported Member Functions

Only functions that expand on the definition supplied in the NiInputSystem reference are explained here. For more information on all supported functions, see the NiInputSystem reference.

virtual NiInputErr HandleDeviceChanges()

Handles insertion and removals of devices on the Win32 platform. See Handling Lost or Removed Devices for more information on this.

bool SetControlMapName(DefaultActions eAction, const char* pcName)

Sets the display string for the given eAction to the given pcName. If pcName is NULL, the default display string will be used.
