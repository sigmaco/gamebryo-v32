Introduction to Action Mapping Mode

Action Mapping Mode refers to the mode of operation where specific input events generate an action that is then reported back to the application. The input system contains a set of functions that allow for the application to retrieve actions that have been queued up during the last update.

When operating in mapped mode, the application should call NiInputSystem::UpdateActionMap once per frame to ensure that the current state of the action queue is correct when they process it. It is important to process the actions each frame as well (in most cases) to avoid buffer overflow in the queueing of actions.

To enable action mapping mode, an application simply has to set the action map in the system via the call NiInputSystem::SetActionMap. For information on setting up an action map instance, please refer to Setting up an Action Map.

For information on processing input while in immediate mode, please refer to Processing Input in Action Mapping Mode.
