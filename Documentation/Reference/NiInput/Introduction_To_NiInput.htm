Introduction to NiInput

NiInput provides a cross-platform interface to input devices on all supported platforms. The only platform-specific code required is the setup of the parameters pass to the static NiInputSystem::Create call.

Game pads are supported on all platforms. Mouse and keyboard support is also supplied on the PC. NiInput only supports a single mouse and a single keyboard, and will use the first instance of these devices it finds.

NiInput provides two modes of operation. The first is Immediate Mode, which requires users to poll the input devices to determine what keys and/or buttons have been pressed, and what analog axis values are currently. The second, Action Mapping Mode, provides users with the ability to map actions to specific input device controls, such as button presses, analog axis ranges, etc. Action Mapping Mode is a super-set of the functionality of Immediate Mode, and allows for devices to be polled in addition to supporting mapped actions.

For more information on Immediate Mode, please see Introduction to Immediate Mode.

For more information on Action Mapping Mode, please see Introduction to Action Mapping Mode.

Note

When using NiInput, if your application 'holds onto' any input device you MUST USE SMART POINTERS! If this is not done, the input system could delete the device in cases such as where they are lost, and your application will likely crash.
