Gamebryo 2.6.0

  • A bug was fixed that prevented NiInputMouse::ButtonWasReleased(Button eButton, unsigned int& uiModifiers) from correctly reporting the modifiers that were active when the mouse button was pressed.

Gamebryo 2.5.0

  • Changed the NiInputSystem class to no longer take an NiRenderer object as input to its creation function or structures.
  • Fixed an infinite loop that would be triggered by having an application request an 800x600 render window.
  • Analog button "pressed" state now properly observes the analog threshold. The threshold may be set with NiInputGamePad::SetDefaultAnalogThreshold().
  • NiInputPS3Gamepad now reflects the correct analog value of SIXAXIS analog buttons. Previously these buttons were interpreted as binary on or off only.
  • NiInputPS3Gamepad now implements support for the SIXAXIS sensors and rumble motors.

Gamebryo 2.3.0

  • Some uninitialized variables in the NiDI8InputSystem::DI8CreateParams class were initialized to prevent deleting uninitialized pointers during shutdown.

Gamebryo 2.2.0

  • Remapped the default action map in NiInputDI8GamePad to match the controls on the Xbox360 controller.

Gamebryo 2.1.0

  • TheNiInputSystem::MapNiActionsToDevices function now returns valid error codes rather than always returning one that represented an error.
  • NiInputKeyboard can now map identical keys with different modifiers to different actions.

Known Issues and Limitations

  • The Win32 implementation of the NiInput library, NiDI8InputSystem, does not allow the shoulder triggers on the Xbox360 controller to be mapped using the NiActionMap interface. This is a limitation of the DirectInput8 system.
  • The Win32 implementation of the NiInput library, NiDI8InputSystem, does not support the NiInputSystem::ConfigureDevices functionality when the NiD3D10Renderer or ecr::D3D11Rendereris used. This is a limitation of the interaction of D3D10 / D3D11 with DirectInput8.
