NiSpotLight

: public NiPointLight

<NiSpotLight.h>

Description

NiSpotLight is an object that represents a spotlight in a scene. Spotlights have a specific location and direction in space, as well as a distance attenuation function and an angle attenuation function. Location and distance attenuation are inherited from the NiPointLight class. The light direction is handled in the same way as it is in NiDirectionalLight--it shines down the model-space positive X-axis of the light.

Constructors

NiSpotLight()

This default constructor creates a spotlight with a direction of [1.0,0.0,0.0], an angular attenuation exponent of 1.0, and an inner and outer spotlight angle of 0.0 degrees.

Supported Member Functions

const NiPoint3& GetWorldDirection() const

Retrieve the direction of the axis of the spotlight cone from the light location in world space.

float GetSpotAngle() const

void SetSpotAngle(float fSpotAngle)

Retrieve and set the outer spotlight angle (in degrees). The angular attenuation equation is described below in the "Notes" section.

void SetInnerSpotAngle(float fSpotAngle)

float GetInnerSpotAngle() const

Retrieve and set the inner spotlight angle (in degrees). The inner spot angle the region of full intensity for a spotlight.

float GetSpotAngleCos()

float GetInnerSpotAngleCos()

These two functions get a cached value containing the cosine of the spot angle's. The cached values are updated whenever SetSpotAngle or SetInnerSpotAngle is called. Pixel and vertex shaders generally use the cosine of the angles rather than the angles themselves with spot lights.

void SetSpotExponent(float fSpotExponent)

float GetSpotExponent() const

Retrieve and set the angular attenuation exponent. This value may or may not be ignored, depending on the renderer. The angular attenuation equation is described below in the "Notes" section. This value is used to attenuated between the inner and outer spot angles.

Notes

Angular Attenuation Equation

Unless the angular attenuation exponent is something other than 1.0, Gamebryo uses linear attenuation from the inner cone angle to the outer cone angle. If the angular attenuation exponent is not 1.0, its effects are renderer-dependent.
