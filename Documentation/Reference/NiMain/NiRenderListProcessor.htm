NiRenderListProcessor

: public NiRefObject

<NiRenderListProcessor.h>

Description

This class is a render list processor that is invoked from NiViewRenderClick to process the array of NiRenderObject objects that is returned from an NiRenderView object. It has two virtual functions: one for pre-processing the geometry array before NiViewRenderClick renders any geometry and the other for post-processing the array of geometry that NiViewRenderClick has just rendered. It is permissible for PreRenderProcessList to actually render geometry instead of deferring that rendering to NiViewRenderClick. The default implementations of both virtual functions do not do anything useful, so derived classes should override these functions to perform meaningful processing.

Supported Virtual Functions

virtual void PreRenderProcessList(const NiVisibleArray* pkInput, NiVisibleArray& kOutput, void* pvExtraData)

This function is called prior to any geometry being rendered. The input array of geometry is provided in pkInput, which may be null. Any geometry not rendered in this function should be returned in the kOutput array. All geometry in the kOutput array after this function returns will be rendered by NiViewRenderClick. The extra data pointer is specified in NiViewRenderClick when a processor is assigned to the render click. Derived classes should override this function to perform custom processing on the array of geometry. The default implementation of this function immediately renders all geometry in the pkInput array. This is done, rather than deferring all rendering to the kOutput array, to avoid duplicating a potentially large array in a very commonly called function.

virtual void PostRenderProcessList(NiVisibleArray& kPreviousOutput, void* pvExtraData)

This function is called after the kOutput array from PreRenderProcessList has been rendered by NiViewRenderClick. kPreviousOutput should contain the same geometry objects as kOutput, only these objects have now been rendered. Derived classes should override this function to perform custom post-processing operations on the rendered geometry. The default implementation of this function does nothing.
