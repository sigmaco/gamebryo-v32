PhysX Trigger Service (TriggerService) Quick Reference

For more information about this service, see the egfPhysX::TriggerService class reference documentation.

Library/Namespace

egfPhysX

Display Name

TriggerService

Default Priority

1310 (when running synchronously), 1810 (when running asynchronously)

Priority for PhysX services varies depending on whether PhysX is being run synchronously or asynchronously; see PhysX system service setup for sample orderings.

Required Services

  • SceneService
  • MessageService

Supporting Services

  • None

Configuration

  • None

Asset Changes Processed

  • None

Entity Properties Recognized

  • PhysXShape model
    • ShapeTranslation
    • ShapeRotation
    • ShapeCollisionGroup
    • ShapeType
    • ShapeDimensions
    • SceneName
    • Position
    • Rotation
    • Scale
  • PhysXTrigger model
    • TriggerOnEnter
    • TriggerOnStay
    • TriggerMinInterval
    • TriggerOnLeave

Messages Sent

  • None

Behaviors Invoked

  • OnPhysXTrigger, when collision requirements are fulfilled for an entity with the PhysXTrigger model. Note that no collision occurs when a trigger is created inside another trigger, or a trigger is destroyed inside another trigger, even if that pair of triggers is flagged to report ON_ENTER or ON_LEAVE.

Related Built-ins

  • None

Notes

  • A collision occurs when two entities with the PhysXTrigger or PhysXShape model intersect. The PhysXTrigger model has properties that determine if OnPhysXTrigger is called when the two entities:
    • start to intersect
    • continue to intersect
    • have just stopped intersecting
  • The PhysXTrigger model mixes in the PhysXShape model. All of the properties that are used in the PhysXShape model are also used for PhysXTrigger entities.
  • Because Gamebryo and PhysX have different conventions for origins of shapes, PhysX trigger volumes may not line up with the visualizers displayed in World Builder (particularly for cubes, cylinders, and capsules), typically being translated upwards by half of the volume's height. This can be checked using nVidia's Visual Remote Debugger, or the PhysX debug geometry, with your application.
