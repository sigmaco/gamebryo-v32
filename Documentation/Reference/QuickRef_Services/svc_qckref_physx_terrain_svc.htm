PhysX Terrain Service (PhysXTerrainService) Quick Reference

For more information about this service, see the egmPhysXTerrain::PhysXTerrainService reference documentation.

Library/Namespace

egmPhysXTerrain

Display Name

PhysXTerrainService

Default Priority

1699

Priority for PhysX services varies depending on whether PhysX is being run synchronously or asynchronously; see PhysX system service setup for sample orderings.

Required Services

  • MessageService
  • PropService
  • SceneGraphService
  • SceneService
  • TerrainService

Supporting Services

  • None

Configuration

  • PhysX.Framework.DisableTerrainDebugRenderer

Asset Changes Processed

  • This service does not process asset changes directly and lets the TerrainService handle those. It will however use the NiTerrainCustomDataPolicy interface to register custom callback allowing the handling of physx data found in the terrain file.

Entity Properties Recognized

  • PhysXTerrain model:
    • SceneName
    • PhysXTerrainMaterialBase

Messages Sent

This service does not send any messages directly. See TerrainService quick reference page to find out the messages handled by the parent class.

Behaviors Invoked

  • None

Related Built-ins

  • None

Notes

  • PhysXTerrainService is responsible for loading PhysX-encoded terrain data from the terrain .dof file and adding it to the scene. Loading and unloading the data uses the NiTerrain streaming system. The physx actors will only be created when a PhysXScene is loaded and out of the Simulation - Fetch step. This can therefore happen when the data is loaded, when a PhysXScene added message is caught or during the service's Tick if creation previously failed (usually due to attempting creation while in a Simulation - Fetch sequence.
