Introduction to the NiPluginToolkit Library

Game developers often need to tweak the art pipeline to suit their title.  For example, adding new classes to the scene graph and manipulating the scene graph structure is very difficult in a static pipeline.  Therefore, Gamebryo includes an extremely flexible and customizable art pipeline, enabling easy extensions of the Gamebryo toolset, and enabling developers to create their own custom tools with a minimum of effort.

A component framework architecture has been developed for the 3D modeler plug-ins:  the Gamebryo 3ds max Plug-in and the Gamebryo Maya Plug-in.  This architecture and the framework to support it have been dubbed the NiPluginToolkit.

The NiPluginToolkit architecture consists of various plug-ins designed to work within a standard framework.  Plug-ins are distributed in the form of dynamic-link libraries (DLLs) that are loaded by the framework when needed.  Multiple plug-ins can reside in a single DLL, if desired.

Note to Artists

See the Scripting the Art Pipeline in 3ds Max or Scripting the Art Pipeline in Maya sections for information on how to use the NiPluginToolkit system when exporting and viewing files from the art package.
