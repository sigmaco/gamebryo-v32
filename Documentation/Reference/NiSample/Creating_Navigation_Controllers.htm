Creating Navigation Controllers

NiSample uses the NiUserInterface library to provide a consistent set of navigation controls for all demos.

NiSample provides several member variables for use with navigation controllers:

  • m_spOrbitController and m_spFlyController—To enforce consistency of navigation controls, these two controllers are created by default.
  • m_bUseNavSystem—Used to determine whether or not a derived class will use the navigation system.
  • m_kNavUpAxis—The default up axis for the navigation controllers. This value should be tweaked by a derived class.
  • m_fNavDefaultScale—The default scalar for interacting with navigation controllers. This value should be tweaked by a derived class.

NiSample provides several methods for interacting with the navigation controllers:

  • CreateUISystem—Create the UI system and the navigation system
  • CreateNavigationControllers—Override or add new controllers. The next navigation controller can be switched to
  • ToggleNavHelp—Toggle the visibility of the navigation help UI.
  • ChangeController—Switch to the next navigation controller in the list.
  • UpdateControllerLog—Updates the log entries associated with the current navigation controller.
