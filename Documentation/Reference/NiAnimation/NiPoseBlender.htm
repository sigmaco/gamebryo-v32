NiPoseBlender

: public NiObject

<NiPoseBlender.h>

Description

If more than one sequence is simultaneously active on an NiControllerManager, data from multiple pose buffers is blended via an NiPoseBlender to form the composite animation state. This final result is stored in a separate NiPoseBuffer owned by the pose blender. Since this final output is dependent on the sequences in question, each controller manager maintains its own pose blender.

The blending operation computes a finalized weight for each data item based on the priority, weight, and transition spinner values of the active sequences. With the exception of referenced evaluators, this finalized weight dictates the item's contribution to the final result. In the case of referenced evaluators, the evaluator with the highest finalized weight is propagated to the final pose buffer.

The pose blender makes use of an active pose list to quickly eliminate active sequences that do not contribute to the final state. Occasionally, this step eliminates all but one active sequence. As an efficiency, the pose blender returns the pose buffer associated with this sequence instead of copying the contained data to the pose blender's own pose buffer.

See this section in Animation Basics for additional information regarding sequence blending with priority and weight.

Supported Member Functions

There are no supported member functions as NiPoseBlender acts as an Emergent internal extension to NiControllerManager.
