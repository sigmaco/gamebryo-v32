Shared Animation Data Objects

Series of keys are often stored in shared data objects such as NiPosData and NiFloatData. Most interpolators have access to their keys through a shared data object instead of directly holding the key arrays. This approach enables multiple interpolators to use the same animation key data without duplicating the keys in memory. These shared data objects are responsible for the eventual deletion of the key arrays passed into them during construction or through Replace calls, so it is not necessary for an application to explicitly track and delete these key arrays.
