NiBoolTimelineInterpolator

: public NiBoolInterpolator

<NiBoolTimelineInterpolator.h>

Description

NiBoolTimelineInterpolator uses NiBoolKeys to animate a bool value over time. It differs from NiBoolInterpolator in its interpolation method. Instead of simply returning the value of the previous key before the update time like NiBoolInterpolator does, NiBoolTimelineInterpolator ensures that keys have not been missed between the two updates. If the value for the current update is the same as that of the last update, the animation keys between the two update times are traversed to make sure that a key with the opposite value is not found. If one is, that opposite value is returned instead of the current value. For example, consider a key array with the following values:

Key Time Key Value
0.0 false
0.5 true
0.501 false
1.0 false

Notice that the time between the middle two keys is very small. If the update frequency is low enough, it is possible for two successive updates to miss the "true" key altogether. NiBoolTimelineInterpolator will not miss the "true" key and will return true the next update after the "true" key occurs. This type of interpolator is primarily used by particle system controllers in Gamebryo to ensure that particle emissions are not missed.

Constructors

NiBoolTimelineInterpolator()

This constructor creates an NiBoolTimelineInterpolator containing invalid data.

NiBoolTimelineInterpolator(NiBoolData* pkBoolData)

This constructor creates an NiBoolTimelineInterpolator with the specified NiBoolData.

NiBoolTimelineInterpolator(bool bPoseValue)

This constructor creates an NiBoolTimelineInterpolator with a pose value of bPoseValue and a NULL NiBoolData.

Virtual Function Overrides

virtual bool Update(float fTime, NiObjectNET* pkInterpTarget, bool& bValue)

Overridden from NiInterpolator. Returns true to indicate that the returned boolean value is valid. If there are animation keys in this interpolator, the value will be the interpolated value. Otherwise, this value will be the pose value if there are no keys and the pose has been set.
