NiColorInterpolator

: public NiKeyBasedInterpolator

<NiColorInterpolator.h>

Description

NiColorInterpolator uses NiColorKeys to animate an NiColorA value over time. This NiKeyBasedInterpolator only supports one key channel.

Constructors

NiColorInterpolator()

This constructor creates an NiColorInterpolator with invalid data.

NiColorInterpolator(NiColorData* pkColorData)

This constructor creates an NiColorInterpolator with the specified NiColorData.

NiColorInterpolator(const NiColorA& kPoseValue)

This constructor creates an NiColorInterpolator with a pose value of kPoseValue and a NULL NiColorData.

Virtual Function Overrides

virtual bool Update(float fTime, NiObjectNET* pkInterpTarget, NiColorA& kValue)

Overridden from NiInterpolator. Returns true to indicate that the returned  NiColorA value is valid. If there are animation keys in this interpolator, the value will be the interpolated value. Otherwise, this value will be the pose value if there are no keys and the pose has been set.

virtual bool IsColorAValueSupported() const

Overridden from NiInterpolator. Returns true indicating that NiColorA values are supported.

Supported Member Functions

NiColorData* GetColorData() const

void SetColorData(NiColorData* pkColorData)

These functions get and set the NiColorData for this NiColorInterpolator.

void SetPoseValue(const NiColorA& kPoseValue)

This function sets the pose value for this interpolator. If the interpolator has an NiColorData object, it is set to NULL.

NiColorKey* GetKeys(unsigned int& uiNumKeys, NiColorKey::KeyType& eType, unsigned char& ucSize) const

This function returns the keys currently held by this interpolator. The number of keys, key type, and key stride are returned in uiNumKeys, eType,and ucSize.

void ReplaceKeys(NiColorKey* pkKeys, unsigned int uiNumKeys, NiColorKey::KeyType eType)

This function specifies the key data to store. Note that NiColorInterpolator will not copy the NiColorKeys -- it assumes ownership of the keys and deletes any previous keys.

void SetKeys(NiColorKey* pkKeys, unsigned int uiNumKeys, NiColorKey::KeyType eType)

This function specifies the key data to store. Note that NiColorInterpolator will not copy the NiColorKeys -- it simply overwrites the pointers to the old keys. This action can easily cause memory leaks, so calls to SetKeys should generally be avoided in favor of ReplaceKeys.
