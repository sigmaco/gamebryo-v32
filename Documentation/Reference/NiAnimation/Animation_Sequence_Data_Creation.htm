Animation Sequence Data Creation

The sequences used with the NiControllerManager and NiActorManager are generated upon export from 3ds max and Maya with the Controller Extractor export plug-in. Refer to the 3ds max and Maya artist manuals for how to create animation sequences contained within KF files.

To create KF files that give good results with the NiControllerManager, some guidelines should be followed when authoring animations. The NiControllerManager class makes certain assumptions about the animation data that it will use. Most of the guidelines are very easy to follow, provided all the sequences for a model are based on the same character.

  1. All controllers in a given sequence should have the same timing information, such as frequency, phase, CycleType, AnimType, and begin and end key times.
  2. All sequences that will be managed by the same NiControllerManager should be based on the same model, meaning the structure of the model as well as the transforms at each NiAVObject. If the base models are different, the animations may not look correct.
  3. All objects with controllers must have unique names that are not duplicated elsewhere in the NiControllerManager-controlled scene graph. The KF file uses these object names to match controllers to the objects they control.
  4. The scene graph pattern of parent and child nodes having similar names whereby the child's name equals the parent's name concatenated with " NonAccum" is reserved for use by Emergent. Artists should avoid this naming convention when creating models in 3ds max and Maya to ensure correct operation of transformation accumulation. For example, a node named "elbow" should not have a child node named "elbow NonAccum".
  5. The NiControllerManager class makes use of NiTextKeys to control the timing of morphs and blends. If a sequence will be used in a morph or blend, it is important to include the NiTextKeys in the sequence. See the artist manuals for 3ds max and Maya for details on creating the NiTextKeys.
