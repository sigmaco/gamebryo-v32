Changing the Display Parameters of an D3D11Renderer

The mechanism for changing display parameters in a D3D11Renderer varies depending on exactly what parameters need to be changed.

For a simple width/height change, such as in response to a WM_SIZE message, applications can call the ResizeBuffers function

efd::Bool ResizeBuffers(efd::UInt32 width, efd::UInt32 height, HWND hOutputWnd = NULL)

The hOutputWnd parameter is used to determine which swap chain to resize - usually, this will be the window that received the WM_SIZE message in applications that have more than one rendering window. Passing in NULL for this parameter will resize the default swap chain. If 0 are passed in for the width and height, D3D11 will automatically resize the buffers to match the window.

For more drastic changes to the display parameters, an application will need to destroy the existing swap chain and create a new one. In the D3D11Renderer, the default swap chain has no particular significance beyond being designated as the "default", so applications can destroy it just like any other swap chain, using DestroySwapChainRenderTargetGroup. The application can then create a new swap chain using CreateSwapChainRenderTargetGroup that has the desired display parameters. An application will then need to designate that new swap chain as the default one by calling SetDefaultSwapChainRenderTargetGroup and passing in the handle to the output window for the new default swap chain.

It is necessary to destroy the existing default swap chain for a given window before creating the new swap chain with the new display parameters. If the application attempts to create a new swap chain first before destroying the existing one, the CreateSwapChainRenderTargetGroup function will simply return a pointer to the existing default swap chain, and the operation will not succeed.
