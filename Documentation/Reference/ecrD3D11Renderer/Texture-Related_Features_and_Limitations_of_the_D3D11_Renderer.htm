Texture-Related Features and Limitations of the D3D11 Renderer

The following sections describe basic texture-related features and limitations of the current version of the D3D11Renderer.

Cube Mapping

The D3D11Renderer supports both rendered and source cube mapping. The DIFFUSE_CUBE_MAP and SPECULAR_CUBE_MAP texture coordinate generation modes are supported.

Specular Bump Mapping

The D3D11Renderer supports specular bump mapping via the NiStandardMaterial pipeline.

Diffuse Bump Mapping

The D3D11Renderer can be used to render using diffuse Dot3 bump mapping by applying a normal map to an object. Normal maps should be in tangent space.

Non-Power-of-Two Texture Size

The D3D11Renderer supports non-power-of-two textures. Note that some D3D11 texture formats, such as the block-compressed formats, may still have some size limitations.

Texture Downsizing

The D3D11Renderer does not downsize textures automatically.

Texture Compression

The D3D11Renderer supports loading pre-compressed texture data, and can support on-the-fly compression if a compressing image converter is used. See the Texturing documentation for details of how to create compressed texture data. The D3D11Renderer will load compressed data directly to the 3D hardware.

Texture Apply Modes Supported

The D3D11Renderer does not support the APPLY_DECAL texture-apply mode, only APPLY_REPLACE and APPLY_MODULATE.

Texture Format Requests

Texture format requests are fully supported via the NiTexture::FormatPrefs object the D3D11Renderer queries for all available Gamebryo texture formats.

Palettized Texture Formats

The D3D11Renderer does not support palettized textures. To compensate for this, palettized textures must be expanded by the image converter at load time. This will be done automatically when an appropriate image converter (such as the NiDevImageConverter or any other converter that can expand palettes) is in use.

Sphere Mapping

The D3D11Renderer defaults to world-space sphere mapping, as do most other renderers in Gamebryo.

Load Texture Files Direct to Renderer Hint

The D3D11Renderer does not support the NiSourceTexture hint to load texture files directly into renderer-specific resources.

Persistent Source Texture Renderer Data

The D3D11Renderer supports NiPersistentSrcTextureRendererData objects as the means to load textures directly into renderer-specific resources.

Dynamic Textures

The D3D11Renderer supports the NiDynamicTexture class.

Texture Input to all Shader Programs

The D3D11Renderer supports sampling shader resources from all shader program types, though NiFragmentMaterial-based shaders can only sample in pixel shaders.

Antialiased Rendered Textures

The D3D11Renderer supports creating antialiased rendered textures by passing in the appropriate Ni2DBuffer::MultiSamplePreference to the NiRenderedTexture ::Create functions.
