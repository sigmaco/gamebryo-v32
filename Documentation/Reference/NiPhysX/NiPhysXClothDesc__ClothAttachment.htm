Inheritance graph for NiPhysXClothDesc::ClothAttachment

Description

Class for attachments.

Each instance of this class holds the attachment information for a cloth object. Multiple instances may be required for one piece of cloth if the cloth is attached to more than one shape or has both shape and collision type attachments.

Public Attributes

NiPhysXShapeDescPtr m_spShape

The shape to which the attachment is made, or NULL if it is a global attachment.

NiUInt32 m_uiVertexCount

The number of vertices that are attached or 0 if this is a "collision" type attachment.

NiUInt32* m_puiVertexIDs

The vertices that are attached.

There must be m_uiVertexCount entries in this array. It should be allocated using NiMalloc or NiAlloc.

NxVec3* m_pkPositions

The locations of the attachment on the shape or in the world.

There must be m_uiVertexCount entries in this array. When m_spShape is non-NULL, these positions are treated as points in the shape's local coordinate system, otherwise they are global locations. The array should be allocated using NiMalloc or NiAlloc.

NxU32* m_puiFlags

Public Methods

ClothAttachment()

Constructor will create an attachment of the form "attach to all shapes currently penetrating", and no flags.

~ClothAttachment()

Destructor frees all memory.
