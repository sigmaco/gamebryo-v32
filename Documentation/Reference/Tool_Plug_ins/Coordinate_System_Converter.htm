Coordinate System Converter

Type: PROCESS

Version: 1.0

<CoordinateSystemConverter.dll>

Description

By default, Maya uses the y-axis to represent the up vector. 3ds max, by default, uses the z-axis for the up vector. Both art packages offer solutions for changing this default behavior, but it might be advantageous to have the art be converted into a consistent orientation. This is done by adding the appropriate rotation matrix on each of the child NiNode objects under the root node. If one of these child nodes contains animation data a parent node must be added. This parent node automatically gets added and the rotation matrix is applied to this parent node. Note that this does cause an extra node to be created that did not exist in the original scene graph. The name of this parent node is "child name" + "Transform". The addition of this node must happen to allow the animation to play correctly.

Options

Coordinate System Converter Options

Please be sure you select the correct option for the art package that you are using. There is no error checking to make sure you are using the correct option. If you are exporting from Maya be sure to have the "Maya to Max" option selected. Undesired behavior may result if you select the incorrect option.
