PhysX Ragdoll

Type: PROCESS

Version: 1.1

<PhysXRagdoll.dll>

Description

This plug-in takes the PhysX ragdoll constructed from a skinned skeleton and attaches the ragdoll to the skeleton so that at runtime it can be used to drive the motion. This gives artists an option when creating their rag-doll: create it directly on the skinned skeleton (only a feasible option in Max) or create a separate rigid-body hierarchy and attach it to the skeleton with this tool.

Options

PhysX Ragdoll Options

  • Skeleton Node name: The name of the root node of the skeleton from which the ragdoll was created. Any node that contains the entire skeleton among its decendents is sufficient, hence this defaults to the scene root.
  • Rag-doll Node name: The name of the node that parents all the rigid body nodes representing the ragdoll and its joints.
  • Prefix: The prefix string that identifies rigid body nodes with bones. If the prefix is XXX and the postfix is YYY, then a bone named "arm" is matched with a rigid body named "XXXarmYYY".
  • Postfix: The postfix string that identifies rigid body nodes with bones. If the prefix is XXX and the postfix is YYY, then a bone named "arm" is matched with a rigid body named "XXXarmYYY".
  • Keep collision shape nodes: The geometry representing the collision shapes is typically not rendered, and hence is deleted. Check this box if you would like to keep, and therefore render, the Gamebryo geometry for the shapes. This has no impact on the shapes used by PhysX; it only impacts those rendered by Gamebryo.
