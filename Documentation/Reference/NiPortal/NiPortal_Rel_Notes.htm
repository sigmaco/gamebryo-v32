Gamebryo LightSpeed 3.0

  • Adding support for tools or other systems to disable portalling. This will result in a normal scene graph traversal. Use the NiRoomGroup::Set/GetPortallingDisabled static methods.

Gamebryo 2.0.0

  • Portals now generate a tighter set of screen-aligned bounding rectangles, generally leading to tighter portal culling of both existing and new portal datasets.
  • The new NiFixtureCuller object (which is used auotmatically by the portal system) avoids adding the same object to the same visible array more than once in a rendering pass. As a result, connecting a given pair of rooms with more than one portal will no longer result in an increased amount of overdraw (or alpha blending issues).
  • The PortalConverterDLL has been rewritten to use a more robust convex hull algorithm, resulting in better, simpler sets of wall planes, which can increase performance. It can also handle multiple tagged NiRoomGroup objects within a single scene graph or NIF file.

Known Issues and Limitations

  • Scene Designer is incompatible with the Gamebryo portal system (NiPortal library). Any NIF files that contain NiPortal classes will fail to load when loaded by Scene Designer. Such NIF files should not be loaded as part of an entity by an application using the NiEntity library, either. Some NiPortal classes cannot be cloned and thus may crash if used with NiEntity's external asset handlers.
