NiSystemCursor

: public NiCursor

<NiSystemCursor.h>

Description

The NiSystemCursor class represents a cursor whose image looks like the Windows system cursors. There are a number of different system cursor types available; they are described in the Notes section below.

It is important that the image SystemCursors.tga be contained in the Data directory of any application that uses NiSystemCursor.

Constructors

The constructor for this class is protected. To create an NiCursor object, the following static NiCursor function is used rather than an NiCursor constructor.

static NiCursor* Create(NiRenderer* pkRenderer, NiRect<int>& kRect, SystemCursor eType, const char* pcImageFilename = NULL, const NiRenderTargetGroup* pkTarget = NULL)

pkRenderer points to the renderer that will be used to render the cursor. kRect is an NiRect object that contains the bounds of the cursor, in screen space pixels. Normally this will be the size of the entire back buffer, but it can also be a subset of it, or it can even extend beyond. The type of cursor, described below, is contained in eType. The image filename indicates the full path for the texture to be used. If pcImageFilename isn't provided a default will be used; "../../Data/SystemCursors.tga" on the PC and PS3, and "D:/Data/SystemCursors.tga" on Xenon. The NiRenderTargetGroup pkTarget is the buffer that the screen space pixels specified in kRect refer to, and it is the surface that the NiCursor will be displayed on. If pkTarget is NULL, then the default render target will be used.

Static Member Functions

static void Shutdown()

Destroys all the static instances of this object for all system cursor types.

static NiSystemCursor* GetCursor(SystemCursor eType);

Returns a pointer to the static instance of this object for the given system cursor type eType.

Supported Member Functions

const SystemCursor GetType() const;

Returns an enumeration indicating the type for a system cursor.

SystemCursor SetType(SystemCursor eType)

Sets the type for a system cursor.
