Preventing Optimization on Objects

To prevent optimization functions from changing or deleting objects or nodes, each object to be retained must be tagged with an NiStringExtraData object containing the value string "NiOptimizeKeep". This string will indicate to the optimization functions that the particular object should not be optimized. It does not, however, prevent optimization of objects below that object in the scene graph. Each object needs to be tagged individually.

In 3ds max, an object is tagged by adding the string "NiOptimizeKeep" to its User-Defined Properties.  In Maya, extra attributes can be added to an object to indicate that it should not be optimized.  Please refer to the documentation for the Gamebryo 3ds max Plug-in and the Gamebryo Maya Plug-in for specific instructions on how to prevent optimization on objects.

An optimization function causes these "NiOptimizeKeep" NiStringExtraData objects to be removed from the objects in the scene graph to save memory.  This function should only be run after all desired optimizations have been performed on the scene graph.

An additional extra data flag is used to prevent mesh-based optimizations, such as vertex cache optimization and view-independent vertex order optimization. Add extra data with the string value "NiNoOptimizeMesh" to a node to prevent optimization of that node, or any of its children. This flag is not added by any of the DCC plug-ins, but it may be added explicitly by an artist or by other Gamebryo tool plug-ins to prevent changes to the vertex ordering in a mesh.
