Description

A clock based on high precision counters (QueryPerformanceCounter on Win32) for accessing "real time" since some arbitrary point in history (such as whenever the hardware last booted or whenever the program initialized).

The units are seconds.

Synchronization between multiple machines is supported, but not provided, so values should not be compared between multiple processes unless you use the synchronization features. Also, unless you synchronize to some fixed point in Calendar Time, you should not persist the resulting absolute values (consider persisting a delta from "now" instead, for example).
