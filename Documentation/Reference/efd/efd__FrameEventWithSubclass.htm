Description

A FrameEventWithSubclass is an IFrameEvent implementation where the frame event instance itself is passed to the event handler functions.

The FinalType template argument must be a class that derives from FrameEventWithSubclass and is expected to implement additional data members or methods that will be available to the handler method. The FinalType instance is allocated and passed to ServiceManager::RegisterFrameEvent and then passes itself to each handler.

Use this class carefully, it performs a static_cast to convert from FrameEventWithSubclass to the FinalType and such conversions are not always safe.

In general frame event objects are not expected to maintain dynamic state, they are primarily just containers for the handler lists. If you need to maintain complex or dynamic state, consider placing that data into a system service and use the FrameEventWithOwner class instead.
