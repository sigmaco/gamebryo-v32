Clipped Projected Textures in DX9

The NiDX9Renderer supports "clipped" projected lights and shadows on most multi-texture capable hardware. Please see the Texturing documentation for details of this feature.

In the shader-driven pipeline employed by NiStandardMaterial, clipping is done mathematically in the pixel shader. However, in the legacy pipeline employed by NiD3DDefaultShader, for the purposes of counting textures towards the maximum number of simultaneous textures supported by the renderer on a single object, each clipped projected effect counts as 2, rather than 1. In addition, each clipped projected effect applied to an object will require a separate rendering pass.
