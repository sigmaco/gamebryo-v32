Renderer options from the Command Line

The NiApplication based object will process the command line if it exists and looks for flags for renderer options. Command line options are processed after loading of settings from the INI file and overrides those settings.

Following is a list of the command-line options. In cases where options have synonyms, they are separated by commas.

-dlg, -dialog, -showdialog

This will cause the renderer settings dialog to reappear, even if the user had previously checked the "Do not show again" option. This is useful if the "do not show" option was used, but it is later decided that a setting needs to be changed.

-d3d9, -dx9

Causes the application to attempt to use the NiDX9Renderer.

-d3d10, -dx10

Causes the application to attempt to use the NiD3D10Renderer. If the application is not running on a machine that is capable of running D3D10, this option will not work.

-d3d11, -dx11

Causes the application to attempt to use the ecr::D3D11Renderer. If the application is not running on a machine that is capable of running D3D11, this option will not work.

-full, -fullscreen

Causes the application to run in full screen mode.

-win, -window

Causes the application to run in a windowed mode.

-perfhud, -nvperfhud, -nvperf

Allows the application to make use of NVIDIA's NVPerfHUD utility, by requesting the appropriate adapter.

-ref, -refrast

Selects the REF (reference) device. This is a high-quality, low performance software reference rasterizer for Direct3D. This option is useful for testing a hardware card's actual behavior in comparison with a slower, but possibly more reliable reference behavior. This option is also useful for pixel shader debugging. Note that using this option will naturally cause the application to run much slower due to the absence of hardware support.

-hal, -halrast

Selects the HAL (hardware) device.

-pure

Selects a pure device. This will only have any effect if the REF device is not selected.

-hwvertex

Specifies that vertex processing will be done in hardware.

-mixvertex

Specifies that vertex processing can be switched between hardware and software.

-swvertex

Specifies that vertex processing will be done in software. This may be useful for testing a hardware card's vertex processing capabilities or debugging vertex shaders; however it will also slow down the application.

-vsync

Forces the application to wait for the monitor's vertical sync before displaying the next frame.

-ffp

Forces the renderer to use the Gamebryo legacy pipeline that makes use of the D3D fixed-function rendering pipeline.
