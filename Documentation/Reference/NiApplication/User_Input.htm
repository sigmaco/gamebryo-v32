User Input

NiApplication uses the NiInputSystem class to handle user input. Input happens in two stages, each handled by a virtual NiApplication function.

virtual void UpdateInput()

Updates the state of all devices held by the input system. It is not usually necessary to override UpdateInput.

virtual void ProcessInput()

Implements the response to input events. In order to react to user input, it is necessary to override ProcessInput.

For more detailed information please refer to the NiInput documentation.

Accessing the Input System

The input system created by NiApplication can be accessed calling NiApplication::GetInputSystem.
