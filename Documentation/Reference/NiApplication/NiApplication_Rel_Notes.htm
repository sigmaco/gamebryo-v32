Added Features, Changed Behavior, and Fixed Bugs

Gamebryo LightSpeed 3.2

  • None in this release.

Gamebryo LightSpeed 3.1

  • Fixed an issue in the ReadConfig method of NiRendererSettings. The issue was that unless every property configuration was specified, items that were unspecified would have their default values overwritten with an erroneous value.

Gamebryo LightSpeed 3.0

  • The settings dialog logic used in Gamebryo samples on Win32 has been refactored into its own separate library. Also, its internals have been rewritten to be more stateless.
  • The default visual tracker setup now includes a separate tracker for time spent in DisplayFrame. This can often be used as an approximation of how GPU-bound an application is.

Gamebryo 2.6.0

  • PS3 only: A new method (WaitForUserToExit) has been added to ensure that the user quit message has been received before exiting main. It may also be called in exceptional circumstances that are non-recoverable. For more information, see the NiApplication reference documentation.
  • PS3 only: Support for the foot switch has been added. It is only enabled in GAMEBRYO_PROFILE builds and by default takes a screenshot. The behavior can be changed by overriding the OnFootSwitchPressed method defined in NiApplication.

Gamebryo 2.5.0

  • The operation of the "-nvperf" command-line argument has been updated to work with NVPerfHUD 5.0.
  • The library now uses the NiFile::GetFile method for loading assets.

Gamebryo 2.3.0

  • The camera created in NiApplication::CreateCamera is now set up with an aspect ratio that matches that of the renderer.
  • NiApplication has been modified to optionally support rendering using the new frame rendering system. A description of the changes made can be found here.
  • Two new utility functions have been added: FindFirstCamera and AdjustCameraAspectRatio.
  • NiApplication has been updated to link in both the NiDX9Renderer and NiD3D10Renderer libraries, so an application can choose between the rendering APIs at runtime.
  • Added support for the PS3 platform.

Gamebryo 2.0.0

  • The NiApplication::MeasureTime function has been changed to take advantage of NVIDIA's NVPerfHUD 3 utility on Win32.. When NiApplication::m_bNVPerfHUD is true, then MeasureTime will return true when the elapsed time is 0.0. This allows NVPerfHUD to freeze the frame for scene analysis.
  • NiApplication for Win32 now assumes the presence of a keyboard and mouse in NiApplication::CreateInputSystem. An error message box is displayed if either fail to be initialized or if the NiInputSystem fails during its creation.
  • NiCursor::Draw no longer accepts any parameters. A camera is no longer needed.
  • NiFrameRate::AddToCamera/RemoveFromCamera have been replaced by NiFrameRate::GetElements, which returns the screen-space geometry that represents the visible geometry for frame rate feedback.

Known Issues and Limitations

There are no known issues or limitations with this library at this time.
