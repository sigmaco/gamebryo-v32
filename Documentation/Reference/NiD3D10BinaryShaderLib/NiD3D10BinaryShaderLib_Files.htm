NiD3D10BinaryShaderLib Files

Description

These files, NiD3D10BinaryShaderLib.cpp and NiD3D10BinaryShaderLib.h, provide the interface function for static lib based builds of the shader library. It defines a single function for creating the library, which the application must call during ClassCreationCallback. Refer to the documentation on Exposing your ShaderLibrary to the tool-chain for more details.

Functions

bool NiD3D10BinaryShaderLib_LoadShaderLibrary(NiRenderer* pkRenderer, int iDirectoryCount, char* apcDirectories[], bool bRecurseSubFolders, NiShaderLibrary** ppkLibrary)

This function is provided for static lib based builds of applications that wish to utilize the NiD3D10BinaryShaderLib. The function simply creates an instance of the NiD3D10BinaryShaderLibLibrary and returns it. The application MUST be aware of this function if it is using static lib-based builds.

NID3D10BINARYSHADERLIB_ENTRY bool LoadShaderLibrary(NiRenderer* pkRenderer, int iDirectoryCount, char* apcDirectories[], bool bRecurseSubFolders, NiShaderLibrary** ppkLibrary)

This function defines the interface that the DLL-based shader factory system will use to get an instance of the library.

NID3D10BINARYSHADERLIB_ENTRY unsigned int GetCompilerVersion(void)

This function simply returns a value that identifies the version of the MSVC compiler used to build this library. This prevents libraries built with differing compiler versions from being used together and causing memory corruption errors. Refer to the documentation on Visual Studio compiler versions for more information.

BOOL WINAPI DllMain(HINSTANCE hinstDLL, ULONG fdwRead, LPVOID lpvReserved)

Function supplied strictly for DLL-based builds. Calls the NiD3D10BinaryShaderLibSDM::Init function when the process is attached, and NiD3D10BinaryShaderLibSDM::Shutdown when the process is detached.
