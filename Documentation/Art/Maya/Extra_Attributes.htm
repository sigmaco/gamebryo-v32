Extra Attributes

Within Maya, users can add "Extra Attributes" to objects. These "Extra Attributes" are used to store user-defined information associated with the object. The Gamebryo Maya Plug-in supports a large number of different useful data types. Unfortunately, the Maya user interface only supports the creation of a limited number of these data types. The "addAttr" command in Mel script is necessary to create all of the different data types.

In the example below, a string attribute named "NiData_ObjectString" has been added to the selected object.

addAttr -ln NiData_ObjectString -dt "string"

Extra Attributes

There is a simple naming convention that must be observed so that the Gamebryo Maya Plug-in can know which attributes you wish to export. All extra attributes you wish to export must start with "NiData_". Once exported, your "Extra Attribute" will be attached to an equivalent object in the scene graph as an NiExtraData. At this time "Extra Attributes" can only be added to shapes and groups. The Gamebryo attribute name will match the attribute name found in Maya.
