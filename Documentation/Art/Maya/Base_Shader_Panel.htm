Base Gamebryo Shader Panel

The base panel is used to easily update the current shader options.

Base Gamebryo Shader panel

Type

The standard Maya interface for selecting the Maya shader node.

Gamebryo Shader

This dropdown shows you all the shaders currently found in your Shader directory. By default, the Gamebryo Maya plug-in uses the following location as the shader directory:

%EMERGENT_PATH%\Media\Shaders\Data

Note

The EMERGENT_PATH environment variable is set during installation.

When the Gamebryo Maya plug-in is loaded the directory is scanned for all .nsf and .fx files. When a shader is successfully loaded the name of the shader is added to the menu. The first entry in the list is "Standard Material". Standard Material represents the Gamebryo fragment shader that automatically generates a shader based on the input textures. For more information please see Multi Texturing Introduction.

Gamebryo Shader Description

Each Shader has a brief description of what it is intended to do. By selecting from the available shaders, you can see the descriptions of each Shader. Note that "Standard Material" does not have a description.

Implementations

Each Gamebryo Shader has one or more implementation that determines how the Shader works. For example you may have a Shader with two implementations, one for the high end video card and a second which only uses features found on a lower end video card. By default the Shader system uses the implementation best suited for the hardware but this can be difficult for an artist to see how things look. "Exporter Implementation" is used for general exporting. "Viewer Implementation" is used when exporting for a viewer. If a shader only has a single inplementation these fields will be hidden from the UI.

Selected Texture

This dropdown menu stores the names of the available textures. Standard Material will display all standard textures regardless if they are being used. Similarly a Custom Shader will only populate the list with what textures are available regardless if they have been assigned. Changing this field will cause all shapes that are linked to the given shader to display the selected texture.
