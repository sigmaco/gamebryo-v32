Collider types

Colliders are another area where Maya and Gamebryo differ considerably.Within Maya, any object can become a "Collider" for a particle system.Gamebryo has only two default types of collider:spherical and planar. Furthermore, Maya "Colliders" are actually geometry within the scene graph and can be dynamically modified by the geometry's parent translations. Within Gamebryo, "Colliders" are a force that acts on a particle system. The position of the "Collider" is relative to the NiParticle System Controller and not independent as in Maya. To bridge the gap between the two systems, Gamebryo Maya Plug-in is very particular and only considers "Sphere" and "Plane" shapes to be "Colliders". For convenience there are icons on the Gamebryo shelf for automatically creating colliders.

Note:

Due to limitations within the Maya API, you must give "Colliders" unique names.

Planar Colliders

Planar Colliders consider the "Translation X,Y,Z" , "Rotation X,Y,Z", and  "Width" and "Height" to define the location, orientation, and size of the plane.  The "Resilience" is used to define "Bounce" within Gamebryo.

Maya Planar Colliders attributes:

  • Translation X,Y,Z
  • Rotation X,Y,Z
  • Width
  • Height
  • Resilience

Spherical Colliders

Spherical Colliders consider the "Translation X,Y,Z" and "Radius" to define the location and size of the Sphere.  The "Resilience" is used to define "Bounce" within Gamebryo. It is important to remember that spherical colliders are created as a child NiNode in Gamebryo. This additional node has its position and scale set to mirror the bound of the collider.

Maya Spherical Colliders attributes

  • Translation X,Y,Z
  • Resilience
  • Radius

Proxy Spherical Colliders

Proxy Spherical colliders create a sphere around the bounds of an arbitrary geometry. It is important to remember that proxy spherical colliders are created as a child NiNode in Gamebryo. This additional node has its position and scale set to mirror the bound of the collider.
