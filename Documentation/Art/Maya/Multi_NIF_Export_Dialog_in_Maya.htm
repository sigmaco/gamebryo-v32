Multi NIF Export Dialog

Click the Multi NIF Export button to view the Multi NIF Export dialog. This interface lets you tag objects in the scene graph so they will be exported as separate NIF files. This information is added to the Maya scene graph as "Extra Attributes," and these are interpreted by Gamebryo's Multi-NIF Export plug-in.

If you don't select an export script that contains the Muti-NIF Exporter, then this will have no effect. It is important to realize the selected object and all of its children will be exported, so you must be careful to include any linked objects. For example, an error will occur if you try to export a skinned object without its corresponding bones. Similarly, an error will occur if you try to export a particle system without its emitter, fields, and colliders.

Multi NIF Export button

Multi NIF export dialog

Use this field... To...
Multi Export Objects Select from the available objects in the scene that are tagged for multi-NIF Export. When you select an object, its current settings display and you can override them here.
File Name Specify the file name for the exported object.
Use Global Multi-Export Settings Set how global multi-export options relate to this object. Un-check this option to override the default options.
Use Object Name As File Name Use the object's name in Maya as the name of the .NIF file.
Delete From Further Exports Remove this object from the scene graph so that it will not appear in further exports. This will not delete the object from Maya, but it will not be in any additional NIF files that are generated with this export.
Translate, Rotate, Scale Specify the transform of the exported object.
Add Selected Object to the Export List Add Multi-NIF export "Extra Attributes" to the selected object.
Remove From the Export List Remove the selected object from the Multi-Export Objects list. Also removes the object's Multi-NIF export "Extra Attributes".

Multi-NIF Global Settings

Multi-NIF Export Global Settings

The global settings for the Multi-NIF exporter are set on a per export script basis. You will need to use the "Script Editor" interface to change the global settings.
