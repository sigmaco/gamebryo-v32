Gamebryo Remote Viewer - Automatic Versus Manual Updates

The Remote Viewer will automatically update particle systems when Gamebryo particle system nodes are added, deleted, or changed in Maya. The viewer will display the following wait icon in the lower right corner when the scene is being updated. If the scene is particularly large, the wait icon will be displayed for longer periods of time while the data is exported and transferred across the network to be displayed in the viewer.

Wait Icon

Which Maya Nodes are Updated in Real-Time?

Changes to the following nodes in Maya are automatically updated in the Remote Viewer in real-time and do not require a manual refresh:

  • NiParticleSystem
  • NiEmitter
  • NiCollider
  • NiForce
  • Ni*Spawner (includes Collision, Living, and Death)
  • Transform (animated transform nodes require manual refresh)

When Should I Press the Refresh Button or F5?

Changes to any node other than those listed above as automatically updating in real-time require the Refresh button to be pressed to update the scene in the Remote Viewer. For example, when a mesh is added to the scene or a material (NiMultiShader) attribute is changed.

If a change is made in Maya and the wait icon is not displayed on the Remote Viewer, this is a good indicator that the Gamebryo plug-in is not monitoring the attribute in real-time since it has not been automatically sent to the viewer.

After the Refresh button or F5 has been pressed, the Gamebryo export progress dialog will be displayed. Depending on the scene’s complexity this dialog will be displayed anywhere from less than a second to a couple of minutes.
