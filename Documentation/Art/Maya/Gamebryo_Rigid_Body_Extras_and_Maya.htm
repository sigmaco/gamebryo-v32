Gamebryo Rigid-Body Extras and Maya

Here we outline the additional features and issues related to creating PhysX rigid-bodies for use in Gamebryo applications. Almost all of the rigid-body modeling functionality is provided by the PhysX Maya Plug-in, so you should read about that before reading this.

Extra Attributes

Gamebryo export understands two Extra Attributes tags. These tags should be added to nodes of type nxRigidBody (the rigid body transform nodes) using the "Attributes->Add Attributes"" menu item in the attributes tab.

  • NiData_NiPhysXActorGroup, of type integer, can be used on an object that will be a PhysX actor. It specifies the actor group which is used in basic collision filtering. Valid values are 0 to 31, inclusive. 0 is the default.
  • NiData_NiPhysXParticleActor with a string value can be used on an object that will be a PhysX actor and tags that actor as the particle for a PhysX-based particle system. The value of the data is the name of the particle system shape.

Global Application Consistency

Material property containers are also global objects. Every time a Maya scene is exported, as many new material containers are created as there are distinct material properties (elasticity, friction) in the scene. These are given indexes (application global) starting at the value given in the PhysX Extractor tool plug-in options. If multiple props are to be used in the same application, then it is essential that the props use non-overlapping material index ranges. A conservative estimate is that there can be no more materials in a prop than the number of Maya objects that are known to the PhysX system. So if one prop has 5 objects and another has 10, then using 0 and 6 as the export material start indexes will ensure non-overlap. Overlapping ranges will result in a random choice of material for some objects.

Finally, the "Scale" parameter in the PhysX Extractor tool plug-in option should be globally consistent across all props that will be simulated in the same PhysX scene. For simplicity, it is probably best to decide on a suitable scale factor for your game and use it for all exported content. Alternatively, always use the Maya working units to determine a scale factor and use the same working units for all the Maya content in your game.
