Controlling object level of detail in the XSI Plug-in

You can control which objects are associated with level of detail (LOD) representations in your exported NIF file. The exporter defines a custom property set named GamebryoLOD for assigning level of detail metadata to objects in the scene.

You can assign the GamebryoLOD property set to the selected object(s) in either of the following ways:

  • Use the CreateGamebryoLOD custom command
  • Click the Create Gamebryo LOD button on the exporter toolbar.

The value of the Group Name property is an arbitrary string that you make up to tie multiple scene graph objects together in a "LOD group". The value of the Camera Distance property is the camera's distance from the object that will cause this particular object to be displayed at runtime instead of other objects in the same LOD group.

For instance, if you have 3 scene graph objects that represent 3 different levels of detail of the same logical object, the first object (representing the highest level of detail) could have a Group Name property value of "MyLODGroupName" with a Camera Distance property value of 0. The second object would have a Group Name property value of "MyLODGroupName" with a Camera Distance property value of 25.5. The third object (the lowest level of detail representation) would have the same Group Name and a Camera Distance of 75. When the camera distance is between 0 and 25.5, the first object is displayed by Gamebryo. When the camera distance is greater than 25.5 but less than 75, the second object is displayed by Gamebryo. When the camera distance is greater than 75, the third object is displayed.
