Added Features, Changed Behavior, and Fixed Bugs

Gamebryo LightSpeed 3.2

  • None in this release.

Gamebryo LightSpeed 3.1

  • Adding support for XSI 7.5.
  • Fixed a bug where exporting only the current selection would cause XSI to crash.

Gamebryo LightSpeed 3.0

  • Adding support for XSI 7.0.
  • A GamebryoAlphaBlending property set has been added to control alpha blending. A button to add this property set has been added to the GamebryoExportToolbar.
  • A GamebryoVertexColor property set has been added to control vertex coloring. A button to add this property set has been added to the GamebryoExportToolbar.
  • Fixed a bug where changing the path for a process or export script could put XSI in a bad state, preventing future exports.
  • Fixed a bug where modifying a process or export script before the XSI Plug-in loaded put XSI in a bad state, preventing future exports.
  • XSI no longer crashes when a timelock has expired.

Gamebryo 2.6

  • The exporter is packaged as an XSI workgroup
  • Z-buffer rendering properties can be specified per object
  • Custom parameters attached to objects in a scene create NiExtraData objects on corresponding Gamebryo scene graph nodes. This feature creates an easy path for adding application-specific metadata to XSI art assets
  • Export options allow you to export selected portions of a scene; you can also choose which runtime platform you're targeting with an export
  • Level-of-detail rendering properties can be specified per object
  • Shadowing properties can be assigned to drive the Gamebryo shadowing system
  • Lighting can be controlled through the use of light groups. These groups define the set of objects that are affected by a light
  • Clips in the XSI Animation Mixer are exported in a manner that allows extraction via the Gamebryo Controller Extractor Plug-in. This enables use of the Gamebryo Animation Tool to control transitions and blends between animation sequences
  • Additional lighting parameters are exported (attenuation, falloff, animated color and intensity)
  • Alternate Bounding Volumes (ABVs) for scene objects can be specified to drive the Gamebryo collision system
  • The two XSI plug-ins that make up the Gamebryo XSI Exporter are listed in the XSI NetView (Alt+5) under the add-ons tab.
  • During export, a mesh profile can be specified to control how data streams are processed.
  • The Gamebryo XSI Exporter plug-in predefines the custom properties it uses to drive exporter behavior. This makes it easier to automate the creation of this metadata. Refer to the screenshot below of the XSI Plugin Manager to see the predefined custom property sets.

Known issues and limitations

  • Export of Bone / Skin LOD requires the SceneGraphOptimization plug-in.
  • If XSI is not installed before the Crosswalk SDK, errors will occur during export - textures and animations will not be handled properly.
  • The default material created by assigning a GamebryoMaterial is created with a base texture assigned that is a type not supported by Gamebryo. Either turn the base map OFF or change this texture to a type that is supported before exporting.
  • The XSI exporter does not support anisotropic filtering as a texture export option at this time.
  • The XSI exporter currently creates the POSITION, NORMAL, and TEXCOORD streams with the CPU_READ access flag. This may lead to performance penalties.
  • The XSI exporter does not support adding shape animations to the animation mixer. This prevents shape animations from being used with the Controller Extractor plug-in.
  • Nesting an XSI object containing a GamebryoLOD property within another XSI object that contains a GamebryoLOD property does not work. If this is done, neither object will be displayed.
  • The GamebryoMaterial allows the simultaneous assignment of more textures to a single material than Gamebryo currently supports. If more textures are assigned than Gamebryo supports, the error shader will be used instead of the material.
  • If Gamebryo lighting/shadows are applied to objects that contain multiple LOD, the generated shadows may not render as expected.
  • When you are using the IK interactor for animation, keyframes must currently be assigned for all of the bones that the interactor is changing, not just the interactor itself.
  • Vertex data is not unique when exporting from XSI - that is, there may be duplicate vertices. This can have memory and speed implications.
  • When an export fails, sometimes the following error message will display:

    Cannot read XML log file

    To see the real error, specify a log file during export and view the contents of that file.

  • Assigning a Gamebryo-specific parameter (GamebryoLOD, GamebryoLighting) to a selection only assigns the parameter to the first item in the selection.
  • Sometimes animating a model that contains non-uniform scaling does not work and gives an error.
  • Exporting a file from XSI without the .nif extension fails. Make sure the .nif extension is specified when doing an export.
  • Do not use the "LightSun" primitive preset. Crosswalk exports, including both Gamebryo NIF and generic .xsi exports, will cause XSI to crash.
