Using lights in the XSI Plug-in

Lights that are defined in the scene are exported. The following light types are supported, with the supported properties of each light type noted:

  • Ambient light
    • Color
  • Directional lights
    • Transform
    • Diffuse, ambient, and specular colors
  • Point lights
    • Transform
    • Attenuation, falloff
    • Diffuse, ambient, and specular colors
  • Spot lights
    • Transform
    • Attenuation, falloff
    • Spot angles
    • Diffuse, ambient, and specular colors
    • Spot light look-at point is converted to a rotation

Additionally, lighting that affects only a subset of the scene is supported as described in the following sections.

Special Lighting Properties

You can control which lights affect objects on the scene and whether or not they generate shadows. You can also control which objects cast and receive shadows. The exporter defines two custom property sets:

  • GamebryoLighting—for assigning lighting metadata to lights
  • GamebryoShadow—for assigning shadowing metadata to objects

You can assign the lighting property set to the selected object(s) in the scene in either of the following ways:

  • Use the CreateGamebryoLighting custom command
  • Click the Create Gamebryo Lighting button on the exporter toolbar.

You can assign the shadow property set to the selected object(s) in the scene in either of the following ways:

  • Use the CreateGamebryoShadow custom command
  • Click the Create Gamebryo Shadow button on the exporter toolbar.

Gamebryo Lighting Property Set

You can assign lighting properties to lights in the scene. Two properties—Group Name and Shadow Technique—govern how lights affect objects in the scene.

  • Group Name— A space- or comma-separated list of arbitrary "lighting group" names that the corresponding light belongs to. If a light has a Group Name property value of "Baked, Room2" for instance, it means the light belongs to two "lighting groups", one called "Baked" and the other called "Room2", and has the potential to affect objects in the scene that are in at least one of the two groups listed. Lights that do not have the GamebryoLighting property set attached to them are global lights and automatically affect all objects in the scene.
  • Shadow Technique—This property defines what Gamebryo shadowing algorithm should be used by the light. Choosing "None" turns shadow generation off for the light. The value of the Group Name property still controls what obejcts are lit by the light however, even if the Shadow Technique is set to "None".

Gamebryo Shadow Property Set

You can assign the shadow property to objects in the scene. Three properties—Group Name, Ignored Shadow Casters, and Ignored Shadow Receivers—govern how lights affect objects in the scene and inter-object shadowing.

  • Group Name—A space- or comma-separated list of arbitrary "lighting group" names that the corresponding object belongs to. Objects in the same group as alight are affected by the light. They are lit by the light and cast shadows from the light. Objects in the same group as other objects affect one another. They cast and receive shadows on one another. These intra-group interactions can be overridden on a per-object basis by the next two properties.
  • Ignored Shadow Casters—A space- or comma-separated list of lighting group names that the corresponding object should not cast shadows in.
  • Ignored Shadow Receivers—A space- or comma-separated list of lighting group names that the corresponding object should not receive shadows in.
