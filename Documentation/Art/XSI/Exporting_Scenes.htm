Exporting Scenes with the XSI Plug-in

The Gamebryo XSI Exporter uses Crosswalk to export an XSI scene. After you install the exporter, a new Export to Gamebryo menu item appears within Crosswalk's File menu.

When you choose the Export to Gamebryo menu item, a Gamebryo export dialog box displays. You can use this dialog box to specify export options.

At a minimum, you should specify a Gamebryo NIF filename to export to. Choosing a Log Filename is optional. The Gamebryo XSI Exporter uses the Gamebryo Tool Plug-in Pipeline to process data extracted from the XSI scene. You can configure the operations that are performed by clicking the ... or Options buttons to select different scripts to process your scene. The settings in this dialog are stored in the scene and are remembered from export to export.

If you want the Gamebryo Asset Viewer application to be launched automatically after the scene is exported, select (check) the Launch Asset Viewer checkbox. The Target Platform drop-down controls the platform-specific behavior of several Tool Plug-ins. For instance, the Stripify Geometry process plug-in optimizes geometry differently depending on the Target Platform chosen.

The Mesh Profile text field allows you to specify a mesh profile, which is used during the Mesh Finalize stage of the tool plug-in process. These profiles specify how data-streams are to be processed at the end of the tool plug-in pipeline.

You can limit which portions of the scene are exported by selecting (checking) the Export Selection Only check-box. Only selected objects in the scene are exported when this box is checked. Note that if a light is not selected, it will not be exported even if objects that are in the same "light group" as the light are selected for export.

If there are problems with the export, you can view a log by clicking the XSI script editor button to view the log after the failed export.
