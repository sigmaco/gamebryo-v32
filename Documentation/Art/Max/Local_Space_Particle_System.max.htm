LocalSpacePSys.max (Local Space Particle System)

Gamebryo exports out 3ds Max particle systems as an emitter node and a particle system object. The object that you manipulate in Max is actually the emitter node. By default, the Gamebryo particle system will be a child of the emitter node, and it will be set up to be in world space. In some cases, you might want the particle system to reside somewhere else in the hierarchy. This sample file illustrates just such a scenario.

In this 3ds Max sample, there are two animated blow torches, each one with a particle system as a child. The blow torch with red flame has its particles emitted in world space, the default setting. As the torch animates, the tip of the flame lags behind a little bit. The blue flame is set to use its torch as the parent of the resulting Gamebryo PSystem. In this setup, the particle system rotates with the torch and the flame appears strait. Other than the parent and the color, both particle systems have the exact same parameters.

Open up the modifier tab on the SuperSpray named 'BlueFlame' in this example file. Note that the Use Local Space PSystem checkbox in the Gamebryo Export Params rollout is checked. Also note that PSystem Parent Node is set up to use 'BlowTorch01'. This makes the particle system a child of the torch. Note that the emitter will stay in the same place in the hierarchy.

By default, this sample installs to the following location:

%EMERGENT_PATH%\Samples\Models\Max Artist Examples\LocalSpacePSys.max
