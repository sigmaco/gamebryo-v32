Q: I have installed Gamebryo and started up Max. Where is Gamebryo in Max?

A: There are a number of locations in the Max interface that enable you to use Gamebryo features.

You can find the Gamebryo Shader in the Material Editor window under the "Shader Basic Parameters" rollout. The "Gamebryo Shader" is right underneath "Blinn" in the rollout.

You can find the Gamebryo Preview windows under the Utilities panel on the right of the command panel. You will need to click the More button under the Utilities rollout the first time you bring it up. You should add it as a button under the Sets button when have a second because you will be accessing this part of Gamebryo often.

You can find the export dialog under File > Export and selecting "Gamebryo .NIF/.KF/.KFM"

You can also find many additional tools under the Maxscript button under Utilities panel.

We recommend that you create a custom toolbar by going to the "Customize / Customize UI Scheme" dialog. Under the "Toolbars" tab, select "Gamebryo Tools" from the "Category" rollout. Add these buttons to an existing rollout or a new rollout. Save this scheme as DefaultUI.CUI so it will load every time you start max.

The rest of the Gamebryo Max Plug-in is made to be as transparent as possible. When you are working with Max and its tools, you are working with Gamebryo also.

Q: I have a Character-Studio character that animates correctly in Max, but when I look at him in Gamebryo, the mesh is offset from the skin. I have attached the skin to the Bip01 of the character. What is wrong?

A: Do not attach a skin that is bound to a bone rig or CS rig to the root node of that hierarchy the space warp is bound to.

Max should not let this be possible. What is actually happening is that you are translating the skin twice: Once for the space-warp binding and then again because it is attached as a child to the moving rig. A skin should never be attached to the hierarchy that is moving it if it has a space-warp (Skin or Physique) binding to it also. You can create a node above the Bip01 node, or use Max5's character node to organize a bone structure and skin together.

Q: Why does it take longer to export when I have the Stripify Option on?

A: It may take longer to export with the Stripify option on in your scene.

In some cases, a complex scene might take up to 10 minutes longer to export. However, you can get a 30 to 70% performance increase using the Stripify option. While building and examining a Gamebryo asset, you may want this to be off for convenience and faster turnover on your iterations. However, make sure that your final exports have this option enabled for best performance.

Q: Why isn't the base mesh of my Morph correct?

A: This symptom often occurs when a morph's base mesh is exported as the first frame of an animation on the time slider.

You should ensure that the first frame matches your base geometry. We also recommend that you use the Morph modifier rather than the Morph compound object.

Q: My particle system uses transparency. Why does it disappear from certain positions?

A: Large particle systems that read and write the z-buffer are the most common cause of this symptom.

To work around this issue, we recommend that you add "zMode10" to the particle system's User-Defined Properties. It is also possible that you will need help from a programmer, who will be able to help make the particle in the scene 'Always draw'. This modification will ensure that the particle system will not clip out of view at unexpected angles.

Q: Why do some animations look different when I view them in both Gamebryo and Max?

A: There are subtle differences between how Max shows time and how Gamebryo shows time.

Through Max's interface, you see time based on frames. On the other hand, Gamebryo displays time continuously, interpolating between frames. That way, if your getting 20 fps or 60 fps, you still see the same result based on the global time, not your frame rate.

This problem arises most often when you view animations in Max in which movement takes a single frame. Max only renders "Frames" at whatever frame rate you specify. Gamebryo will render as fast as it can, sampling the animation curves at the current application time. Because Gamebryo renders much faster, you see the animation as it actually is.

Q: Why aren't all of my animations exporting?

A: In some cases, Gamebryo must sample the animations because the keyframes are either an unsupported type or unavailable to the exporter.

If this is true, the exporter tries to find the time frame to sample against. In some cases, as with Character Studio, the time frame is not accurately reported. In this case, we use the presence of note track keys to indicate the start and stop times for sampling.

Q: Sometimes my animations are bigger in memory than I want. Why is that?

A: In some cases, Gamebryo must sample the animations because the keyframes are either an unsupported type or they are unavailable to the exporter.

For more information, see Animating Position, Rotation, And Scale in 3ds Max.

Q: Why aren't my instanced objects being cloned?

A: This will only occur when unique non-uniform scales have been applied to the object and when the object is skinned.

In both cases, a modifier has been applied that makes the object unique. We support uniform scale, movement and rotation of instances. When a non-uniform scale is applied, the vertex relationships are modified and the object must become unique.

Q: I am using the map type other than Bitmap. Why isn't it exporting?

A: This is probably happening because you are attempting to use an usupported format.

Gamebryo supports BMP, TGA, PNG, DDS, SGI, HDR, and internally stored textures in a .NIF. PICT, TIFF, JPG, and so on are not supported. If you need support for one of these formats, contact Emergent Technical Support and we'll see what we can do!

Q: How do I get transparent textures in Gamebryo?

A: Gamebryo uses the alpha of your Base texture for Transparency.

If you wish to use textures with alpha, use one of the Gamebryo supported formats like a 32-bit TGA format or the DirectX .DDS format that store 8-bit information into the alpha channel of the image. If the image has alpha, the Gamebryo Shader will automatically use it for transparency unless otherwise specified in the Shader parameters.

If you want to have alpha on a per material basis, simply adjust the Material Alpha property in the Material rollout. (Black is transparent and white is opaque.)

Q: I am trying to do Render to Texture lightmaps and the texture is coming out black. What is wrong?

A: Make sure that the ambient color for your render to texture is correct.

You need to make sure that the render to texture color is not black. Make sure that there are lights affecting your object.

Q: I export a NIF file with platform-specific textures for Xbox 360 or Playstation3, but the texture is a colorful checkerboard when I look at the NIF in Asset Viewer or the other tools. What is happening?

A: There are a number of possible causes for images failing to render properly.

If platform-specific textures are used, then assuming that the specific platform can support the pixel format in question, the renderer will load the associated persistent source texture renderer data directly into a renderer-specific resource at application load time. This mechanism bypasses the application-level NiPixelData object and all NiImageConverter-based texture conversion at load time entirely. The primary motivation for such an approach is improved memory usage and performance at load time.

A caveat to this approach is that the texture is configured specifically for a target platform (or graphics card), and may not be loadable for use with alternative platforms or graphics cards, or with tools on the PC. If a texture is configured specifically for one target platform or graphics card, but an attempt is made to load it on a platform or graphics card that can't support it, error replacement textures are used to indicate load-time errors.

If application requirements dictate fully general support of textures within a NIF file on multiple platforms or graphics cards, then NiPixelData-based textures may be used, because they enable on-the-fly texture format conversions at load time. To specify NiPixelData-based textures for export from the art packages, you may select one of four export options: (1) "generic" platform type, but include textures in main NIF, (2) save textures as separate NIFs, (3) textures as original images, or (4) textures as original images VRAM.

However, the best recommendation may be to work with generic textures with all the tools throughout a development cycle. Then, prior to final release, textures may be re-exported specifically for various target platforms. This approach offers the most flexibility during the majority of the development cycle, yet also offers the best performance for the final application version.

See Texture Streaming Options for Internal Textures or the description of NiPersistentSrcTextureRendererData in the Main Class Reference for more details.

Q: All of my viewer options are grayed out, why is this?

A: Your system might not have all the necessary scripts installed.

Make sure that you have a 3dsmax\plugins\maximmerse\scripts folder and that it is not empty. Look for scripts like "Default DX Viewer.script". If they do not exist, then re-install.

Q: I am getting errors loading script files.

A: This could be caused by a script or configuration error.

Make sure that you have scripts stored in the 3dsMax\Plugins\MaxImmerse\Scripts directory. If you are continuing to have errors, delete the script file with the name mentioned in the error message. If you still have errors, delete your mimmerse.ini file under 3dsMax\Plugcfg\.

Q: I am getting error messages when parsing shaders.

A: Make sure that the NSF file properties are correct.

Make sure that the NSF files are marked with read and write properties.

Q: Why can't I launch Max with the Gamebryo Max plug-in? It fails to load the plug-in.

A: This is probably caused by a license or configuration problem.

The Gamebryo plug-in requires a valid license code to work. Additionally, it depends on external DLLs that are installed with the installer. Visit Emergent's Online Support Portal for more information.
