Sort Adjust Node.max

The sort adjust node.max sample provides an example of using an NiSortAdjustNode to override the default alpha sorting behavior on an object.

There are two copies of the object in the Max file. The first one is under the hierarchy named "Regular_Sorting". When previewed, the geometry will change sorting order as you orbit the scene. The second object is under the hierarchy named "SortAdjust_Sorting". This hierarchy takes advantage of the SORTING_OFF capability of NiSortAdjustNodes. The alpha'd objects are placed in the order in which they are to be rendered in the hierarchy. This ordering does not change when the view angle changes. Note that this can still look odd when viewing from below, for instance, but it does eliminate the popping and looks correct from the majority of likely positions of a game camera.
