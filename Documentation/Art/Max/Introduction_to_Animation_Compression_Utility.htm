Introduction to the Animation Compression

The Gamebryo Max Plugin officially supports a subset of the animation controllers in Max. When one of the supported controllers is not used, the plug-in will sample the scene and generate its own keys that match Max. Sampling can introduce a large number of keys into the data set, which means increased memory usage.

The Gamebryo Animation Compression system was designed to help minimize the memory footprint of animations by fitting the keys to a cubic B-Spline. Fitting is a lossy operation and it is often useful for the artist to tweak the output values. This document details the user interface for adjusting the compression settings in Max.

Note that the compression will only occur during the sequence extraction phase using the Controller Extractor Plugin. This means that only animations specified with the Animation Manager can be compressed.

Also note that compression will only occur if the compressed spline will be smaller than the original keys. This guarantees the smallest possible footprint.
