Systems

Gamebryo supports the following particle systems in whole or in part:

  • Spray
  • Super-Spray
  • Snow
  • Blizzard
  • PArray
  • PCloud

Please see the charts below for a full supported feature analysis.

Gamebryo Export Params

In order to properly export out particle systems, Gamebryo adds a rollout of custom attributes at the bottom of the particle system parameters.

Simulation Options:

Loop: This option allows the user to specify whether this is a one-shot particle emission set or loops.

Reset On Loop: This option determines whether the particle system clears itself of all particles upon loop

System Start: This is the frame the particle system starts its simulation.

System End: This is the frame the particle system ends its simulation.

Honor Viewport Counts: This option determines whether the "Basic Parameters/Percentage of Particles" spinner will affect the final number of particles. This is useful if you want to have the particle system look exactly like it does in the max viewport.

Use Local Space PSystem: - This option determines whether or not the particles move relative to their parent node. This parent node is defined by the "PSystem Parent Node" UI item below.

PSystem Parent Node: - This option allows the user to select where the particle system will be placed in the exported scene graph. The default is to place the particle system at the root of the scene. The R button next to this pick button will reset the selected root to be the default scene root.

Maximum Active Particles: This is the maximum number of particles that this system will be required to display on the screen at one time. This determines the amount of memory allocated for this system when exported and will affect the size of the final .nif file. The value needs to include any particles spawned during the lifetime of the system.

Re-estimate Maximum: This button attempts to calculate a rough best guess of the number of particles this system can support. It may need to be tweaked if the particle system doesn't seem to emit properly. If rate is being animated, this value will change according to what the current rate is in the time slider.

Particle Rotation Options:

Speed: This option allows the user to adjust the speed of rotation. The units for this variable are in degrees per second.

Variation: This option allows the user to adjust the randomness of rotation speed. The units for this variable are in degrees per second.

Bi-Directional Rotation This option allows the user to specify whether or not the particles are allowed to rotate both clockwise and counter-clockwise or only clockwise.

Bounding Volume Options:

Bounds Update Type: This option allows the user to specify whether the bounding volume of this particle system is static (pre-computed) or dynamically updated as the particle system is simulating. Note that static bounding volumes should be considered local to the particle system object, even if the system is set to use world-space particles.

Update Every X Frames: This option determines how often the bounds are recomputed when the bounds are dynamic.

Static Bound Position/Radius: This option allows the user to set the bounding volume position and radius in local space for the static bounding volume.

Use Current Frame Bounds: This option takes a snapshot of the current frame's bounding volume in Max and uses it to compute the position and radius.

Gamebryo Extra Params

Note that the Gamebryo Extra Params only apply to sprite (not mesh) particles.

Alignment

Normal Align: Use this to determine what axis each particle's normal vector will face. By selecting 'Fixed', you can specify an axis in the particle system's local space.

Up Align: Use this to determine what axis each particle's upward vector will face.

Scale

Scale Factor: This value controls the rate at which the particle stretches in the U or V directions as a particle's speed increases. A value of 0.0 indicates not to change the particle size in that direction.

Scale Rest: The U or V multiplier when the particle has a speed of zero.

Scale Limit: The maximum U or V multiplier that will be applied to a particle. This limit the maximum scale that will be applied as speed approaches infinity.

When velocity scaling is active, the scale multiplier is determined by the following equation:

Scale = (Factor x Velocity x Limit + Rest) / (Factor x Velocity + 1)

UV Animation

In order for UV animation to work properly, create a particle texture that is divided into evenly spaced cells that go all of the way to the edge of the texture.

Frames Across/Down: Describe how many cells the source texture is divided into.

Initial Frame: What frame to start the animation on.

Initial Frame Variance: The range of frames from the initial frame that each new particle will start on. The starting frame is evenly distributed over the range [Initial Frame +/- Initial Frame Variance].

Frames/Life: The total number of frames of animation the particle will cycle through over the course of its lifespan.

FPL Variance: The range of frames from the Frames/Life parameter to use for the final frame count over lifespan. The total number of frames over a particles life is evenly distributed over the range [Frames/Life +/- Variance].

Space Warps

Gamebryo supports many of the forces and space warps available in Max. The complete list of supported space warps is as follows:

  • Gravity
  • Wind
  • SDeflector
  • Deflector
  • PBomb
  • Drag

Please see the charts below for a full supported feature analysis. All objects can have their transforms manipulated, however, do not use scale when animating transforms. This can have an unpredictable effect.

Particle Systems & Materials

Particle Systems require special care when using materials. In order to use Particle Age materials and Particle Opacity you will need to turn on Vertex Colors in the Gamebryo Shader.

Since particles are aligned quads, the default Gamebryo Shader material configuration is probably not appropriate for rendering. This is because it doesn't make sense to use diffuse lighting with particles since their normals change depending on camera position.

In order to make particles independent of scene lighting, the following material settings are recommended:

  • Diffuse Color: Black
  • Emissive Color: White
  • Vertex Colors: On
  • Src Mode: EMISSIVE
  • Light Mode: E

Particle System & Space Warps Supported Features In Depth

The following chart shows supported parameters for each particle system and notes about achieving visual fidelity with Max and Gamebryo particle systems. Most particle system exports assume a frame rate of 30.

Particle System Charts

Blizzard Rollouts:

Rollout Supported Options
Basic Parameters
  • Width & Length are supported but cannot be animated
  • Percentage of Particles is obeyed when the custom attribute "Honor Viewport Counts" is checked
Particle Generation
  • Use Rate & Use Total are both supported. Rate can be animated
  • Speed, Variation, Tumble, & Tumble Rate are supported but only speed can be animated. Tumble and Tumble Rate are specifically only supported for instanced geometry.
  • Emit Start & Stop, Life, and Variation are honored. Use the custom attributes "Loop Start" & "Loop End" to set the loop timing. Display Until and Subframe sampling are NOT supported. The Life can be animated.
  • Particle Size, Size Variation, Grow For, and Fade For are supported. The Size can be animated.
Particle Type
  • Standard & Instanced particles are supported
  • Facing particles are the only supported type
  • Instancing Parameters: Object selection is supported and subtree selection is always on. Animation Offset keying is NOT supported. Material Mapping only comes from the Instanced Geometry.
Rotation And Collision
  • Phase, and Phase Variation are supported.
  • Instead of using Spin Time and Spin Time Variation, use the Gamebryo Export Params' rotation Speed and Variation.
  • Random and User Defined Axes are supported.
  • Interparticle collisions are NOT supported.
Object Motion Inheritance
  • Object motion inheritance is NOT supported. Please make sure the influence is set to 0%: to achieve visual fidelity.
Particle Spawn
  • None, Spawn on Collision, and Spawn on Death are supported. Die after collision and Spawn Trails are not supported.
  • Direction chaos is supported.
  • Speed chaos is supported as are the Slow & Fast switches. Both Inherit Parent Velocity and Use Fixed Value are not supported.
  • Scale chaos and Lifespan Value Queue are NOT supported.
  • Object mutation queue is supported.
Load/Save Presets
  • This rollout is supported as long as the parameters it loads are supported.

Space Warps

Gravity Rollouts:

Rollout Supported Options
Basic Parameters
  • Strength, Planar, & Spherical are supported but only strength can be animated.

Wind Rollouts:

Rollout Supported Options
Basic Parameters
  • Strength, Planar, & Spherical are supported but only strength can be animated.
  • Turbulence and Frequency are supported although they may deviate from Max

Drag Rollouts:

Rollout Supported Options
Basic Parameters
  • Time on and off are supported
  • Only Linear damping is supported
  • Unlimited Range, All axes, Specified Ranges & Falloffs are supported but cannot be animated

PBomb Rollouts:

Rollout Supported Options
Basic Parameters
  • Spherical,Cylindrical,and Planar are supported
  • Strength, Decay, Start Time, and Duration are supported

Deflector Rollouts:

Rollout Supported Options
Basic Parameters
  • Bounce, Width, & Length are supported but cannot be animated.

SDeflector Rollouts:

Rollout Supported Options
Basic Parameters
  • Bounce & Diameter are supported but cannot be animated.
