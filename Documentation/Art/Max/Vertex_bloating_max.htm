Vertex bloat.max

The vertex bloat.max sample provides two examples of how smoothing groups and UVW mapping affect vertex count and performance.

3ds Max can have more than one normal and UV (within a single UV channel) per vertex, whereas most real-time engines cannot. To resolve this incompatibility, the Gamebryo 3ds Max Plug-in will add additional vertices so that the NiMesh that is created has only one normal and one UV per vertex. The NiMesh will look the same as the Max mesh, but it will have more vertices. This vertex bloat will push the transform-rate, but it is seldom a big problem. Using a lot of smoothing groups or complicated UV Mappings will make the problem worse, but some vertex bloat is unavoidable.

Having things smooth is faster than having things flat. This sample is also a good example of this phenomenon. A 120-triangle sphere with 1 smoothing group has 62 vertices; a flat or un-shaded version has 358 vertices.
