Scripting the Art Pipeline in 3ds Max

It is often useful to customize your export out of Max or Maya. Perhaps you want to convert your level into portalled geometry. Perhaps you need to create custom objects. Perhaps you want to save into a different file format. All of these options are possible by taking advantage of the Gamebryo Plug-in Framework.

Information on how to take advantage of the Tool Plug-in Framework can be found in the Tool Plug-in Framework section. The rest of the information in this section applies specifically to how to use the Tool Plug-ins within 3ds Max.

Script Types

Scripts are divided into four separate categories to allow reusability. The Gamebryo Max Plug-in's Export Options dialog allows you to choose a process and export script to be executed each time you export the scene. When previewing the scene, only the process script will get executed. Instead of executing the export script, a viewer script will be used depending upon what type of preview is being done.

  • Import Scripts are composed entirely of process plug-ins. These plug-ins should not be used with 3ds Max.
  • Process Scripts are composed entirely of process plug-ins. These plug-ins are always run whenever you export or view the file.
  • Viewer Scripts are composed of one or more viewer plug-ins. Additional process scripts for the viewer can be specified here but are not easily edited in the user-interface. These scripts directly correspond with the radio buttons in the Viewer Selection panel in the Gamebryo Utility. New viewer plug-ins can be used through the Custom Viewer radio button. These scripts are only run when the View button is pressed in the Gamebryo Utility.
  • Export Scripts are composed of one or more export scripts. These kinds of scripts are run only when you export out a file.

Script Directories

The Gamebryo Max plug-in loads the script template files from:

\3dsmax\plugins\MaxImmerse\scripts

Plug-in Directories

The Gamebryo Max plug-in loads all of the plug-ins from the directory %EMERGENT_PATH%\SDK\Win32\DLL\ToolPlugins. By default, the EMERGENT_PATH environment variable is set by the installer.

Provided Default Scripts

The following scripts are provided by default in the Gamebryo Max Plug-in installation.

Process Scripts:

  • Default Process Script This script runs the Create ABV, Optimization, and Stripification plug-ins with their default parameters.
  • Scene Graph Optimizations Only This script just runs the SceneGraph Optimization plug-in with its default settings.

Viewer Scripts:

  • Default DX Viewer Script This script is used by the DX Viewer radio button in the Gamebryo Utility→Viewer Selection rollout.
  • Default KFM Viewer Script - This script is used by the KFM Viewer radio button in the Gamebryo Utility→Viewer Selection rollout.
  • Default SceneImmerse Viewer Script - This script is used by the Scene Viewer radio button in the Gamebryo Utility→Viewer Selection rollout.

Export Scripts:

  • Default Export Script This script exports out a single NIF file.
  • Multiple KF Export With NIF Script This script exports out a single NIF file and multiple KF files depending on what animations the Controller Extractor plug-in finds.
  • Multiple KF Export Without NIF Script This script exports multiple KF files depending on what animations the Controller Extractor plug-in finds but does not export out the NIF file.
  • Single KF Export With NIF Script This script exports out a single NIF file and a single composite KF file of the animations that the Controller Extractor plug-in finds.
  • Single KF Export Without NIF Script This script exports out a single composite KF file of the animations that the Controller Extractor plug-in finds, but does not export out the NIF file.
