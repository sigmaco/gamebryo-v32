PhysX Constraints in Max

The PhysX Max Plug-in adds a Constraint object to 3ds Max, used to create, edit and export a completely general constraint between two rigid bodies. Use of the constraint and its parameters are documented by the PhysX Max Plug-in documentation, although some additional information is listed here.

Working with constraints

When preparing a constraint for use with Gamebryo, keep the following things in mind:

  • At least the child rigid body should be specified in order to export correctly. If only one body is specified, the constraint is anchored to the world coordinate system - use this rather than connecting to a static actor.
  • The in-Max PhysX preview can give behavior that differs from Gamebryo export, and the behavior appears somewhat inconsistent as a constraint is edited. When configuring constraints, it is recommended that you export the constraint and preview in a Gamebryo application regularly.

Axis alignment considerations

The PhysX Max Plug-in associates the following axes with the swing and twist parameters:

  • Swing1 is rotation about the constraint's local Z axis.
  • Swing2 is rotation about the constraint's local Y axis.
  • Twist is rotation about the constraint's X axis.

It may be important to note that this is different from how the swing and twist parameters are interpreted at runtime. The PhysX SDK interprets Swing1 as the local Y axis and Swing2 as the local Z axis. The Gamebryo Max Plug-in automatically swaps these axes on export so that you get consistent behavior, but it may be relevant to recognize this distinction when communicating with someone using the PhysX Maya Plug-in or the PhysX SDK.
