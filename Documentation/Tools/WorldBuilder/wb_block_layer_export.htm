Exporting blocks/layers

In order for a block or layer to be used by the runtime, it needs to be exported. This export process can be customized by a studio to meet their needs.

Studios can use the Emergent.Toolbench.WorldBuilder.Exporter extension point to customize the export by injecting their own custom export class. Please see the documentation for this extension point to take advantage of this feature.

The default exporter, "XmlBlockExporter" will convert a block file into an XML x-block file. Any layers that have their "Generates own xblock" flag set will also generate x-block files. It is up to the application to choose when and how to load the exported x-blocks. In this way, level designers can produce larger levels and still visualize how the segments of the level interconnect.

Since the runtime can load multiple x-blocks simultaneously and in the background, there are many ways to take advantage of block and layer exporting. Some examples of how one might break up exported data:

  • Rooms are stored in separate exported layers. Doorways between layers have custom triggers to load the exported layer files.
  • Common data to the level can be stored on one or more exported layers.
  • Data needed for a server version of the game can be one one or more layers, while data only needed for the client is on other layers.
  • Data needed for a specific platform can be stored on one or more layers.

Note

An x-block can be exported only if the block or layer it represents is fully loaded in Toolbench. If a layer is currently unloaded, then the x-block that contains it will not be exported and a warning will appear in the Output view.

However, when running from the command-line, local settings (that among other things determine if a layer is loaded or not) are ignored. This means exporting from the command-line will always export everything, regardless of which layers were loaded and unloaded the last time you saved your Block in Toolbench.

Refer to Loading and Unloading Layers and Toolbench Command-Line for more information.
