Exporting Entities to DCC Tools

It is sometimes valuable to take the data that exists in World Builder and make it available in DCC tools (Max, Maya, XSI). For example, a level designer may rough out their environment in World Builder, but an environment artist will need to finish it. For this situation, you can export data to FBX, which is commonly read by most DCC tools.

This tool can be accessed from the World Builder menu with the menu item "Export to Autodesk FBX". This option may take some time depending on how many entities exist in your scene and what options were chosen at export time.

Limitations of FBX export

The following data is exported to FBX:

  • Entity transforms as a top-level child of the scene root
  • NiNode transforms in the scene graph
  • Vertex positions and normals for unskinned geometry
  • Entity visibility

Any other data is explicitly not supported. This includes:

  • Vertex UV coordinates and colors
  • Packed or otherwise optimized vertex data
  • Non-triangle based geometry
  • Skinned or instanced meshes
  • Lights, cameras, or other NiAVObject derivates
  • Mesh shading or material information

Options Dialog

  • Which Types of Entities to Export - Change these options to adjust the kinds of entities that will be exported.
    • Export Whitebox Geometry - These are entities that mix-in the "Whitebox" model.
    • Export Meshes Marked With "IsStatic" True - These are entities that mix-in the "Mesh" model and have their "IsStatic" property set to true.
    • Export Any Entity With An Associated Scene Graph - These are entities that have some kind of scene graph associated with the entity instance. This incorporates all "Mesh", "Whitebox", "Terrain", "Actor" and other mix-ins. All of the previously mentioned limitations apply to this exported content, so it may not look 100% accurate.
  • Entity Filter Options - Change these options to adjust the filters applied when exporting entity types.
    • Only Export Selected Entities - Only entities that are currently selected in the UI.
    • Only Export Currently Visible Entities - Only entities that have visible layers and their tool-time visibility set to true.
    • Only Export Entities From Active Layer And Child Layers - Only entities that exist on the active layer or its child layers.

Tips for bringing content cleanly in and out of FBX

There are many options in the DCC tools when importing FBX files. Each importer is different, but the following options should be preserved:

  • Do not adjust the units.
  • Preserve the up-axis set in the FBX file. For Maya, this means that you need to set Maya to Z-up before importing.

If you follow the above guidelines, when you export the adjusted geometry back to NIF, it will come in at the same scale as when the data was exported.
