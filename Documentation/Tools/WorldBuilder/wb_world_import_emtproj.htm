Importing Entity Modeling Tool Projects

Before adding entities to a world, the World Builder block file must reference one or more Entity Modeling Tool projects that will provide the entity models that define the entities. When creating a new game solution, a content wizard can automatically import EMT projects and establish the necessary references. At other times, these two steps must be performed explicitly.

Toolbench allows for an existing EMT project to be imported into an opened game solution. This feature is useful when models need to be shared between several game solutions and those game solutions fall under a common asset web.

To import an Entity Modeling Tool project

  1. Right-click the game solution root item in the Game Solution view and choose Add > Existing Content from the context menu.
  2. When the standard file selection dialog appears, filter on Entity Modeling Tool projects by selecting the "emtproj" extension type filter.
  3. Finally, select (double-click) the desired Entity Modeling Tool project file.
  4. If successful, the EMT project will have been imported into the current solution.

Note

One important caveat is that the selected project must exist below the asset web root. If the selected project does not exist under the asset web root, the project will not successfully import into the game solution. To import a project from outside the current asset web, the desired project (and all its dependent assets) must first be copied into the asset web. For packaged libraries, this can be done with the Integration Wizard. For other projects, this is a manual process that must be performed outside of the tool. When copying projects and dependent assets from one asset web into another, it may be necessary to preserve the relative paths with respect to the original asset web root.

Once an EMT project is in the game solution, it must still be referenced by .block files in order to contribute to the palette of entity models available in World Builder. An EMT project may be referenced directly or indirectly via another project that already has a reference to it.

To reference an Entity Modeling Tool project

  1. Open the block file that will reference the EMT project.
  2. Return to the Game Solution view and expand the block file node to reveal the References folder.
  3. Right-click the References folder and choose Add Reference from the context menu.
  4. When the standard file selection dialog appears, select the EMT project to be referenced. This will establish the reference and make the project's content available to the block file.
