Troubleshooting communication between the Script Debugger and your game

Communication between the Toolbench-based Script Debugger and game-side Lua Behavior Debugger is logged to the Toolbench Output View. You can use the "Game" filter to view important notifications. You can use the "Debug" filter to view a complete record of command and response messages. Timestamped request and response messages are listed in this log. You can use these messages to troubleshoot connection problems.

Be sure that your game supports the LogService, which enables remote logging. This service is automatically included as part of the RapidIterationService.
