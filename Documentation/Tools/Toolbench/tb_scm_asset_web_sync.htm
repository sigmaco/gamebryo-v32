Syncing to Your Asset Web from Revision Control

During development, it is necessary to synchronize your local asset web with changes made by others. Since Toolbench does not detect changes to files made by external applications, it is necessary to close and reopen the game solution to force all the assets to be reloaded.

Follow these steps when updating your asset web from revision control:

Perforce

  1. In Toolbench, close the game solution (File -> Close All).
  2. Switch to P4V (or P4Win) client and get latest revision from depot.
  3. Resolve any integration conflicts.
  4. In Toolbench, reopen the game solution (File -> Recent Files -> [games solution].gsl).

Subversion

  1. In Toolbench, close the game solution (File -> Close All).
  2. Switch to svn command line (or TortoiseSVN) and update local asset web
  3. Resolve any conflicts.
  4. In Toolbench, reopen the game solution (File -> Recent Files -> [games solution].gsl).
