Setting Up Revision Control for a Game Solution

Revision control settings are stored per game solution and are not global to Toolbench. (In other words, you have to specify revision control settings for each game solution.) When you enable revision control in Toolbench, you specify the revision control system that will be used (Perforce or Subversion). You also specify additional configuration settings like your Username and password.

If no revision control settings are defined for a game solution, or if settings are defined but no connection to the revision control system server is available, a red status icon displays in the lower left corner of Toolbench.

Toolbench Status Bar (lower left corner of window)

To set up revision control for a game solution

  1. Open an existing (or create a new) game solution. The asset web for the solution must be located within a revision controlled directory on the local file system. For example: c:\depot\projects\MyGame (where c:\depot is the Perforce root) or c:\projects\MyGame (where c:\projects is a directory checked-out from Subversion).
  2. From the Toolbench menu bar, click Utilities > Settings.
  3. In the Settings dialog, select the game solution tab (for example, MyGame.gsl).
  4. In the Settings tree on the left-hand side of the dialog, select the Revision Control node.
  5. In the Provider drop-down, select either Perforce or Subversion. (Default: none)
  6. Enter additional revision control settings in the fields and click the Test Connection button. If you have entered valid information, a message will confirm that a successful connection has been made.
    • Perforce—If you are using Perforce and you leave the Username, Password, Host, and Workspace settings blank, the current ticket created by P4Win or P4V will be used to access the Perforce server. Note that Host is the machine name and port of the Perforce server. For example, p4server:1666.
    • Subversion—Toolbench does not submit changes to the Subversion server at this time. The Username, Password, and Host fields can be left blank. Note that Host is the repository that is being used for revision control. For example, https://svn.example.com/projects.
  7. If you are using Perforce, select the Perforce node in the Settings tree and enter a valid Perforce workspace (clientspec) name.
  8. Click OK and confirm that the Revision Control status icon changes from red to green in the lower left corner of the Toolbench window. Also, confirm that (not in revision control) icons are displayed in the Game Solution View.
