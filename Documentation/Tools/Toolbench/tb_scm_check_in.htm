Checking in Content to Revision Control

All content check-ins are committed outside of Toolbench using a supported revision control client (for example, P4V or P4Win client for Perforce, or TortoiseSVN for Subversion.) Because Toolbench does not detect files that change underneath it, you might have to close and reopen the game solution to ensure that Toolbench is updated with files that have been pulled from source control.

If you are using Perforce, your revision control changes will be grouped in the changelist that you specified when you set up revision control for your game solution. For more information about setting up the changelist, see Setting Up Revision Control for a Game Solution.
