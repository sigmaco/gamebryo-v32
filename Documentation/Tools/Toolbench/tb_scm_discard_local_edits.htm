Discarding Local Edits to Revision Controlled Content

To discard changes made in Toolbench, use the revision control system's revert functionality. Since Toolbench does not detect changes to files made by external applications, it is necessary to close and reopen the game solution to ensure reverted files are loaded.

Follow these steps when updating your asset web from revision control:

  1. In Toolbench, close the game solution (File -> Close All).
  2. In the revision control client (for example: P4V or TortoiseSVN), revert (or undo) the local changes.
  3. In Toolbench, reopen the game solution (File -> Recent Files -> [games solution].gsl).

Note

If Toolbench fails to properly load the game solution and reports errors, it may be necessary to reconcile your local file system with the revision control server or simply force synchronize to the latest asset web in revision control..
