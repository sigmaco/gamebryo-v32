Reconciling Offline Work with Revision Control

Perforce

If a Perforce client (like Toolbench) is disconnected from the Perforce server, revision control operations cannot be recorded in a changelist. Thus, when disconnected from the server, changes made to the local file system by Toolbench are unknown to Perforce.

Once the connection to the server has been reestablished, it is the user's responsibility to synchronize the changes between the local file system and Perforce. Fortunately, P4V provides a tool to help with this situation. Follow these steps to reconcile local files with the Perforce server:

  1. In Toolbench, close the game solution (File > Close All).
  2. Follow instructions in P4V help (see "Reconciling Offline Work" or http://kb.perforce.com/UserTasks/WorkingDisconnected.
  3. In Toolbench, reopen the game solution (File > Recent Files > [games solution].gsl).

Subversion

By default, Subversion works in an offline mode. Synchronizing with the server basically means uploading changes and fetching the latest versions of files in the asset web. Follow these steps to reconcile local files with the Subversion server:

  1. In Toolbench, close the game solution (File > Close All).
  2. Use Subversion client to commit changes to server.
  3. Use Subversion client to update changes from server to local repository.
  4. In Toolbench, reopen the game solution (File > Recent Files > [games solution].gsl).
