Revision Control Integration

Toolbench integrates with Software Configuration Management (SCM) systems to enable you to place your game solution content under revision control. In both the Toolbench user interface and in the documentation, this functionality is referred to as "Revision Control".

The Toolbench revision control integration automatically performs add, edit, delete, and rename actions on game solution content. For example, if you have configured a game solution to use revision control, and you add an entity model to your solution, you will not need to add the new entity model to the revision control tool in a separate, explicit operation. Instead, when you add the new model, it will be added automatically to the revision control changelist. Similarly, when you edit existing content, the corresponding files will be checked-out automatically from your revision control system. The revision control client (for example, P4V or TortoiseSVN) is required to submit and synchronize game solution content with the revision control server.

Game solution content that is under revision control will always be saved to the local file system, even if the corresponding revision control actions fail. For example, if you add a new block, a new block file is saved to the local file system before the add to revision control action is even attempted. This ensures that your data is preserved if the revision control server is offline or configured incorrectly. For information on synchronizing files with revision control after a failed revision control action, see Reconciling Offline Work with Revision Control.

By default, Toolbench includes integrations with both Perforce and Subversion. In addition, it uses a pluggable interface that allows other revision control providers to be added. For more information about extending Toolbench's out-of-box revision control capabilities, see Revision Control Wrapper Overview.
