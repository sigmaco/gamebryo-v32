EEDeploy Command-line Usage

The EEDeploy tool allows for script-driven deployment of game assets. It can selectively filter files or subject them to transformations via any external tool.

Usage

EEDeploy scriptFile [verbose] [key=value [key2=value2]...]

Required Arguments

scriptFile

Provides the path to the script file that drives deployment. For more information about deployment script format, see EEDeploy Script File Format.

Optional arguments

verbose

Enables verbose mode, where additional operational information is displayed.

key[=value]

Defines a variable with an optional value. There may be zero or more of these on the command line. For more information about variable, see the Notes below.

Notes

Variables—EEDeploy supports the notion of variables so that conditional logic can be executed based on the value or existence of a variable. You can use the define command to define variables in the script. Alternately, you can specify them on the command-line using the format variableName=value or just variableName (in which case a default value of "" is inferred).

Some example uses of variables include:

  • Specifying the target platform (for example, platform=ps3 or platform=win32)
  • Specifying the target configuration (for example, config=shipping)
  • Meta-processing like verbose mode (see below)

All commands in the script are processed in a linear order, including defines and undefines. This means that a variable may be evaluated to different values for different commands, depending on where it is in relation to define and undefine commands. Variables that are specified on the command-line are treated as if they were defined just before the script started executing.

Variables are not case sensitive. Defines will internally go to lowercase, and comparisons are done in a case-insensitive manner.

Verbose mode—You can specify the verbose variable on the command line when invoking EEDeploy. Alternately, you can specify the variable by using a define command in the script. When the variable is defined, EEDeploy will print out additional information about its inner workings to stdout. It will also echo the contents of any description commands in the script.
