Mixing characters and sequences in the Animation Tool

You can use the Animation Tool to mix and match sequences onto different character models. This technique enables you to create and model different versions of a character without having to animate each version independently.

To mix characters and sequences

  1. Launch the Animation Tool and load the following example:

    %EMERGENT_PATH%\Samples\Models\AnimationToolExamples\Tutorials\Redcoat\Redcoat.kfm

  2. Preview the different sequences in the project.
  3. Load the following file:

    %EMERGENT_PATH%\Samples\Models\AnimationToolExamples\Tutorials\Blacksmith\Blacksmith.kfm

  4. Preview the different sequences in the project. These two characters share very similar hierarchies, which means that we can mix and match sequences and models.
  5. Create a new project file by choosing File > New.
  6. In the Project View, select Character Model and click the Select File button in the Edit View.
  7. Select the following file:

    %EMERGENT_PATH%\Samples\Models\AnimationToolExamples\Tutorials\Redcoat\Redcoat.nif

  8. Set the character root to be Bip01.
  9. Add all of the KF files that appear in the following folder:

    %EMERGENT_PATH%\Samples\Models\AnimationToolExamples\Tutorials\Redcoat

  10. Now add all of the KF files in the following folder:

    %EMERGENT_PATH%\Samples\Models\AnimationToolExamples\Tutorials\Blacksmith

    Note that the project now contains all of the sequences for both the Redcoat and Blacksmith characters. Preview the different sequences.

  11. In the Project View, select Character Model again and click the Select File button in the Edit View.
  12. Select the following file:

    %EMERGENT_PATH%\Samples\Models\AnimationToolExamples\Tutorials\Blacksmith\Blacksmith.nif

  13. Set the character root to be Bip01.
  14. Now preview the animation sequences.

You have now successfully mixed and matched animation sequences and character models. Note that these two cases matched up very well. Other cases may not have hierarchies that match so closely. When this occurs, the tool will warn you that certain sequence evaluators did not match up correctly. For example, you can try adding the following file:

%EMERGENT_PATH%\Samples\Models\AnimationToolExamples\Soldier\Soldier_walk.kf

You will be warned that the musket and several of the coat nodes were not found. These evaluators will not animate, but the evaluators that did match up will still animate properly.
