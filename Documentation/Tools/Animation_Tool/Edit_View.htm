Edit View

You can use the Edit View in the Animation Tool to examine and modify various editable properties of objects that you select within the Project View. The contents of the Edit View are dependent on the context of what is currently selected.

To display this view, select View > Edit View or click the Edit View button on the main toolbar.

The Edit View is designed to operate best with a vertical layout. It uses a roll-out mechanism for displaying various components of the currently selected item. The following screenshot of the Edit View for a sequence group illustrates a few of the view's typical components.

Edit View for sequence group

1. Roll-out Buttons—These buttons enable you to hide or un-hide sections. The screenshot shows one closed section followed by two opened sections.

2. Splitter Control—Occasionally, a section will have a splitter-control that enables you to expand the real estate for a particular region. This often occurs after a list of items. Resizing the list can make using a scroll bar for that list unnecessary.

3. Editable properties are available with various forms of input. In this example, various properties such as priority, weight, and so on are available. Additionally, you can set keystroke settings to activate each sequence in the group.

You can use the Edit View to perform almost any input modifications for a particular item. Though the screenshot above shows the available properties for a sequence group in the Edit View, similar object-specific properties are available for transitions, sequences, keystrokes for Interactive Preview, and so on.

Edit View Editing Contexts

There are numerous contexts in which the Edit View may be used. A brief discussion of these contexts follow.

Character Model

The Edit View for the character model allows the user to specify the model to use by selection of a NIF file. The view also allows the user to specify the model root for the character model.

Sequence and Sequence Group Folders

When a folder is selected, the Edit View will display buttons for the creation of additional folders and also for creating sequences or sequence groups, depending on the context.

Interactive Preview

If the Interactive Preview Item is selected, the user enters a mode whereby they can activate sequences by using either keyboard or mouse input. The appropriate transitions between each sequence will occur automatically.

When the Edit View is under the Interactive Preview context, the user is presented with two roll-outs.

In the first roll-out, the user can select the "Idle Sequence" and also choose a sequence to activate from a list of the available sequences. The user can simply "check" a sequence to activate it. Sequences that cannot be transitioned to from the current sequence are shown as grayed out in the check box list. The "Idle Sequence" is the sequence that is activated when no keyboard keys are pressed.

The second roll-out contains a list of all sequences with their associated keystroke bindings. If no bindings are set, or if the user wishes to change the existing bindings, the user can simply double-click in the desired keystroke field and modify the binding keystroke (with modifiers, if desired).

Upon entering a keystroke, the input focus will move to the next keystroke field in the list so that the user can rapidly modify all settings. At any time, the user can hit the ESC or ENTER key or select elsewhere with the mouse to remove focus from the field. Note that for number pad keys, NUMLOCK affects which key is generated by the Windows OS. With NUMLOCK on, SHIFT + NUMPAD8 will generate the UP key. If NUMLOCK is off, SHIFT + NUMPAD8 will generate the SHIFT + UP keys.

Sequence IDs and Group IDs

If, in the Project View, the user selects "Sequence ID Overview" or "Sequence Group ID Overview", then the Edit View will contain a list box with the IDs in the left column and the names of the sequences (or sequence groups) on the right.

The user can modify IDs simply by double-clicking in the ID field and changing the value. When the user hits Enter or Tab, the input focus will move to the next ID field. This is a convenient way of rapidly changing ID's of sequences and sequence groups.

Sequence Groups

If a Sequence Group is selected, the user enters a mode whereby they can edit and test the parameter settings for the selected Sequence Group.

The first roll-out in the Edit View contains general information about the Sequence Group. It informs the user about the Group ID and Name of the selected Sequence Group.

The second roll-out is broken up into two sub-sections. The first section is simply a list of all the sequences in the Sequence Group with their associated weights. To the left of each sequence listed is a check-box that indicates, when checked, that the sequence is active. The user can activate and deactivate sequences by simply clicking on the check-box with the mouse or by using keys that have been assigned to those sequences. This section is also a convenient way to quickly modify the weights of all sequences in the group.

The second section of the roll-out allows the user to modify the parameters of a particular sequence in the group. When the user selects on a sequence from the list of sequences, that sequence will have its properties displayed. The properties displayed will include:

  • Synchronize To—Shows which sequence, if any, is used for synchronization. The user is free to specify that no synchronization is to take place or, if preferred, a particular sequence to synchronize to can be specified.
  • Priority
  • Weight
  • Ease-In and Ease-Out
  • Key Assignment— This is the key that can be used to activate and deactivate the sequence within the Sequence Group. This value can be changed in the "Keystroke Settings" roll-out as described below.

The third roll-out is the "Keystroke Settings" roll-out. It contains a list of all sequences with their associated keystroke bindings. If no bindings are set, or if the user wishes to change the existing bindings, the user can simply double-click in the desired keystroke field and modify the binding keystroke (with modifiers, if desired).

Upon entering a keystroke, the input focus will move to the next keystroke field in the list so that the user can rapidly modify all settings. At any time, the user can hit the ESC or ENTER key or select elsewhere with the mouse to remove focus from the field. Note that for number pad keys, NUMLOCK affects which key is generated by the Windows OS. With NUMLOCK on, SHIFT + NUMPAD8 will generate the UP key. If NUMLOCK is off, SHIFT + NUMPAD8 will generate the SHIFT + UP keys.

Transitions

There are several transition types, each with different parameters and editable properties. The following is a discussion of each of the transitions and how they relate to the Edit View.

  • Delayed Blend

    When editing delayed blend transitions, the user can edit the duration of the blend. In addition, new blend pairs may be created from existing text keys. To do this, the user simply uses the two combo boxes featured at the bottom of the dialog to choose the text keys indicating the frame in the source sequence at which the transition should be started and the frame to blend to in the destination. Then, the user just hits the "Add" button to add the new pair to the list. To remove a blend pair, the user selects the desired pair and clicks the "Delete" button.

    Note

    Using the Transitions View, it is possible to select multiple Delayed Blend type transitions at one time. When this happens, if no other transition types are selected in the mix, the user is allowed to edit all the Delayed Blend types all at once. The Blend Pairs section will only show Blend Pairs that are common to all the Delayed Blend transitions selected. Also, the user will only be allowed to add new Blend Pairs that are capable of being added to each and every Delayed Blend type selected.

    Edit View for Delayed Blend

  • Immediate Blend

    Immediate blends are like their delayed blend counterparts except they do not have any of the blend pair controls. Instead, the user can only choose a transition duration and a text key indicating the frame in the destination to blend to.

    Note

    When multiple Immediate Blend transitions are selected, only Durations may be edited.
  • Morph and Cross Fade

    Morphs (synchronized) and cross fades (unsynchronized) only allow the user to specify the duration of the transition.

    Note

    When multiple Morph or Cross Fade transitions are selected, the Duration may be edited and the result is applied to all the selected transitions.
  • Default Sync and Default NonSync

    When under the context of editing Default Sync and Default NonSync transitions, the Edit View presents two buttons: one for changing the default settings and the other for making the transition "unique". Making a default transition unique will change its type to be the actual type indicated by the default. For example, if Default NonSync transitions are set to be a cross fade, making a Default NonSync transition unique will change the transition type to Cross Fade. Only when the transitions are unique can the duration be modified for these transitions on an individual basis.

    Note

    When multiple Default Sync or Default NonSync transitions are selected, they may all be made unique.
  • Chain

    When a chain is selected, the complete chain transition will be shown in a list box under the Chain Sequence section. The user can specify the length of time for each transition in the chain (with the exception of the source and destination sequences) by simply clicking on the sequence and setting the duration in the "Selected Sequence" section. This duration is the amount of time that the animation system will wait after the previous transition is complete to trigger the next transition in the chain. The default duration is set to wait until the end of the current sequence before transitioning to the next sequence in the chain, but the duration can also be specified in seconds.

    Note

    Each transition in the chain uses the settings stored in the project to transition from the current sequence to the next sequence in the chain.

    Note

    Using the Transitions View, it is possible to select multiple Chain type transitions at one time. If no other transition types are selected other than Chains, the user will be allowed to edit all of the Chains at once.

    The Chain Sequence section will only show intermediate chains that are common (starting from the end of the chain) to all the Chains selected. If any intermediates differ, the Chain Sequence will display "<INTERMEDIATES DIFFER>" followed by the common ones. Likewise, the text "<Multiple Sources>" and "<Multiple Destinations>" will be shown if the source or destination sequences are different among the selected.

    For intermediates that are in common, if the durations for those intermediates are also in common, the duration will be shown in parenthesis next to the intermediate and the user may modify the durations in mass by selecting on that intermediate. If however, the durations are different, the text "(DURATIONS DIFFER)" will be shown instead and the user may not modify the durations in mass.

    Edit View for Chain

    The Edit View isn't completely sufficient for real-time editing of chains, so the "Edit Chain" button in the Edit View will spawn a modal dialog where further editing of the chain can be achieved. That dialog appears below:

    Edit Chain dialog

    In this dialog, the user is shown the intermediate sequences between the source and destination sequences of the chain on the right half of the dialog. When adding sequences to the chain, the user will typically want to see sequences that have a valid transition specified from the last intermediate sequence in the chain. Other times, the user may want to know what sequences have valid transitions that can bridge the immediate last sequence and the final end sequence. For these two scenarios, the user can select among the "Possible Next Sequences" radio buttons, which will filter the sequences shown on the left half of the dialog appropriately.

    When this dialog is used to edit chains in mass, the user is free to add and delete common intermediates. If the chains have intermediates that differ, this will be reflected in the "Intermediate Sequences" section by the first entry reading "<INTERMEDIATES DIFFER>". Immediately following this entry will be a list of all the common intermediates, if any.

Additional Notes on the Edit View with Multiple Selections

Whenever multiple sections are made in the Transition View, the Edit View attempts to allow the user to edit the data that the transitions have in common. The one field that all transitions have in common is "Durations" and is therefore always editable. When multiple transitions are selected and the durations for the selected are the same in every transition, then the value for "Duration in Seconds" will indicate that value. If however, some of transitions have durations of different values, the value for the field "Duration in Seconds" will be 0.0 and will be printed in red text to indicate such.
