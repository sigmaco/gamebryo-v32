Simulation Debugger Entity View

The Entity View provides detailed information about the internal state of the entity selected via the World View, including:

  • The entity model name
  • The game-generated entity ID
  • The Toolbench-generated GUID, if loaded from a file
  • A list of all entity properties

Properties shown in red have gone "stale" and may not reflect the current state of the entity. This will always be the case for Position and Rotation properties, which have special handling. They are updated periodically when entities have moved a sufficient distance, which can be tuned through Simulation Debugger settings. Other properties may go stale due to auto-throttling, which attempts to avoid flooding the network with updates for frequently changing properties. In these cases, the red properties will never update until the session is reset.

When connected to a game and in the live debugging state, the property values may be edited. These edits will be applied to the running game and any errors will be returned to the Game channel of the Output View. Note that existing collection-based property values may be changed, but items may not be added to or removed from collection properties, nor may map keys be edited.
