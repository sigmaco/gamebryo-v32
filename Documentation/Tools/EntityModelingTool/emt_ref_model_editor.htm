Entity Model Editor

The Entity Model Editor is where you define the properties and behaviors of the entity models in your project. The Entity Model Editor also displays a list of all the properties and behaviors that have been inherited from the current entity model's parent items. These inherited properties and behaviors are called "Mix-ins". You can override the values of these mix-in properties and behaviors within this editor.

Interface Element Description
General
Name The name of the entity model.
Description Additional information that further describes the entity model.
Traits Defines the specific characteristics of the entity model. You can select multiple options from this list.
Mix-ins Displays a list of parent entity models from which properties and/or behaviors are inherited. If there are several mix-ins, the order in which they appear in the list determines their priority. If there is a conflict between two mix-ins, the mix-in with the highest priority will be used. A mix-in conflict occurs when the same property is inherited from multiple mix-ins all with different property values. To re-order the mix-in list, select a mix-in and then use the context menu or simply drag the mix-in to the new location in the list.
Note: Only parent items are included in this list and not grandparents.
Properties


(Expand/Collapse)		 Displays or hides the following fields that correspond to the current property.
  • Traits— the automated management of the properties, such as whether they are "Persisted", or can be accessed on a client. You can select multiple options from this list.
  • Restriction Type— whether a restriction, in the form of a particular value or range of values, is placed on the property. Depending on which option you select, additional input fields may be displayed. For example, if you select the "Range" option, two new fields are displayed where you can enter a minimum and maximum value for the range.
  • Description— information that further describes the property.
Name The name given to the property applied to the current entity model.
Type The type of property applied to the current entity model. There are numerous types of properties, including String, Float32, and Color. When selecting a property type, you must also specify whether the property value will use a container type:
  • None— single property value exists.
  • List— list of possible property values exist.
  • Map— property values are mapped to specific keys.
Note: These three options are available at the bottom of the Type list.
Default Value The default value of the corresponding property. Since there are numerous types of properties, the default value can be in the form of a numerical value, a filename, a string, a 3D coordinate in your world, and so on.
Note: If an expand icon is displayed to the left of the Default Value field, it means that additional editing tools exist for the current property values. Click the icon to display the additional editing tools. Click the icon a second time to hide them again.
Adds a new property to the property list. After a new property is added, you can then define the name, type, and value for the new property.
Removes the selected property from the property list.
Mix-in Properties


(Expand/Collapse)		 Displays or hides the following fields that correspond to the current property.
  • Traits— the automated management of the property, such as whether it is persisted, or can be accessed by a client. Check the override box for the property to enable trait editing. You can select multiple options from this list.
  • Restriction Type— read only information on any restriction applied to this property by the mixin model that supplied it.
  • Description— read only description for this property provided by the mixin model that supplied it.
Check Box Determines whether the parent property values are inherited or overridden. Select the check box to override the property value.
Name The name given to the property inherited from the parent entity model.
Note: The properties are sorted by parent entity model.
Default Value The default value of the corresponding property. Since there are numerous types of properties, the default value can be in the form of a numerical value, a filename, a string, a 3D coordinate in your world, and so on.
Behaviors


(Expand/Collapse)		 Displays or hides the following fields that correspond to the current behavior.
  • Traits— how a behavior can be called. You can select multiple options from this list.
  • Description— information that further describes the behavior.
Name The name given to the behavior applied to the current entity model.
Target The language in which the specific behavior is defined. You can choose between the following languages:
  • C
  • Lua
  • Python
  • Built-in
  • Abstract
Opens the section of code where the currently selected behavior is defined. If the document containing the script does not yet exist, one will be created.
Adds a new behavior to the behavior list. After a new behavior is added, you can then define the name and target for the new behavior.
Removes the selected behavior from the behavior list.
Mix-in Behaviors


(Expand/Collapse)		 Displays or hides the following fields that correspond to the current behavior.
  • Traits— how a behavior can be called. Check the override box for the behavior to enable trait editing. You can select multiple options from this list.
  • Description— read only information that further describes the behavior, provided by the mixin model that supplied it.
Check Box Determines whether the parent behaviors are inherited or overridden. Select the check box to override the behavior target.
Name The name given to the behavior inherited from the parent entity model.
Target The language in which the specific behavior is defined. You can choose between the following options:
  • C
  • Lua
  • Python
  • Built-in
  • Abstract
Opens the section of code where the currently selected behavior is defined. If the document containing the script does not yet exist, one will be created.
Property Visualizers


(Expand/Collapse)		 Displays or hides the property dependencies that drive a visualizer.
Name The name given to the property visualizer applied to the current entity model.
Type The type of the property visualizer. The following types currently exist:
  • Point
  • Sphere
  • Radius
  • Cube
  • Frustum
  • Connection
  • PhysXBox
Dependencies The property dependencies that drive the property visualizer.
Adds a new property visualizer to the property visualizer list. After a new visualizer is added, you can then define the name, type, and dependencies for the new visualizer.
Removes the selected property visualizer from the property visualizer list.
