Find in Behavior Files

Sometimes it is usefult to search all of your game's Behaviors at once for an occurance of a particular text string. Toolbench offers a "Find in Files" feature that is loosely based on the one that software developers are familiar with in Visual Studio.

To search for text in Behavior scripts

  1. Run Toolbench (Start > Emergent > Gamebryo LightSpeed > Tools > Toolbench).
  2. Open your game solution (see Creating/Opening Game Solutions).
  3. In the Edit menu, choose Find in Files. Alternatively, press on Ctrl+Shift+F on your keyboard.
  4. In the Find in Files dialog, enter the appropriate parameters and click the Find All button. The Find in Files dialog will automatically close, and the Find Results view will open. More information is available below.

Find in Files Options

The following image shows the 4 sections of the Find in Files dialog:

Find in Files options

  1. Find what

    Here you can enter the text you want to search for. A history of previously searched text is available in the combo box. Options include:

    • Regular expressions: If checked, the searched text is interpreted as a Regular Expression. For example you could search for all function names starting with "On" and ending with "Killed" using the following Regular Expression: On.*Killed. Standard .Net Framework Regular Expressions are used. For details on how to construct Regular Expressions, please consult the .NET Framework Regular Expressions page on MSDN.

      Warning

      You can use "\n" in your regular expressions to match a line ending. However, if your regular expression starts with a "\n", the line that will appear in the search results may not be what you expected, because it will be the line at the beginning of the match, in this case the line before the "\n". For example, if you search for "\n123", the line that will appear in the results will not be the line that contains "123" but the line just above it.
    • Case sensitive: If checked, the search will be case sensitive, meaning that uppercase and lowercase letters will be matched as-is. When unchecked, the search is said to be case insensitive, so uppercase and lowercase letters are considered to be the same.
  2. Look in

    Here you can specify in which folder(s) the search should be performed. If searching in multiple folders, separate them with semicolons, for example: c:\MyProjects;d:\Data.

    Clicking on the button will let you browse and select a folder to replace the currently displayed path. On the other hand, the button will let you browse and select a path that will be appended, with a semicolon, to the existing search path(s).

    By default this field contains Current Asset Web, which is a special value representing the current solution's Asset Web folder. Use this value if searching for behaviors in your solution.

    Options in this section are:

    • Include sub-folders: If checked, the search is recursive, meaning that all sub-folders of the specified folders will also be searched.
    • Include hidden folders: If checked, hidden folders will be searched.
  3. Look at these file types

    This is where you specify which files to search in. The default is *.lua;*.py, which means that all LUA and Python files will be searched.

    Standard wildcard characters can be used:

    • Asterisk (*): Represents any number of any characters. For example test*.py will search all files starting with "test" and ending with the ".py" file extension, while *test*.py will search all files containing "test" in their filename and ending with the ".py" file extension.
    • Question Mark (?): Represent any single character. For example, test???.py will search all files starting with "test", followed with any 3 characters, and ending with the ".py" file extension.

    You can specify multiple filename patterns by separating them with semicolons.

    Additional options are available:

    • Include hidden files: If checked, hidden files will be searched.
    • Skip files bigger than: If checked, files that are larger than the specified size (in kilobytes) will be skipped. We suggest you leave this option enabled to avoid accidentally searching in very large files (such as executables or zip files) which may slow down the search and use a lot of memory.
  4. Buttons
    • Reset...: Lets you reset all of the options to their default values.
    • Find All: Starts the search.
    • Stop Find: Available only while a search is currently being performed, this button lets you stop that search.

Find Results View

This view, which opens automatically when starting a search, displays search results. It is also accessible from the View menu.

Let's break down the view in 4 sections:

Find in Files Results

  1. Search Parameters: This section gives you an overview of the parameters that were used to perform this search. Note that this is also available as the first line of the search results themselves.
  2. Toolbar: Buttons let you:
    • Find...: Open the Find in Files dialog to start a new search.
    • Stop: Stop the search that is currently being performed.
    • Clear All: Clear search results.
    • Auto Scroll: Turn automatic scrolling on or off. When enabled, the text control automatically scrolls to the last line every time new results are displayed.
  3. Results: This is where the actual search results are displayed. Here you can:
    • Double-click on a match to open that file in the Code Editor and automatically move the caret to the appropriate line.

      Note

      This works only for LUA and Python files that are part of your Asset Web. Double-clicking on results for any other file has no effect.
    • Select and copy text using the mouse and standard keyboard shortcuts.
    • Navigate using the Home and End keys to go to the beginning and end of the current line or Ctrl+Home and Ctrl+End to go to the beginning and end of the results. Page Up, Page Down and the Arrow Keys also work as usual.
    • Search for text within this text box using:
      • Ctrl+F to open a small Find dialog that lets you perform simple text searches within the text box.
      • F3 and Shift+F3 to search the same text again forward or backward.
      • Ctrl+F3 and Ctrl+Shift+F3 to search for the currently selected text, or for the word under the caret if there is no selection, forward or backward.
  4. Search Statistics: Displays the current status (Searching, Cancelled or Finished Searching) as well as information such as the number of files that were searched, how many of them contained at least one match, and the total number of matches found so far. Note that this information is also available, once searching is finished or cancelled, as the last line in the results text box.
