Adding/Removing Property Visualizers from an Entity Model

You can add property visualizers to a model to define how properties should be visually represented in World Builder. A property visualizer is a graphical view of various properties on an entity to allow easy visualization of such things as attack radius, walk range, trigger bounds, and so on. You can add any number of property visualizers to a model. Properties can also be easily removed when they are no longer required.

To add a property visualizer to an entity model

  1. Load an entity model into the Mix-in Editor. (You can also select a model in a diagram to load it into the Entity Model Editor.)

    A list of the model's properties and behaviors is displayed in the Entity Model Editor.

  2. In the Property Visualizers section of the Entity Model Editor, click the Add button. (You can also add a new property by right-clicking a row in the Property Visualizer list and selecting Insert from the context menu.)

  3. A new property visualizer is added to the Property Visualizer list.

  4. In the Name field, type a name that best describes the property.
  5. From the Type list, select the type of property visualizer that you want to create. There are numerous types of visualizers, including Point, Connection, and Sphere.
  6. Click the Expand arrow beside the property visualizer name.

    Several new fields are displayed that allow you to control the property dependencies of the visualizer.

To remove a property visualizer from an entity model

  1. Load an entity model into the Mix-in Editor. (You can also select a model in a diagram to load it into the Entity Model Editor.)

    A list of the model's properties and behaviors is displayed in the Entity Model Editor.

  2. From the Property Visualizer list, select the property visualizer that you want to remove. (Use Ctrl+click to select multiple visualizers.)
  3. Click the Remove button or press the Delete key.

The property visualizer is removed from the list and is no longer assigned to the entity model.
