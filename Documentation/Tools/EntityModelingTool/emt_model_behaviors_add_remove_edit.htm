Adding/Removing/Editing Behaviors from Entity Models

You can add scripted behaviors to a model to define an entity's actions in game. Behaviors are basically executable code that accesses and modifies a model's properties. At any point, you can edit the script file that defines the behavior directly in EMT. You can also remove behaviors from a model when they are no longer required.

To add a behavior to an entity model:

  1. Load an entity model into the Mix-in Editor. (You can also select a model in a diagram to load it into the Entity Model Editor. )

    A list of the model's properties and behaviors is displayed in the Entity Model Editor.

  2. In the Behaviors section of the Entity Model Editor, click the Add button. (You can also add a new behavior by right-clicking a row in the Behavior list and selecting Insert from the context menu.)

    A new behavior is added to the Behavior list.

  3. In the Name field, type a name that best describes the behavior.
  4. From the Target list, select the language in which the behavioral script is written. Several scripting languages are supported, including Lua, C, and Python.
  5. Click the Expand arrow beside the behavior name.

    Several new fields are displayed that help you further define the behavior.

  6. From the Traits list, select zero or more traits that you want associated with the behavior. The traits restrict how the behaviors are called.
  7. In the Description field, enter additional information that further describes the behavior.

To edit a model's behavior

  1. Load an entity model into the Mix-in Editor. (You can also select a model in a diagram to load it into the Entity Model Editor.)

    A list of the model's properties and behaviors is displayed in the Entity Model Editor.

  2. From the Behaviors list, select the behavior that you want to edit.
  3. Do one of the following:
    • Click the Edit button.
    • Right-click the behavior in the list and select Edit from the context menu.

    The script file opens in the Editor. (If a script file does not already exist, one is created for you automatically.)

  4. Edit the script and then click the Save Content icon in the toolbar.

To remove a behavior from an entity model:

  1. Load an entity model into the Mix-in Editor.

    A list of the model's properties and behaviors is displayed in the Entity Model Editor.

  2. From the Behaviors list, select the behavior that you want to remove.
  3. Click the Remove button.

    The behavior is removed from the list and is no longer assigned to the entity model.
