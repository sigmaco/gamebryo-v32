Overriding Mix-in Properties and Behaviors

When Mix-ins are created between models, the parent model's properties and behaviors are mixed with those of their children. These property values and behavior scripts can be mixed as is using their default values or they can be overridden to further differentiate a model or preset from others inheriting the same properties and behaviors.

When you remove the override on a property value or behavior, it automatically reverts back to the original value inherited from the parent.

To override mix-in properties

  1. Load an entity model into the Mix-in Editor. (You can also select a model in a diagram to load it into the Entity Model Editor.)

    A list of the model's properties and behaviors is displayed in the Entity Model Editor.

  2. Bring forward the Mix-in Properties section of the Entity Model Editor.

    A list of properties inherited from the model's parents is displayed.

  3. To override a mix-in property, do one of the following:
    • Select the check box beside the property name and then modify the property value.
    • Modify the property value directly. As soon as you modify the property value of a Mix-in property, the check box becomes checked.

      After a property has been overridden, you can also modify a property's traits.

To override mix-in behaviors

  1. Load an entity model into the Mix-in Editor. (You can also select a model in a diagram to load it into the Entity Model Editor.)

    A list of the model's properties and behaviors is displayed in the Entity Model Editor.

  2. Bring forward the Mix-in Behaviors section of the Entity Model Editor.

    A list of behaviors inherited from the model's parents is displayed.

  3. To override a mix-in behavior, select the check box beside the behavior name and then modify the Target value.

    After a behavior has been overridden, you can also modify a behavior's traits.

To remove property and behavior overrides

  1. Load an entity model into the Mix-in Editor. (You can also select a model in a diagram to load it into the Entity Model Editor.)

    A list of the model's properties and behaviors is displayed in the Entity Model Editor.

  2. Bring forward the Mix-in Properties or Mix-in Behaviors section of the Entity Model Editor.
  3. To remove an override, clear the check box beside the property or behavior name.

    The property value or behavior target returns to the value inherited from the parent model.
