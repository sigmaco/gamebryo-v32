Defining Entity Model Properties

After a model is created, you can begin defining its properties. A property is a member variable of an entity model and has a data type, and an initial or default value when the entity is initialized. Properties also have traits associated with them, which are used to control automated management of the properties, such as whether they are persisted, or can be accessed on a client.

You can define any number of properties. The following table shows you a few examples of the types of properties that can be assigned to different models along with their names and default values.

Model Name Property Name Property Type Property Default Value
Placeable Position Point3 0, 0, 0
Renderable IsVisible Boolean True
Level NifAsset Asset Scene_proxy.nif

If a model has been mixed with another model, its properties will be combined with those of the parent model. You can leave these mix-in properties at their default values or you can override them to further customize the model.
