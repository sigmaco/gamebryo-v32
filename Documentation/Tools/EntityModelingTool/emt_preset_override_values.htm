Overriding Property Values

Because several presets can inherit their properties and behaviors from the same parent entity model, it is important that you be able to customize property values to differentiate one preset from another. To do so, you simply need to override the parent model's property values and then define a new value.

When you remove the override on a property value, it automatically reverts back to the original value inherited from the parent model.

To override a parent model's property values

  1. Load a preset into the Mix-in Editor. (You can also select a preset in a diagram to load it into the Entity Model Editor.)

    A list of the properties inherited from the parent model is displayed in the Entity Model Editor.

  2. To override a mix-in property, do one of the following:
    • Select the check box beside the property name and then modify the property value.
    • Modify the property value directly. As soon as you modify the property value of a Mix-in property, the check box becomes checked.

Note

When you override a property value, the property name and value are automatically displayed in the preset box in the Mix-in Editor.
