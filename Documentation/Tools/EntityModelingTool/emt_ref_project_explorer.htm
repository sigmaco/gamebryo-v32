EMT Project Explorer

The EMT Project Explorer is the primary means for organizing and accessing entity modeling content. By default, it is visible in a left-docked pane when an EMT project is opened.

Entity models and presets can be dragged from this view and dropped into Diagram Editor and Mix-in Editor.

A thumbnail image and description for the selected content is displayed at the bottom of the view.

The view has two modes for working with EMT content. They operate similarly to how Windows Explorer works with the file system.

Browse Mode

The Browse tab is used for navigating a virtual folder hierarchy of EMT content. This may be done exclusively in the Contents sub-pane to conserve screen space. Alternatively, the Folders sub-pane can be added on the left for greater flexibility. There is a draggable splitter between the two sub-panes to specify how the space should be shared.

Project Explorer - Browse mode

The root of the hierarchy contains the open EMT project and read-only links to all projects it references. Within these projects are entity models, presets, diagrams, and virtual folders. Virtual folders can in turn contain additional EMT content, including virtual subfolders.

There are several controls for navigating and editing the content hierarchy in Browse Mode.

  • Use the Location breadcrumb control at the top of the pane to quickly move between folders and referenced EMT projects.
  • Use the Filter edit box to specify a substring that limits the display of items at the current level in the hierarchy.
  • Use the Folders sub-pane to expand, collapse, and view different folders and projects.
  • Double-click folders in the Contents sub-pane to navigate into them.
  • Use the backspace key to navigate up one level in the virtual hierarchy.
  • Drag and drop items within or between the two sub-panes to reorganize content.

Note

It is important to remember that the EMT Project Explorer contains the items in one open EMT project, plus read-only access to items in all referenced EMT projects. Project references are managed in the Game Solution view. Items cannot be moved from the open EMT project into a referenced project. Similarly, items cannot be moved between referenced projects.

Organizational work done in the EMT Project Explorer will also be leveraged by the World Builder Palette view, which uses a similar UI to present the same virtual hierarchy. However, the virtual hierarchy can only be modified in the Entity Modeling Tool.

Search Mode

The Search tab is used to quickly locate EMT content that might be scattered throughout an EMT project or even multiple referenced projects. The query-building controls provide several different criteria for searches. Matching entity models and presets are returned in the Results list.

Project Explorer - Search mode

A checkbox to the left of each option determines which search criteria are active. The Advanced rollout can be expanded for access to additional search options. The Execute Query button performs the search. If the EMT Project Explorer view has focus, simply hitting the enter key will also perform the search. The following search parameters can be specified.

Search Parameter Details
Search Under Specify an EMT project to search, or uncheck to search all referenced projects.
Name Includes Specify a substring to find in model and preset names.
Has Mixin Specify a model name substring that must be mixed in.
Has Property Specify a property name substring that must be present.
Has Behavior Specify a behavior name substring that must be present.

Results are listed with Name and Virtual Path. The path reflects the EMT project and virtual folders that contain an item.

Commands

EMT content items listed in both Browse and Search tabs provide context menus with a variety of commands. The available commands depend on the type and context of content selected. EMT Project Explorer supports multi-select, which will limit the available commands. Note also that different commands are available when the selected content is from a referenced EMT project.

Command Details
Open Opens the selected item for viewing or editing. Models and presets open in the Mix-in Editor. Diagrams open in the Diagram Editor. Open is the default command when double-clicking an item.
Open in Project Opens the selected item from a referenced project in its own project context. This is necessary for editing referenced items.
Delete Deletes the selected items from the project.
Rename Renames the selected item. Naming restrictions are enforced where appropriate.
View Asset Opens an external tool for viewing the NIF or KFM asset associated with the selected entity model or preset. This feature is configured in user settings.
Browse to Location When used in Search Mode, this command will transition the view to Browse mode, arriving at the virtual path of the selected item.
Add > New Model Creates a new entity model in the current virtual folder.
Add > New Preset Creates a new preset in the current virtual folder.
Add > New Diagram Creates a new diagram in the current virtual folder.
Add > New Folder Creates a new virtual folder as a child of the current virtual folder.
Revision Control > Add / Edit Adds new files to revision control or checks them out if they already exists.
Revision Control > Refresh Updates the displayed status of all revision-controlled files.

Thumbnails View

Each mode of EMT Project Explorer can be configured to use either of two visualizations. These visualizations are toggled via the Views dropdown. The default visualization is Details.

Thumbnails view can be useful for quickly locating entity models and presets that have associated visual assets.

Project Explorer - Thumbnails View

The Zoom slider controls the size of the thumbnail images, trading image detail for viewable image count. Hovering over the label for a thumbnail will display a tooltip with the full name and virtual path of the item.

The thumbnails are provided via the NiThumbnailExtractor COM DLL.
