Converting Particles to Gamebryo LightSpeed 3.0

In general, particle systems that are created in DCC tools and loaded from file will not require any conversion. However, changes must be made to convert particle systems that are created in code.

In the following classes, the Create function has a new set of arguments that control new particle system features:

  • NiPSParticleSystem
  • NiMeshParticleSystem

In the following classes, the constructor has a new set of arguments that control additional emitter properties.

  • NiPSEmitter
  • NiPSVolumeEmitter
  • NiPSMeshEmitter
  • NiPSBoxEmitter
  • NiPSSphereEmitter
  • NiPSCylinderEmitter
  • NiPSSpawner

Conversion at a glance

NiPSParticleSystem::Create()

  • Add a normal align method
  • Add a normal align direction vector
  • Add an up align method
  • Add an up align direction vector
  • Add a boolean for if it has a living spawner
  • Add a boolean for if it has animated textures

NiPSEmitter::NiPSEmitter()

  • Add a speed flip ratio value
  • Remove the color value
  • The concept of "radius" has been changed to "size"

NiPSSpawner::NiPSSpawner()

  • Remove spawned speed chaos value
  • Add spawn speed factor value
  • Add spawn speed factor variance value
  • Add a boolean for if it has relative speed
