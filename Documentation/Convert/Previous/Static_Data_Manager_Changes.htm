Static Data Manager Changes

The NiStaticDataManager class and associated macros have been modified in Gamebryo 2.5 to provide control over the order in which static data is initialized.

Classes that have static data initialization needs must still register with the NiStaticDataManager::AddLibrary function. This function now takes the name of the library (as a string) and a list of library names on which the library depends (a space-separated string of names). The NiSystem library should not appear in the list - all libraries depend on it and the dependence is implicit. Only libraries that have SDM functions themselves should be on the list. That will be a subset of the libraries required to successfully link with a particular library.

Libraries that dynamically unlink themselves (mostly shader libraries) should note that the arguments to NiStaticDataManager::RemoveLibrary have changed to be the library name.

The NiImplementSDMConstructor and NiImplementDLLMain macros have been changed appropriately. These macros transparently implement the changes above.
