Converting to Gamebryo LightSpeed 3.1.1 Overview

Some Gamebryo LightSpeed 3.1 interfaces have changed in Gamebryo LightSpeed 3.1.1. These changes are listed in this section to help you identify actions you might need to take to ensure a smooth conversion path from Gamebryo LightSpeed 3.1 to Gamebryo LightSpeed 3.1.1.

Note

For an overview of the features in this release, see What's New in Gamebryo LightSpeed. Additional discussion of new features, fixed issues, and changes in behavior are outlined in the Release Notes.

The following notes apply to the 3.1-to-3.1.1 conversion path:

  • PhysX Changes
  • Particle Emitter Changes

Changes Related to the PhysX SDK Update

The PhysX SDK for Gamebryo 3.1.1 is PhysX 2.8.1. Some Gamebryo-PhysX API changes were needed to allow for easy usage of PhysX 2.8.3, in case developers require that.

  • The NiPhysXClothDesc class has some additonal parameters to match those added in the PhysX 2.8.3 SDK. Code that uses the method NiPhysXClothDesc::SetConstantParameters must be updated to specify the additional parameters. See the reference documentation for the new method signature. These changes apply on all platforms but only affect behavior on the PS3.
  • The PhysX Mesh Converter Tool Plug-in has been removed from the product. The PhysX SDK no longer includes the libraries necessary to build it. The script should only have been used in batch-updating assets from Gamebryo versions prior to 2.5. Should you require such functionality, contact Gamebryo support.

Particle Emitter Changes

The signature of the NiPSEmitter::ComputeInitialPositionAndVelocity method has been changed to take the Emitter-to-Particle System transformation. The corresponding method in NiPSVolumeEmitter has also been changed. Any custom particle emitters that implement these methods must be updated. All derived emitters must also now implement the NiPSEmitter::GetEmitterObj method, returning NULL if there is no emitter object for defining a transformation.
