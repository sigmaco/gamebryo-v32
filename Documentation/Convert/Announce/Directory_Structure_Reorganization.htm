Gamebryo LightSpeed 3.0 Directory Structure Reorganization

The release of Gamebryo LightSpeed 3.0 includes a significant reorganization of the directory structure of the engine.

The most immediate change that you will notice is that the scope of Gamebryo as an engine has greatly expanded. The portions of the runtime code that comprised previous versions of Gamebryo, including the libraries that handled scene graph management and rendering, now represent a single subsystem in Gamebryo, known as the Core Runtime subsystem. There are two other subsystems that have been added to the engine to handle other areas of functionality: Foundation and Game Framework.

In the process of adding this new functionality, a number of existing portions of the Gamebryo code base have been moved around. This document describes where each library or application in the Gamebryo 2.x code base can be found in Gamebryo LightSpeed. Additional information on platform-specific libraries can be found on these pages:

  • Directory Structure Reorganization - PS3
  • Directory Structure Reorganization - Xbox360

An overview of the new structure can be found in the Gamebryo Orientation doc.

SDK directory changes

Applications whose build projects are set up to include and link files from the Gamebryo SDK will need to be changed to work with the new SDK. All existing Gamebryo header files remain in the same location in the SDK, but the directory containing the Win32 static libraries have been renamed from SDK\Win32\Lib\VC*0\DebugLib, ReleaseLib, and ShippingLib to Debug, Release, and Shipping. Note that the location is the same, but the "Lib" suffix was removed.

In addition, the locations of other files in the SDK have also changed.

  • Shader libraries have been moved from SDK\Win32\Shaders\Lib to SDK\Win32\DLL\ShaderLibs
  • Shader files have been moved from SDK\Win32\Shaders\Data to Media\Shaders\Data
  • Tool Plugins have been moved from SDK\Win32\ToolPlugins\Lib to SDK\Win32\DLL\ToolPlugins
  • Mesh Profiles have been moved from SDK\Win32\ToolPlugins\Data\MeshProfiles to Media\ToolPluginsData\MeshProfiles
  • Data and executables for the art tools' console on-target viewers or the in-memory Windows viewer have been moved from SDK\Win32\ToolPlugins\Data to SDK\Win32\DLL\ToolPlugins\ConsoleViewers

Engine and Tool source changes

  • NiApplication and NiSample have been moved from AppFrameworks to Source\CoreRuntime\AppFrameworks
  • NiTerrain has been moved from AppFrameworks to Source\CoreRuntime\EngineLibs
  • NiTerrainPhysX has been moved from AppFrameworks to Source\CoreRuntime\PhysXLibs
  • AppFrameworks\UtilityLibs have been moved to Source\CoreRuntime\UtilityLibs
  • CoreLibs have been moved to Source\CoreRuntime\EngineLibs
  • CoreLibsPhysX have been moved to Source\CoreRuntime\PhysXLibs
  • NiMilesAudio has been moved from CoreLibs to Source\CoreRuntime\UtilityLibs
  • NiBinkVideo has been moved from Samples\Demos\VideoTexture to Source\CoreRuntime\UtilityLibs
  • The Max, Maya, and XSI plugins have been moved from ToolLibs\ArtPlugins to Source\Tools\ArtPlugins
  • Shader libraries have been moved from ToolLibs\D3DShaderLibs to Source\CoreRuntime\ShaderLibs
  • Tool plugins have been moved from ToolLibs\ToolPlugins to Source\Tools\ToolPlugins
  • The remaining ToolLibs have been moved to Source\CoreRuntime\ToolPlugins

Samples directory changes

  • The GUI files used by NiUserInterface have been moved from Samples\Data to Media\Samples\SampleUI
  • The Demos directory has been renamed to GraphicsTechDemos
  • The FullDemos directory has been renamed to GameDemos
  • Tutorials have been moved to Samples\Tutorials\Graphics
  • TutorialsPhysX have been moved to Samples\Tutorials\PhysX
  • SceneDesignerPlugins have been moved to Samples\SampleLibs\SceneDesignerPlugins
  • ToolPlugins\SamplePluginsDLL has been moved to Samples\SampleLibs\SamplePluginsDLL

Environment Variable Changes

2.x Environment Variable 3.0 Equivalent
%EGB_PATH% %EMERGENT_PATH%
%EGB_TOOL_PLUGIN_PATH% %EMERGENT_PATH%\sdk\Win32\DLL\ToolPlugins
%EGB_SHADER_LIBRARY_PATH% %EMERGENT_PATH%\Media\Shaders\Data

Additionally, as the %EGB_SHADER_LIBRARY_PATH% no longer exists, it cannot be set to point at your game's shader directory so that shaders from a game can be used directly by Gamebryo's tools. In order to make a game's shaders visible to the toolchain, they should be copied into the new shader directory under the Gamebryo root.
